`position` 是 CSS 中**控制元素定位规则与文档流行为**的核心属性，它决定了元素的位置计算基准、是否脱离正常文档流，以及与其他元素的堆叠关系，通常配合 `top/right/bottom/left`（偏移属性）和 `z-index`（层级属性）使用。

### 一、核心属性值详解

`position` 有 5 个核心取值，各自的定位规则和文档流行为差异极大，下面逐一说明：

#### 1. static（静态定位，默认值）

- 核心规则：元素完全遵循**正常文档流**布局，块级元素从上到下排列，行内元素从左到右排列。
- 关键特性：`top/right/bottom/left`、`z-index` 属性完全失效，无法通过这些属性调整元素位置和层级。
- 适用场景：无需特殊定位的普通元素，页面绝大多数元素的默认状态。

#### 2. relative（相对定位）

- 核心规则：**不脱离文档流**，元素原本的占位完全保留，仅相对于**自身在文档流中的原始位置**进行偏移。
- 关键特性：
    
    - 偏移后，原位置不会被其他元素挤占，页面整体布局不发生变化。
    - 最常用的作用是作为 `absolute` 绝对定位元素的**定位父级（包含块）**。
    - `top/right/bottom/left`、`z-index` 完全生效。
    
- 示例：`top: 10px; left: 20px` 表示元素相对于自身原始位置，向下偏移 10px，向右偏移 20px。
- 适用场景：元素位置微调、为绝对定位子元素提供定位基准。

#### 3. absolute（绝对定位）

- 核心规则：**完全脱离文档流**，元素原始占位消失，不影响其他元素的正常布局；位置相对于**最近的 `position` 非 `static` 的祖先元素**计算，若找不到符合条件的祖先，则相对于根元素（html / 浏览器视口）定位。
- 关键特性：
    
    - 脱离文档流后，块级元素宽度默认由内容撑开，不再默认撑满父容器。
    - 偏移值相对于定位父级的内边距边界（padding-box）计算。
    - `z-index` 生效，可自由控制元素堆叠层级，重叠时 `z-index` 数值越大，层级越靠上。
    
- 适用场景：弹窗、悬浮标签、覆盖层、跟随父容器的固定装饰元素。
- 常见坑：若未给父级设置非 `static` 的 `position`，元素会相对于整个页面定位，滚动时位置会发生偏移。

#### 4. fixed（固定定位）

- 核心规则：**完全脱离文档流**，位置默认相对于**浏览器视口（viewport）** 计算，页面滚动时，元素始终固定在视口的指定位置，不随滚动条移动。
- 关键特性：
    
    - 不受普通祖先元素的影响，唯一例外：若祖先元素设置了非 `none` 的 `transform`/`perspective`/`filter` 属性，`fixed` 元素会相对于该祖先元素定位，而非视口（高频踩坑点）。
    - 偏移值相对于视口的边缘计算。
    - `z-index` 生效，常用于页面最高层级的固定元素。
    
- 适用场景：固定顶部导航栏、回到顶部按钮、悬浮客服弹窗、固定底部操作栏。

#### 5. sticky（粘性定位）

- 核心规则：**融合了 relative 和 fixed 的特性**，元素在滚动到指定阈值前，保持 `relative` 行为（不脱离文档流、保留占位）；当滚动到设定的阈值时，切换为 `fixed` 行为，固定在视口的指定位置。
- 关键特性：
    
    - 必须设置 `top/right/bottom/left` 中的至少一个值，否则粘性效果不生效。
    - 仅在**最近的可滚动祖先元素**内生效，若该祖先设置了 `overflow: hidden/auto/scroll`，粘性效果会失效。
    - 粘性效果仅在父容器的可视范围内生效，父容器滚出视口后，粘性元素会跟随父容器一起滚走。
    
- 适用场景：表格吸顶表头、分类导航吸顶、侧边栏滚动固定。

### 二、核心配套属性与关键注意事项

1. **偏移属性（top/right/bottom/left）**
    
    仅当 `position` 取值非 `static` 时生效，用于定义元素的偏移量，支持 px、%、em、vw/vh 等单位，正负值均可使用。
    
2. **z-index（堆叠层级）**
    
    仅对 `position` 非 `static` 的元素生效，用于控制元素的上下堆叠顺序，数值越大，层级越靠上；默认值为 `auto`，同级元素遵循 “后来者居上” 原则。
    
1. **文档流行为汇总**
	
	| 定位类型     | 是否脱离文档流               |
	| -------- | --------------------- |
	| static   | 不脱离                   |
	| relative | 不脱离                   |
	| absolute | 完全脱离                  |
	| fixed    | 完全脱离                  |
	| sticky   | 阈值前不脱离，阈值后不脱离文档流但固定位置 |
    
4. **高频踩坑避坑**
    
    - `sticky` 不生效：优先检查是否设置了 `top` 等值、父级是否有 `overflow: hidden`、父容器高度是否小于粘性元素。
    - `fixed` 不相对于视口定位：检查祖先元素是否有 `transform` 相关属性。
    - `z-index` 层级混乱：受堆叠上下文影响，父级堆叠上下文层级低，子元素 `z-index` 再大也无法超越外部更高层级的元素。
