CSS 全称是**层叠样式表（Cascading Style Sheets）**，「层叠」是 CSS 的核心机制：当多个样式规则同时作用于同一个 HTML 元素时，浏览器会按照一套固定的优先级规则，决定最终生效的样式。

优先级分为两大核心维度：**顶层优先级（来源与重要性）** > **选择器权重优先级**，再辅以源码顺序、继承规则作为补充，共同构成完整的层叠体系。

## 一、顶层优先级：来源与重要性（从高到低）

这是优先级的第一判断标准，优先级高的规则，无论选择器权重多低，都会覆盖优先级低的规则。以下是**W3C 官方标准的完整排序（越靠前优先级越高）**：

| 优先级层级  | 类型                      | 说明                                              |
| :----- | :---------------------- | :---------------------------------------------- |
| 1（最高）  | 正在执行的过渡样式（transition）   | 正在运行的过渡动画样式，优先级最高                               |
| 2      | 用户样式表的 `!important` 声明  | 用户在浏览器中自定义的带 `!important` 的样式（如视力障碍用户设置的强制字体大小） |
| 3      | 内联 `!important` 样式      | 写在 HTML 标签 `style` 属性中，且带 `!important` 的样式      |
| 4      | 开发者样式表的 `!important` 声明 | 我们编写的 CSS 中，带 `!important` 的样式声明                |
| 5      | 正在执行的动画样式（@keyframes）   | 正在运行的关键帧动画样式，优先级高于普通样式                          |
| 6      | 内联普通样式                  | 写在 HTML 标签 `style` 属性中的普通样式（无 `!important`）     |
| 7      | 开发者普通样式                 | 我们编写的外部 / 内部 CSS 样式，优先级由「选择器权重」决定               |
| 8      | 用户普通样式                  | 用户在浏览器中自定义的普通样式                                 |
| 9      | 浏览器用户代理默认样式             | 浏览器自带的默认样式（如 ul 的内边距、p 的外边距、a 标签的下划线）           |
| 10（最低） | 继承样式                    | 元素从父元素继承来的样式，优先级最低                              |

### 关键说明：`!important`

`!important` 是 CSS 中的优先级强制提升标记，写在属性值末尾，会打破常规的层叠规则，仅低于用户自定义的`!important`和正在执行的过渡样式。

```css
/* 强制覆盖所有同属性的普通样式 */
.title {
  color: red !important;
}
```

> 最佳实践：不到万不得已绝不使用`!important`，它会破坏 CSS 的自然层叠逻辑，导致后期维护和样式调试极度困难。

## 二、核心权重：选择器特异性计算

当两个规则属于同一顶层优先级（比如都是开发者普通样式）时，浏览器会通过**选择器权重（特异性）** 决定谁生效，采用**四位四元组（a,b,c,d）** 计算规则，核心原则是：**从左到右逐级比较，高位数值大的优先级更高，数位之间不进位**。

### 1. 权重四元组定义

|位次|对应选择器类型|计数规则|
|:--|:--|:--|
|a（千位）|内联样式|存在内联样式则 a=1，否则 a=0|
|b（百位）|ID 选择器|统计选择器中 ID 选择器（`#xxx`）的数量|
|c（十位）|类选择器、属性选择器、伪类选择器|统计三者的总数量（`.xxx`、`[type=text]`、`:hover`、`:nth-child()`等）|
|d（个位）|标签选择器、伪元素选择器|统计两者的总数量（`div`、`p`、`::before`、`::after`等）|

### 2. 核心比较规则

1. 先比较高位，高位大的，无论低位数值多少，优先级一定更高；
2. 高位相同，再依次比较低位；
3. **权重永不进位**：100 个类选择器（0,0,100,0），优先级永远低于 1 个 ID 选择器（0,1,0,0），不会出现低位满 10 向高位进位的情况。

### 3. 权重计算示例

|选择器|四元组权重|拆解说明|
|:--|:--|:--|
|`style="color: red"`|(1,0,0,0)|内联样式，a=1|
|`#header .nav a`|(0,1,1,1)|1 个 ID、1 个类、1 个标签|
|`div .list li.active`|(0,0,2,3)|2 个类、3 个标签|
|`[type="checkbox"] + label`|(0,0,1,1)|1 个属性选择器、1 个标签|
|`a:hover::before`|(0,0,1,2)|1 个伪类、1 个标签、1 个伪元素|
|`*`|(0,0,0,0)|通配符，无权重|

### 4. 无权重的选择器

以下选择器**本身不影响权重**，仅内部嵌套的选择器会参与计算：

- 通配选择器 `*`
- 关系选择器：子代`>`、相邻`+`、兄弟`~`、后代空格
- 否定伪类 `:not()`：本身不计权重，括号内的选择器正常计算权重

```css
/* 权重：(0,1,0,0)，:not()本身不影响，仅内部#box计入 */
:not(#box) {
  color: blue;
}
```

## 三、层叠补充规则

### 1. 源码顺序：后来者居上

当两个规则的**顶层优先级、选择器权重完全相同时**，写在后面的样式会覆盖前面的样式。

```css
/* 最终生效颜色：blue，因为写在后面 */
.title {
  color: red;
}
.title {
  color: blue;
}
```

> 外链 CSS 与内联 CSS 的顺序，以它们在 HTML 中 `<link>`和 `<style>`标签的书写顺序为准。

### 2. 继承规则：优先级最低

子元素会自动继承父元素的部分可继承属性（如`color`、`font-size`、`font-family`等），但继承的样式优先级是最低的，哪怕是通配符`*`的样式，也会覆盖继承样式。

```html
<style>
  /* 权重(0,0,0,0)，依然会覆盖父元素继承的color */
  * {
    color: blue;
  }
  .parent {
    color: red;
  }
</style>
<div class="parent">
  <p>我最终是蓝色，不是红色</p>
</div>
```

## 四、常见误区与踩坑点

1. **权重进位误区**：10 个类选择器 ≠ 1 个 ID 选择器。权重是分级比较，低位再多也无法超越高位，这是新手最容易踩的坑。
2. **伪类与伪元素权重混淆**：`:hover`、`:nth-child()` 是伪类，计入十位权重；`::before`、`::after` 是伪元素，计入个位权重，两者优先级不同。
3. **`!important` 覆盖逻辑**：只有同属性的`!important`才能覆盖`!important`，普通样式无论权重多高，都无法覆盖带`!important`的样式。
4. **行内样式无法被外部样式覆盖**：外部 CSS 中，无论 ID 选择器权重多高，都无法覆盖内联普通样式，除非使用`!important`。

## 五、最佳实践

1. **优先使用类选择器**：日常开发以类选择器为主，保持低权重、高复用性，避免滥用 ID 选择器提升权重。
2. **控制选择器长度**：选择器层级尽量不超过 3 层，避免过长的选择器导致权重过高、后期无法覆盖。
3. **用层叠层替代硬拼权重**：现代开发中，使用 CSS 原生`@layer`层叠层主动控制样式优先级，替代靠选择器长度硬拼权重的方式。
4. **`!important` 仅用于兜底**：只在覆盖第三方库内联样式、全局强制兜底等极端场景使用`!important`，禁止日常开发滥用。
5. **善用浏览器调试工具**：Chrome 开发者工具的 Elements 面板中，会直接显示样式的优先级、被覆盖的原因，是调试样式冲突的最佳工具。
