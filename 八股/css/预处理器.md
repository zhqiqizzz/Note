CSS 预处理器是一种**扩展 CSS 语法的工具**，它通过自定义的高级语法（如变量、嵌套、混合、函数等）编写样式代码，再通过编译工具生成标准的 CSS 文件，最终在浏览器中生效。

它的核心价值是**解决原生 CSS 复用性差、维护成本高、缺乏逻辑控制等痛点**，让样式开发更高效、更模块化。

## 一、为什么需要 CSS 预处理器？

原生 CSS 存在以下明显局限：

1. **无变量机制**：颜色、字体等重复值无法统一管理，修改时需全局替换；
2. **选择器冗余**：嵌套层级的选择器需重复写父级，代码冗长；
3. **代码复用困难**：相似样式（如按钮、卡片）只能复制粘贴，难以抽象复用；
4. **缺乏逻辑控制**：无法做条件判断、循环等逻辑操作，动态生成样式困难。

预处理器正是为解决这些问题而生。

## 二、主流 CSS 预处理器对比

目前最常用的预处理器有 **Sass/SCSS**、**Less**、**Stylus** 三种，它们的核心功能相似，但语法和实现方式略有区别：

|特性|Sass/SCSS|Less|Stylus|
|---|---|---|---|
|**诞生时间**|2006 年|2009 年|2010 年|
|**语法风格**|SCSS（兼容 CSS 语法，用 `{}` 和 `;`）/ Sass（缩进语法，无 `{}` 和 `;`）|完全兼容 CSS 语法，扩展简单|极其灵活，可选省略 `{}`、`;`、`:`|
|**实现语言**|Ruby（早期）/ Dart（现在官方推荐）|JavaScript|JavaScript|
|**生态成熟度**|最成熟，社区庞大，Bootstrap 4+ 采用|较成熟，Vue 早期生态常用|相对小众，灵活性高但学习曲线略陡|
|**核心功能**|变量、嵌套、混合、继承、函数、条件 / 循环|变量、嵌套、混合、函数、条件|变量、嵌套、混合、继承、函数、条件 / 循环|

## 三、预处理器核心功能（以 Sass/SCSS 为例，附代码）

以下是预处理器最常用的核心能力，通过示例直观展示其优势：

### 1. 变量（Variables）

用 `$` 定义变量，存储颜色、字体、尺寸等重复值，统一修改更高效。

```scss
// 定义变量
$primary-color: #1890ff;
$font-size-base: 14px;
$border-radius: 4px;

// 使用变量
.button {
  background-color: $primary-color;
  font-size: $font-size-base;
  border-radius: $border-radius;
}

.card {
  border: 1px solid $primary-color;
  border-radius: $border-radius;
}
```

### 2. 嵌套（Nesting）

按 HTML 结构嵌套选择器，避免重复写父级选择器，代码更清晰。

```scss
// 原生 CSS 写法（冗余）
.nav {
  list-style: none;
}
.nav li {
  display: inline-block;
}
.nav li a {
  color: #333;
}
.nav li a:hover {
  color: $primary-color;
}

// SCSS 嵌套写法（简洁）
.nav {
  list-style: none;
  
  li {
    display: inline-block;
    
    a {
      color: #333;
      
      &:hover { // & 表示父选择器（这里是 .nav li a）
        color: $primary-color;
      }
    }
  }
}
```

### 3. 混合（Mixins）

可复用的代码块，支持传参数，类似函数，用于抽象相似样式（如按钮、flex 布局、清除浮动等）。

```scss
// 定义混合：带参数的按钮样式
@mixin button($bg-color, $text-color: #fff) {
  display: inline-block;
  padding: 8px 16px;
  background-color: $bg-color;
  color: $text-color;
  border-radius: $border-radius;
  text-decoration: none;
  
  &:hover {
    opacity: 0.9;
  }
}

// 使用混合
.btn-primary {
  @include button($primary-color); // 传1个参数，用默认文字颜色
}

.btn-danger {
  @include button(#ff4d4f, #fff); // 传2个参数
}
```

### 4. 继承（Extend）

让一个选择器继承另一个选择器的所有样式，减少代码重复（与 Mixins 不同，Extend 会合并选择器，生成的 CSS 更简洁）。

```scss
// 基础按钮样式
.btn-base {
  display: inline-block;
  padding: 8px 16px;
  border-radius: $border-radius;
  text-decoration: none;
}

// 继承基础样式
.btn-primary {
  @extend .btn-base;
  background-color: $primary-color;
  color: #fff;
}

.btn-danger {
  @extend .btn-base;
  background-color: #ff4d4f;
  color: #fff;
}
```

生成的 CSS 会合并选择器：

```css
.btn-base, .btn-primary, .btn-danger {
  display: inline-block;
  padding: 8px 16px;
  border-radius: 4px;
  text-decoration: none;
}
.btn-primary { background-color: #1890ff; color: #fff; }
.btn-danger { background-color: #ff4d4f; color: #fff; }
```

### 5. 函数（Functions）

内置或自定义函数，用于处理颜色、计算数值等（Sass 内置了大量颜色函数、数学函数）。

```scss
// 内置函数：颜色变亮/变暗
$primary-light: lighten($primary-color, 10%); // 颜色变亮10%
$primary-dark: darken($primary-color, 10%);   // 颜色变暗10%

// 自定义函数：计算 rem（假设基准是 16px）
@function px-to-rem($px) {
  @return $px / 16px * 1rem;
}

// 使用函数
.card {
  padding: px-to-rem(20px); // 转换为 1.25rem
  background-color: $primary-light;
}
```

### 6. 条件与循环（Logic Control）

支持 `@if`/`@else` 条件判断，`@for`/`@each`/`@while` 循环，动态生成样式（如网格系统、间距类）。

```scss
// 条件判断：根据主题色调整文字颜色
@mixin theme-button($bg-color) {
  background-color: $bg-color;
  
  @if lightness($bg-color) > 70% {
    color: #333; // 背景色浅时用深色文字
  } @else {
    color: #fff; // 背景色深时用白色文字
  }
}

// 循环：生成间距类（.mt-1, .mt-2, ..., .mt-5）
@for $i from 1 through 5 {
  .mt-#{$i} {
    margin-top: $i * 8px; // 8px 基准，生成 8px、16px...40px
  }
}
```

## 四、预处理器工作流程

使用预处理器需要经过 “编写 → 编译 → 输出” 三个步骤：

1. **编写**：用 `.scss`/`.sass`（Sass）、`.less`（Less）、`.styl`（Stylus） 后缀文件编写代码；
2. **编译**：通过工具将预处理器代码转换为标准 CSS：
    
    - **命令行工具**：Sass CLI、Less CLI；
    - **构建工具**：Webpack（配合 sass-loader/less-loader）、Gulp（配合 gulp-sass/gulp-less）；
    - **编辑器插件**：VS Code 的 “Live Sass Compiler” 等（适合小型项目）；
    
3. **输出**：生成 `.css` 文件，在 HTML 中引入即可。

## 五、预处理器的优缺点

### 优点

- **提高开发效率**：变量、混合、嵌套等减少重复代码，开发速度提升；
- **增强可维护性**：统一管理变量，修改全局样式只需改一处；
- **模块化能力**：通过 `@import` 拆分文件（如 `_variables.scss`、`_mixins.scss`），代码结构更清晰；
- **生态丰富**：大量开源库（如 Sass 的 Compass、Less 的 Lesshat）可直接复用。

### 缺点

- **需要编译**：增加了构建流程，小型项目可能觉得繁琐；
- **学习成本**：需要额外学习预处理器语法；
- **调试稍复杂**：编译后的 CSS 与源文件不一致，需配合 **Source Map** 才能直接调试源文件。

## 六、补充：CSS 后处理器（PostCSS）

预处理器是 “先写高级语法，再生成 CSS”，而 **PostCSS** 是 “对已有的 CSS 进行处理”，两者常配合使用。

PostCSS 的核心是插件机制，常用插件包括：

- **Autoprefixer**：自动添加浏览器前缀（如 `-webkit-`、`-moz-`）；
- **PostCSS Preset Env**：让你提前使用未来的 CSS 语法（如 CSS 变量、嵌套）；
- **cssnano**：压缩 CSS 文件。

**典型工作流**：

编写 SCSS → 编译为 CSS → PostCSS（Autoprefixer + 压缩）→ 最终 CSS。

## 七、总结

CSS 预处理器是现代前端开发的常用工具，其中 **Sass/SCSS** 是目前生态最成熟、使用最广泛的选择。如果你刚开始学习，建议优先掌握 SCSS（兼容 CSS 语法，学习成本低），再根据项目需求了解 Less 或 Stylus。

配合 PostCSS 使用，能进一步提升样式开发的效率和兼容性。
