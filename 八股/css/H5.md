## 一、先分别搞懂两个核心概念

### 1. H5 是什么？

我们常说的 **H5**，其实是「**HTML5 移动端网页**」的简称（不是单指 HTML5 这一门技术）。

#### 人话理解

H5 就是**在手机浏览器、微信、APP 内嵌网页里打开的网页**，不需要下载安装，点开就能用。

- 典型例子：微信里的活动页、手机百度的搜索结果页、APP 里的内嵌商城页。
- 核心技术栈：HTML5（结构）+ CSS3（样式）+ JavaScript（交互）。
- 最大特点：**跨平台**（安卓、苹果都能用）、**无需下载**（即用即走）。

#### H5 开发最头疼的问题：移动端适配

手机的屏幕尺寸千差万别（小到 iPhone SE 的 4.7 英寸，大到 iPad 的 12.9 英寸，还有各种安卓机型的不同分辨率），H5 开发的核心需求就是：**让同一个页面，在不同尺寸的手机上，都能显示得 “比例协调、不跑偏”**。

而 `rem`，就是解决 H5 移动端适配的最经典工具之一。

---
### 2. rem 是什么？

`rem` 是 CSS 中的一个**相对长度单位**，它的全称是 `root em`，意思是**相对于根元素（`<html>` 标签）的字体大小（`font-size`）的单位**。

#### 先对比一下：px、em、rem 的区别

为了让你更清楚 `rem` 的特点，我们对比三个最常用的长度单位：

| 单位       | 是什么    | 相对于谁                           | 特点                      |
| :------- | :----- | :----------------------------- | :---------------------- |
| `px`（像素） | 绝对长度单位 | 无，固定大小                         | 最直观，但在不同屏幕上大小固定，适配差     |
| `em`     | 相对长度单位 | 父元素的 `font-size`               | 相对，但父元素一变，子元素全乱，容易 “套娃” |
| `rem`    | 相对长度单位 | **根元素 `<html>` 的 `font-size`** | 只相对于根元素，稳定，不会乱，非常适合适配   |
#### rem 的核心规则

`rem` 的大小完全由 `<html>` 标签的 `font-size` 决定：

- 如果 `<html>` 的 `font-size` 是 `16px`（浏览器默认值），那么 `1rem = 16px`，`2rem = 32px`；
- 如果 `<html>` 的 `font-size` 改成 `100px`，那么 `1rem = 100px`，`2rem = 200px`。

---

## 二、核心：rem 怎么解决 H5 移动端适配？

`rem` 适配 H5 的核心思路非常简单，一句话总结：

> **根据不同手机的屏幕宽度，动态修改 `<html>` 的 `font-size`，页面里所有元素的尺寸都用 `rem` 来写，这样元素的大小会随着屏幕宽度自动 “等比例缩放”，实现适配。**

我们用一个**贯穿式的例子**，一步一步讲明白这个过程。

### 第一步：确定设计稿的基准宽度

通常 UI 设计师给的 H5 设计稿，会以**750px**（iPhone 6/7/8 的 2 倍屏宽度，行业通用基准）作为基准宽度。

假设我们拿到的设计稿是 **750px 宽**，设计稿上有一个按钮，宽度是 **300px**，高度是 **80px**。

### 第二步：设定一个 “方便计算的基准 font-size”

为了计算方便，我们通常会在**750px 的基准屏幕**上，把 `<html>` 的 `font-size` 设为 **100px**（为什么是 100px？因为好算！100px = 1rem，设计稿上的 px 直接除以 100 就是 rem）。

- 基准屏幕：750px
- 基准 font-size：100px
- 此时：1rem = 100px

那么设计稿上的按钮：

- 宽度：300px = 300 / 100 = **3rem**
- 高度：80px = 80 / 100 = **0.8rem**

我们在 CSS 里就这么写：

```css
.btn {
  width: 3rem;   /* 对应设计稿 300px */
  height: 0.8rem; /* 对应设计稿 80px */
  background: #ffcc00;
}
```

### 第三步：动态修改 `<html>` 的 font-size（核心适配逻辑）

现在问题来了：如果手机屏幕不是 750px，而是 375px（iPhone SE），或者 1080px（大屏安卓），怎么办？

我们需要用**JavaScript**，根据手机的实际屏幕宽度，动态计算并设置 `<html>` 的 `font-size`，让元素等比例缩放。

#### 核心计算公式

```javascript
// 基准屏幕宽度（设计稿宽度）
const baseWidth = 750;
// 基准屏幕下的 font-size（我们设的 100px）
const baseFontSize = 100;

// 当前手机的屏幕宽度
const currentWidth = document.documentElement.clientWidth;

// 计算当前屏幕下的 html font-size
// 逻辑：当前屏幕宽度 / 基准屏幕宽度 = 当前 font-size / 基准 font-size
const currentFontSize = (currentWidth / baseWidth) * baseFontSize;

// 设置给 html 标签
document.documentElement.style.fontSize = currentFontSize + 'px';
```

#### 代入不同屏幕看效果

我们把不同的屏幕宽度代入公式，看看按钮的实际大小：

| 手机屏幕宽度     | 计算出的 html font-size        | 按钮 CSS 写的是 3rem | 按钮实际宽度                    |
| :--------- | :------------------------- | :-------------- | :------------------------ |
| 750px（基准）  | (750/750)*100 = **100px**  | 3rem            | 3*100 = **300px**（和设计稿一致） |
| 375px（小屏）  | (375/750)*100 = **50px**   | 3rem            | 3*50 = **150px**（等比例缩小一半） |
| 1080px（大屏） | (1080/750)*100 = **144px** | 3rem            | 3*144 = **432px**（等比例放大）  |

完美！这样按钮在不同屏幕上的比例，和设计稿完全一致，实现了适配。

### 第四步：加上 viewport（必须做，否则适配无效）

在 H5 的 `<head>` 标签里，必须加上这一行代码，否则 rem 适配会失效：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
```

#### 这行代码的作用

- `width=device-width`：让页面的宽度等于手机的屏幕宽度；
- `initial-scale=1.0`：初始缩放比例为 1；
- `maximum-scale=1.0, user-scalable=no`：禁止用户手动缩放页面。

## 三、一个完整的 H5 + rem 适配代码示例

你可以把这段代码复制到本地，用浏览器的 “移动端模式” 打开，切换不同的手机尺寸，看看效果：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <!-- 必须加 viewport！ -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
  <title>rem H5 适配示例</title>
  <style>
    /* 页面元素全部用 rem */
    body {
      margin: 0;
      padding: 0;
      font-size: 0.28rem; /* 对应设计稿 28px */
    }

    .container {
      padding: 0.3rem; /* 对应设计稿 30px */
    }

    .title {
      font-size: 0.36rem; /* 对应设计稿 36px */
      font-weight: bold;
      margin-bottom: 0.3rem;
    }

    .btn {
      width: 3rem; /* 对应设计稿 300px */
      height: 0.8rem; /* 对应设计稿 80px */
      background: #ffcc00;
      border: none;
      border-radius: 0.1rem;
      font-size: 0.32rem;
      color: #333;
    }
  </style>
</head>
<body>
  <div class="container">
    <div class="title">rem H5 适配示例</div>
    <p>切换不同手机尺寸，按钮和文字会等比例缩放。</p>
    <button class="btn">我是按钮</button>
  </div>

  <script>
    // 核心适配逻辑：动态设置 html 的 font-size
    function setRem() {
      const baseWidth = 750; // 设计稿基准宽度
      const baseFontSize = 100; // 基准 font-size
      const currentWidth = document.documentElement.clientWidth;
      const currentFontSize = (currentWidth / baseWidth) * baseFontSize;
      document.documentElement.style.fontSize = currentFontSize + 'px';
    }

    // 页面加载时执行
    setRem();
    // 屏幕尺寸变化时（比如横屏）重新执行
    window.addEventListener('resize', setRem);
  </script>
</body>
</html>
```

## 四、rem 适配的常见问题与优化

### 1. 为什么基准 font-size 设为 100px？

纯粹是为了**好算**！设计稿上的 px 直接除以 100 就是 rem，不用计算器。

- 设计稿 150px → 1.5rem
- 设计稿 28px → 0.28rem

如果你设为 50px，设计稿 150px 就是 3rem，也可以，但 100px 是最常用的。

### 2. 除了 JS 动态设置，还有其他方法吗？

有，还可以用 **CSS 媒体查询（@media）**，但不如 JS 灵活：

```css
/* 媒体查询方式：针对不同屏幕宽度设置不同的 html font-size */
html { font-size: 50px; } /* 375px 屏幕 */
@media (min-width: 750px) {
  html { font-size: 100px; } /* 750px 屏幕 */
}
@media (min-width: 1080px) {
  html { font-size: 144px; } /* 1080px 屏幕 */
}
```

媒体查询的缺点是不能覆盖所有屏幕宽度，JS 动态设置是更通用的方案。

### 3. 现在还有更简单的适配方案吗？

有！现在越来越多的项目开始用 **vw/vh 单位**（视口单位）：

- `1vw` = 屏幕宽度的 1%
- `1vh` = 屏幕高度的 1%

对于 750px 的设计稿，`1vw = 7.5px`，设计稿 300px 就是 `300 / 7.5 = 40vw`，不用 JS 动态设置，直接写 CSS 就行。

不过 `rem` 的兼容性更好（尤其是旧设备），而且很多老项目还在用，所以 `rem` 仍然是 H5 适配的核心技能之一。

---

## 五、总结

1. **H5**：移动端网页，核心需求是「不同屏幕尺寸的适配」；
2. **rem**：相对于 `<html>` 标签 `font-size` 的相对长度单位，稳定、不乱；
3. **rem 适配 H5 的核心**：根据屏幕宽度动态修改 `<html>` 的 `font-size`，页面元素用 `rem` 写，实现等比例缩放；
4. **必须做的**：加上 `<meta name="viewport">` 标签。
