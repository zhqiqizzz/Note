在 JavaScript 开发中，排序算法无处不在，从表格数据的展示到复杂的后端数据处理，都离不开它。虽然在实际工作中我们最常使用的是 JavaScript 内置的 `Array.prototype.sort()` 方法，但理解底层的排序算法原理对于提升编程思维和应对技术面试至关重要。

这里为你整理了 JavaScript 中最常见的几种排序算法，涵盖了从基础入门到高效实现的经典方案，并附带了代码实现。

### 1. 冒泡排序 (Bubble Sort)

这是最基础、最容易理解的排序算法。

- **原理**：它像水中的气泡一样，重复地遍历数组，依次比较相邻的两个元素。如果前一个比后一个大，就交换它们。每一轮遍历都会将当前未排序部分中最大的元素“浮”到数组末尾。
- **特点**：代码简单，但效率较低，适合小规模数据或教学演示。
- **时间复杂度**：平均和最坏情况均为 O(n²)。

```javascript
function bubbleSort(arr) {
    const len = arr.length;
    for (let i = 0; i < len - 1; i++) {
        // 每一轮将最大的元素移到最后
        for (let j = 0; j < len - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                // 交换位置
                [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
            }
        }
    }
    return arr;
}
```

### 2. 选择排序 (Selection Sort)

一种简单直观的排序算法，逻辑类似于我们在手中整理扑克牌。

- **原理**：将数组分为“已排序”和“未排序”两部分。每次从未排序的部分中选出最小（或最大）的元素，将其放到已排序部分的末尾。
- **特点**：交换次数比冒泡排序少，但比较次数依然很多。
- **时间复杂度**：无论数据如何，均为 O(n²)。

```javascript
function selectionSort(arr) {
    const len = arr.length;
    for (let i = 0; i < len - 1; i++) {
        let minIndex = i; // 假设当前位置是最小的
        // 寻找未排序部分的最小值
        for (let j = i + 1; j < len; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }
        // 将找到的最小值交换到当前位置
        if (i !== minIndex) {
            [arr[i], arr[minIndex]] = [arr[minIndex], arr[i]];
        }
    }
    return arr;
}
```

### 3. 插入排序 (Insertion Sort)

一种简单且高效的算法，特别适合**数据量较小**或**基本有序**的数组。

- **原理**：构建有序序列，对于未排序数据，在已排序序列中从后向前扫描，找到相应位置并插入。就像整理扑克牌时，摸一张牌插入到手中已排好序的牌堆中。
- **特点**：在数据量小或近乎有序时，效率非常高，甚至优于快速排序。
- **时间复杂度**：平均 O(n²)，最好情况（已排序）为 O(n)。

```javascript
function insertionSort(arr) {
    const len = arr.length;
    for (let i = 1; i < len; i++) {
        let current = arr[i]; // 待插入的元素
        let j = i - 1;
        // 在已排序序列中寻找插入位置
        while (j >= 0 && arr[j] > current) {
            arr[j + 1] = arr[j]; // 元素后移
            j--;
        }
        arr[j + 1] = current; // 插入元素
    }
    return arr;
}
```

### 4. 快速排序 (Quick Sort)

这是实际应用中**最常用**的高效排序算法之一，也是很多语言内置排序方法的基础（或变种）。

- **原理**：采用**分治策略**。
    1. 从数组中挑选一个基准值（Pivot）。
    2. 将数组分为两部分：比基准值小的放左边，比基准值大的放右边。
    3. 递归地对左右两部分重复上述过程。
- **特点**：速度极快，适合大规模数据。
- **时间复杂度**：平均 O(n log n)，最坏情况 O(n²)。

```javascript
function quickSort(arr) {
    if (arr.length <= 1) return arr;
    
    const pivotIndex = Math.floor(arr.length / 2);
    const pivot = arr.splice(pivotIndex, 1)[0]; // 选取基准值
    const left = [];
    const right = [];
    
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] < pivot) {
            left.push(arr[i]);
        } else {
            right.push(arr[i]);
        }
    }
    
    // 递归排序并合并
    return [...quickSort(left), pivot, ...quickSort(right)];
}
```

### 5. 归并排序 (Merge Sort)

一种**稳定**的排序算法，性能表现非常稳健。

- **原理**：也是采用**分治法**。
    1. 将数组递归地拆分成两半，直到每个子数组只有一个元素。
    2. 将两个有序的子数组合并成一个大的有序数组（合并过程）。
- **特点**：无论输入数据如何，速度都很稳定，且是稳定排序（相同元素顺序不会变），但需要额外的内存空间。
- **时间复杂度**：始终为 O(n log n)。

```javascript
function mergeSort(arr) {
    if (arr.length <= 1) return arr;
    
    const mid = Math.floor(arr.length / 2);
    const left = arr.slice(0, mid);
    const right = arr.slice(mid);
    
    return merge(mergeSort(left), mergeSort(right));
}

function merge(left, right) {
    const result = [];
    while (left.length && right.length) {
        if (left[0] <= right[0]) {
            result.push(left.shift());
        } else {
            result.push(right.shift());
        }
    }
    // 拼接剩余元素
    return result.concat(left, right);
}
```

### 📊 算法性能对比总结

|算法|平均时间复杂度|最好情况|最坏情况|空间复杂度|稳定性|适用场景|
|---|---|---|---|---|---|---|
|**冒泡排序**|O(n²)|O(n)|O(n²)|O(1)|稳定|学习、极小数据量|
|**选择排序**|O(n²)|O(n²)|O(n²)|O(1)|不稳定|小数据量、对内存敏感|
|**插入排序**|O(n²)|O(n)|O(n²)|O(1)|稳定|数据量小、基本有序|
|**快速排序**|O(n log n)|O(n log n)|O(n²)|O(log n)|不稳定|**通用、大数据量**|
|**归并排序**|O(n log n)|O(n log n)|O(n log n)|O(n)|稳定|需要稳定排序、大数据量|

### 💡 实际开发建议

1. **首选内置方法**：在实际项目中，优先使用 JavaScript 原生的 `arr.sort((a, b) => a - b)`。现代 JS 引擎（如 V8）对 `sort` 进行了深度优化（通常使用 Timsort 或快速排序的变体），性能远优于手写算法。
2. **手写用于学习**：手写这些算法主要用于理解算法逻辑、优化思维以及在面试中展示基础能力。
3. **场景选择**：如果面试官要求手写，**快速排序**通常是首选，因为它兼顾了效率和代码量；如果要求**稳定排序**，则选择**归并排序**。
