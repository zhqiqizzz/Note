JavaScript 的数据类型分为**两大类共 8 种**：7 种**原始类型（基本数据类型）** + 1 种**引用类型（复杂数据类型）**，所有 JS 值都归属于这 8 种类型之内。

## 一、原始类型（基本数据类型）

原始类型是**不可变的、按值访问**的基础类型，值直接存储在**栈内存**中，是 JS 中最基础的数据单元。

ES6 之后标准的原始类型共 7 种，按出现顺序如下：

| 类型          | 核心定义                       | 取值范围 / 特点                                                                                | 示例                                                       | typeof 检测结果              |
| :---------- | :------------------------- | :--------------------------------------------------------------------------------------- | :------------------------------------------------------- | :----------------------- |
| `Undefined` | 未定义类型，仅 1 个唯一值 `undefined` | 变量声明但未赋值、函数无返回值、对象不存在的属性，默认值均为 `undefined`                                               | `let a; console.log(a); // undefined`                    | `undefined`              |
| `Null`      | 空值类型，仅 1 个唯一值 `null`       | 表示「空对象指针」，用于手动标记变量为空、无引用对象                                                               | `let obj = null; // 手动清空对象引用`                            | `object`（历史遗留 bug，非标准错误） |
| `Boolean`   | 布尔类型，仅 2 个合法值              | `true`（真）、`false`（假），用于逻辑判断                                                              | `const isLogin = true;`                                  | `boolean`                |
| `Number`    | 数字类型，用于表示整数和浮点数            | 包含整数、浮点数，以及特殊值：`NaN`（非数字）、`Infinity`（正无穷）、`-Infinity`（负无穷）；安全整数范围 `-(2^53-1)` ~ `2^53-1` | `let num = 100; let float = 0.1; let nan = NaN;`         | `number`                 |
| `String`    | 字符串类型，用于表示文本数据             | 由单引号`''`、双引号`""`、反引号（模板字符串）包裹，**不可变**（修改字符串只会生成新字符串，原字符串不变）                              | `const str = 'hello'; const template =` ${str} world`;`  | `string`                 |
| `Symbol`    | ES6 新增，唯一符号类型              | 表示**全局唯一、不可修改**的原始值，核心用途是作为对象的唯一属性名，避免属性名冲突                                              | `const key = Symbol('key'); const obj = { [key]: 123 };` | `symbol`                 |
| `BigInt`    | ES2020 新增，大整数类型            | 用于表示超出`Number`安全范围的超大整数，解决大数精度丢失问题，字面量以`n`结尾                                             | `const bigNum = 9007199254740993n;`                      | `bigint`                 |

### 原始类型核心共性

1. **不可变性**：原始值一旦创建就无法修改，所谓的 “修改” 本质是重新赋值，开辟新的内存空间；
2. **按值访问 / 比较**：两个原始值对比时，只对比值是否相等，值相同则相等；
3. **栈内存存储**：值直接存在栈中，读写速度快，内存占用固定。

## 二、引用类型（复杂数据类型）

引用类型只有**1 个顶层类型：`Object`**，我们日常使用的数组、函数、日期等，本质都是`Object`的子类，都属于引用类型。

引用类型是**可变的、按引用地址访问**的复杂类型，**指针存在栈内存，实际数据存在堆内存**。

### 常用的引用类型子类

1. **普通对象 `Object`**：最基础的引用类型，由键值对组成的无序集合
    
```javascript
    const user = { name: '张三', age: 20 };
```
    
2. **数组 `Array`**：有序的、可存储任意类型数据的集合
    
```javascript
    const list = [1, '2', true, {}];
```
    
3. **函数 `Function`**：可执行的对象，JS 中函数是一等公民，既可以执行，也可以作为值传递
    
```javascript
    function add(a, b) { return a + b; }
```
    
4. **特殊内置对象**：
    
    - 日期 `Date`、正则 `RegExp`
    - 键值对集合 `Map`、`WeakMap`
    - 唯一值集合 `Set`、`WeakSet`
    - 异步对象 `Promise`、`Error` 错误对象等
    
5. **包装对象**：`Number`、`String`、`Boolean`（和原始类型的同名包装类，用于给原始值提供方法支持，比如`'abc'.toUpperCase()`，JS 会临时创建包装对象，执行完立即销毁）

### 引用类型核心共性

1. **可变性**：可以随时修改对象的属性、方法，内存地址不变；
2. **按引用地址访问 / 比较**：两个对象对比时，只对比内存地址是否相同，哪怕内容完全一致，地址不同也不相等；
    
```javascript
    const obj1 = { a: 1 };
    const obj2 = { a: 1 };
    console.log(obj1 === obj2); // false，地址不同
```
    
3. **堆内存存储**：实际数据存在堆中，栈中只存储指向堆的指针，可存储大小不固定的复杂数据。

## 三、原始类型 vs 引用类型 核心区别

|对比维度|原始类型|引用类型|
|:--|:--|:--|
|存储位置|栈内存|指针在栈，实际数据在堆|
|可变性|不可变，修改即重新赋值|可变，可修改属性 / 内容，地址不变|
|比较方式|按值比较，值相等即相等|按引用地址比较，地址相同才相等|
|赋值行为|拷贝完整的值，两个变量完全独立，互不影响|拷贝内存地址，两个变量指向同一个对象，修改一个影响另一个|
|类型检测|`typeof` 可检测大部分（除`null`）|`typeof` 仅能识别函数，其余需用`instanceof`/`toString`检测|

## 四、补充：数据类型的精准检测方法

1. **`typeof`**：适合检测原始类型（除`null`）和函数
    
```javascript
    typeof 123; // 'number'
    typeof 'abc'; // 'string'
    typeof undefined; // 'undefined'
    typeof function(){}; // 'function'
    typeof null; // 'object'（历史bug）
    typeof {}; // 'object'（无法区分数组、日期等）
```
    
2. **`instanceof`**：适合检测引用类型，判断对象的原型链上是否存在构造函数的`prototype`
    
```javascript
    [] instanceof Array; // true
    {} instanceof Object; // true
    new Date() instanceof Date; // true
```
    
3. **`Object.prototype.toString.call()`**：**万能精准检测方法**，可检测所有类型
    
```javascript
    Object.prototype.toString.call(123); // '[object Number]'
    Object.prototype.toString.call(null); // '[object Null]'
    Object.prototype.toString.call([]); // '[object Array]'
    Object.prototype.toString.call(function(){}); // '[object Function]'
    Object.prototype.toString.call(Symbol()); // '[object Symbol]'
```

## 五、高频易错点

1. **`null` 和 `undefined` 的区别**：
    
    - `undefined` 是「未定义」，系统默认的空值；
    - `null` 是「手动置空」，开发者主动标记的空对象；
    - `null == undefined` 为`true`，`null === undefined` 为`false`。
    
2. **`NaN` 的特殊性**：
    
    - `NaN` 是唯一**不等于自身**的值，`NaN === NaN` 为`false`；
    - 检测`NaN`必须用`Number.isNaN()`，不能用全等判断。
    
3. **原始值调用方法的原理**：
    
    比如`'abc'.toUpperCase()`，原始字符串本身没有方法，JS 会临时创建对应的`String`包装对象，调用方法后立即销毁包装对象。
