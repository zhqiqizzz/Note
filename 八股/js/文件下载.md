文件下载在前端里本质是：

```
前端向后端请求文件
后端返回文件流或文件地址
浏览器拿到文件数据
前端触发浏览器保存文件
```

常见有两种方式：**直接链接下载** 和 **Blob 下载**。

**一、最简单的下载：a 标签**

如果文件是公开地址，可以直接写：

```
<a href="https://example.com/files/report.xlsx" download>
  下载报表
</a>
```

或者 JS 触发：

```
const a = document.createElement('a')
a.href = 'https://example.com/files/report.xlsx'
a.download = '报表.xlsx'
a.click()
```

适合：

```
公开文件
不需要复杂鉴权
文件地址固定
```

缺点：

```
不好加 Authorization 请求头
不好统一处理接口错误
跨域时 download 文件名可能不生效
```


**二、常见业务下载：Blob 下载**

后台系统更常见的是：点击“导出”，前端请求接口，后端返回文件流。

完整流程：

```
1. 用户点击导出
2. 前端带查询条件请求导出接口
3. 后端生成 Excel/PDF/ZIP 等文件
4. 后端返回二进制文件流
5. 前端把响应转成 Blob
6. 前端用 URL.createObjectURL 生成临时地址
7. 创建 a 标签并触发 click
8. 浏览器下载文件
9. 前端释放临时地址
```

示例：

```js
async function exportUsers() {
  const res = await fetch('/api/users/export', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      keyword: 'Tom',
      status: 'enabled'
    })
  })

  const blob = await res.blob()

  download(blob, '用户列表.xlsx')
}

function download(blob, filename) {
  const url = URL.createObjectURL(blob)
  const a = document.createElement('a')

  a.href = url
  a.download = filename
  a.click()

  URL.revokeObjectURL(url)
}
```

如果同一个文件又下载了一次，**重命名通常是浏览器/操作系统下载管理器处理的**，不是前端代码主动处理。

比如你设置：

```js
a.download = '用户列表.xlsx'
```

第一次下载：

```
用户列表.xlsx
```

第二次下载时，如果下载目录已有同名文件，浏览器一般会自动保存成：

```
用户列表 (1).xlsx
用户列表 (2).xlsx
```

不同浏览器和系统的命名细节可能略有差异，但总体是浏览器下载流程处理。前端只是“建议文件名”，最终怎么落盘由浏览器决定。

如果你想避免依赖浏览器自动重名，可以前端主动加时间戳：

```js
download(blob, `用户列表_${Date.now()}.xlsx`)
```

或者更可读一点：

```js
function formatTime() {
  const d = new Date()

  const y = d.getFullYear()
  const m = String(d.getMonth() + 1).padStart(2, '0')
  const day = String(d.getDate()).padStart(2, '0')
  const h = String(d.getHours()).padStart(2, '0')
  const min = String(d.getMinutes()).padStart(2, '0')
  const s = String(d.getSeconds()).padStart(2, '0')

  return `${y}${m}${day}_${h}${min}${s}`
}

download(blob, `用户列表_${formatTime()}.xlsx`)
```

这样下载出来就是：

```
用户列表_20260615_143012.xlsx
```

**三、Blob 是什么**

`Blob` 可以理解为浏览器里的“二进制文件对象”。

它可以表示：

```
Excel
PDF
图片
压缩包
文本文件
任意二进制数据
```

比如：

```js
const blob = new Blob(['Hello'], {
  type: 'text/plain'
})
```

这就是一个文本文件数据。

下载它：

```js
download(blob, 'hello.txt')
```

浏览器会保存一个 `hello.txt` 文件。


**四、URL.createObjectURL 是什么**

浏览器不能直接把 Blob 当成链接下载，所以需要生成一个临时 URL：

```js
const url = URL.createObjectURL(blob)
```

生成的地址类似：

```
blob:http://localhost:5173/3c2f...
```

这个地址只在当前页面有效，指向内存里的 Blob 数据。

然后：

```js
a.href = url
```

浏览器就能像访问普通链接一样访问这个文件数据。

最后要释放：

```
URL.revokeObjectURL(url)
```

否则大文件下载多了可能占内存。

**五、后端需要返回什么**

后端通常返回：

```http
Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
Content-Disposition: attachment; filename="users.xlsx"
```

`Content-Type` 表示文件类型。

常见类型：

```
Excel xlsx:
application/vnd.openxmlformats-officedocument.spreadsheetml.sheet

PDF:
application/pdf

ZIP:
application/zip

CSV:
text/csv

普通二进制:
application/octet-stream
```

`Content-Disposition` 告诉浏览器这是附件下载：

```http
Content-Disposition: attachment; filename="users.xlsx"
```

如果文件名有中文，更推荐：

```http
Content-Disposition: attachment; filename*=UTF-8''%E7%94%A8%E6%88%B7%E5%88%97%E8%A1%A8.xlsx
```

前端可以从这个响应头里解析文件名。

---

**六、从响应头获取文件名**

后端如果返回了：

```http
Content-Disposition: attachment; filename="users.xlsx"
```

前端可以解析：

```js
function getFilenameFromDisposition(disposition) {
  if (!disposition) {
    return ''
  }

  const match = disposition.match(/filename="?([^"]+)"?/)
  return match ? decodeURIComponent(match[1]) : ''
}
```

fetch 示例：

```js
const res = await fetch('/api/export')
const blob = await res.blob()

const disposition = res.headers.get('Content-Disposition')
const filename = getFilenameFromDisposition(disposition) || 'download.xlsx'

download(blob, filename)
```

注意：如果是跨域请求，后端还需要暴露响应头：

```http
Access-Control-Expose-Headers: Content-Disposition
```

否则前端 JS 读不到 `Content-Disposition`。

**七、Axios 下载写法**

axios 要设置：

```
responseType: 'blob'
```

示例：

```js
async function exportUsers() {
  const res = await axios.post(
    '/api/users/export',
    {
      keyword: 'Tom'
    },
    {
      responseType: 'blob'
    }
  )

  const blob = res.data
  const filename = '用户列表.xlsx'

  download(blob, filename)
}
```

如果不设置 `responseType: 'blob'`，axios 可能会按 JSON 或文本解析，文件就坏了。

**八、错误处理很重要**

文件下载有一个坑：后端成功时返回文件流，失败时可能返回 JSON。

例如失败响应：

```json
{
  "code": 500,
  "message": "导出失败"
}
```

但你前端设置了 `responseType: 'blob'`，错误 JSON 也会变成 Blob。

所以要判断 Blob 类型。

```js
async function handleDownloadResponse(res, filename) {
  const blob = res.data

  if (blob.type.includes('application/json')) {
    const text = await blob.text()
    const error = JSON.parse(text)

    throw new Error(error.message || '下载失败')
  }

  download(blob, filename)
}
```

fetch 也可以判断：

```js
const contentType = res.headers.get('Content-Type') || ''

if (contentType.includes('application/json')) {
  const error = await res.json()
  throw new Error(error.message || '下载失败')
}

const blob = await res.blob()
download(blob, filename)
```

**九、完整 fetch 版**

```js
async function exportFile() {
  const res = await fetch('/api/users/export', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      keyword: 'Tom'
    })
  })

  if (!res.ok) {
    throw new Error('下载失败')
  }

  const contentType = res.headers.get('Content-Type') || ''

  if (contentType.includes('application/json')) {
    const error = await res.json()
    throw new Error(error.message || '下载失败')
  }

  const blob = await res.blob()

  const disposition = res.headers.get('Content-Disposition')
  const filename = getFilenameFromDisposition(disposition) || '用户列表.xlsx'

  download(blob, filename)
}
```

下载函数：

```js
function download(blob, filename) {
  const url = URL.createObjectURL(blob)
  const a = document.createElement('a')

  a.href = url
  a.download = filename
  document.body.appendChild(a)
  a.click()
  a.remove()

  URL.revokeObjectURL(url)
}
```

加 `appendChild` 兼容性更稳一点。

**十一、下载进度**

fetch 原生处理下载进度比较麻烦。

axios 可以用：

```js
axios.get('/api/file', {
  responseType: 'blob',
  onDownloadProgress(event) {
    const percent = Math.round((event.loaded * 100) / event.total)
    console.log(percent)
  }
})
```

注意：有些响应没有 `Content-Length`，那 `event.total` 可能是 `0` 或不可用。

**十二、为什么不能直接打开接口**

有些接口可以直接：

```js
window.location.href = '/api/export'
```

这样浏览器会下载。

但它有局限：

```
不好设置 Authorization 请求头
不好用 POST body
不好处理错误信息
不好显示 loading 和进度
不好统一鉴权
```

所以后台管理系统更常用 Blob 下载。

**十三、下载流程总结**

你可以记成：

```
请求接口
拿到二进制响应
转成 Blob
生成临时 URL
创建 a 标签
设置 href 和 download
模拟点击
释放 URL
```

核心代码就是：

```
function download(blob, filename) {
  const url = URL.createObjectURL(blob)
  const a = document.createElement('a')

  a.href = url
  a.download = filename
  a.click()

  URL.revokeObjectURL(url)
}
```

一句话总结：

**Blob 下载的原理，就是把后端返回的文件流变成浏览器内存里的临时文件地址，再用 a 标签触发浏览器下载。**


**十四、大文件更推荐后端给下载 URL**

大文件更好的流程：

```
前端请求创建导出任务
后端异步生成文件
生成完成后返回文件下载 URL
前端用普通链接下载
浏览器直接接管下载
```

例如：

```js
window.location.href = downloadUrl
```

或者：

```vue
<a href="downloadUrl">下载</a>
```

这种方式浏览器可以边下边写入下载管理器，不需要前端 JS 把整个文件读成 Blob，内存压力小很多。

**后端返回的文件下载 URL 是服务器上的真实 URL**

比如后端异步生成文件后，返回：

```json
{
  "downloadUrl": "https://api.example.com/files/export/user-list-abc123.xlsx"
}
```

这个 URL 指向服务器、对象存储或 CDN 上的真实文件。

特点：

```
后端生成
服务器/CDN/对象存储可访问
可以设置有效期
可以支持大文件下载
可以支持断点续传
浏览器下载管理器直接接管
```

前端可以这样下载：

```js
window.location.href = downloadUrl
```

或者：

```js
const a = document.createElement('a')
a.href = downloadUrl
a.click()
```

---

**后端异步生成文件一般怎么做**

典型流程是这样：

```
1. 前端点击导出
2. 前端请求：创建导出任务
3. 后端返回 taskId
4. 后端在后台慢慢生成 Excel/ZIP/PDF
5. 前端轮询 taskId 状态
6. 后端生成完成后，把文件存到服务器/对象存储
7. 后端返回 downloadUrl
8. 前端拿 downloadUrl 触发下载
```

接口可能长这样：

```
POST /api/export-tasks
```

返回：

```json
{
  "taskId": "task_123"
}
```

前端轮询：

```
GET /api/export-tasks/task_123
```

如果还没完成：

```json
{
  "status": "processing",
  "progress": 60
}
```

如果完成：

```
{
  "status": "finished",
  "downloadUrl": "https://api.example.com/downloads/user-list-task_123.xlsx"
}
```

前端下载：

```js
window.location.href = result.downloadUrl
```

---

**后端文件一般存在哪里**

常见几种：

```
服务器本地磁盘
对象存储：OSS、S3、COS、MinIO
CDN
临时文件服务
```

更推荐对象存储/CDN，尤其是大文件。

后端生成完文件后，可能会得到一个地址：

```
https://cdn.example.com/exports/task_123.xlsx
```

或者生成一个有时效的签名 URL：

```
https://oss.example.com/exports/task_123.xlsx?signature=xxx&expires=xxx
```

这种 URL 可能 5 分钟、30 分钟、1 小时后失效，防止文件长期暴露。

---

**两种 URL 对比**

```
前端 blob URL：
blob:http://localhost:5173/xxx
本地临时地址，指向浏览器内存里的 Blob

后端 downloadUrl：
https://api.example.com/files/xxx.xlsx
真实网络地址，指向服务器/存储上的文件
```

一句话：

**`URL.createObjectURL(blob)` 是前端把已经拿到的文件数据变成临时下载地址；后端返回的 `downloadUrl` 是服务器已经生成并保存好的真实文件地址。**
