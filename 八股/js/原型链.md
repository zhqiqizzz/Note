原型链是 JavaScript 实现**面向对象编程**和**继承**的核心机制，它的本质是一条由「原型对象」串联起来的引用链，解决了 “对象如何共享属性和方法” 的问题，是理解 JavaScript 继承、代码复用的关键。

## 一、前置基础：构造函数、原型对象、实例的三角关系

要理解原型链，必须先明确三个核心概念及其关系：

### 1. 构造函数（Constructor）

构造函数是用于创建对象的函数，通常首字母大写，通过 `new` 关键字调用。

```javascript
// 构造函数：用于创建 Person 实例
function Person(name) {
  this.name = name; // 实例自身的属性
}
```

### 2. 原型对象（Prototype）

每个**构造函数**都有一个 `prototype` 属性，它指向一个**普通对象**，这个对象就是「原型对象」。

- 原型对象的作用是：存放所有实例**共享的属性和方法**，避免每个实例都重复创建相同的方法，节省内存。

```javascript
// 在 Person 的原型对象上定义共享方法
Person.prototype.sayName = function() {
  console.log('My name is ' + this.name);
};
```

### 3. 实例（Instance）

通过 `new 构造函数()` 创建的对象就是「实例」，每个实例都有一个**隐式原型属性** `__proto__`（注意：前后各两个下划线），它指向**构造函数的原型对象**。

```javascript
// 创建两个 Person 实例
const person1 = new Person('Alice');
const person2 = new Person('Bob');

// 实例的 __proto__ 指向构造函数的 prototype
console.log(person1.__proto__ === Person.prototype); // true
console.log(person2.__proto__ === Person.prototype); // true
```

### 4. 三角关系闭环

原型对象还有一个 `constructor` 属性，指回**构造函数本身**，形成完整的三角关系：

```javascript
console.log(Person.prototype.constructor === Person); // true
console.log(person1.constructor === Person); // true（实例的 constructor 继承自原型）
```

用图总结三角关系：

```plaintext
构造函数 Person
    | prototype
    ↓
原型对象 Person.prototype ← constructor ←
    | __proto__                          |
    ↓                                    |
实例 person1 / person2 ------------------
```

## 二、原型链的核心：属性 / 方法的查找规则

当访问一个实例的**属性或方法**时，JavaScript 引擎会按照以下规则查找，这就是「原型链」的工作原理：
### 1. 查找顺序

1. **先在实例自身找**：如果实例自身有这个属性 / 方法，直接使用，停止查找；
2. **再去实例的原型对象找**：如果实例自身没有，就通过 `__proto__` 去构造函数的原型对象找；
3. **继续往上找原型的原型**：原型对象也是普通对象，它也有自己的 `__proto__`，如果原型对象上也没有，就去原型的原型找；
4. **直到找到顶端 Object.prototype**：所有对象的原型链最终都会指向 `Object.prototype`（JavaScript 的 “根原型”）；
5. **顶端仍没有则返回 undefined**：`Object.prototype.__proto__` 是 `null`，查找结束，返回 `undefined`。

### 2. 代码示例：直观演示查找过程

```javascript
// 1. 构造函数 + 原型方法
function Person(name) {
  this.name = name;
}
Person.prototype.sayName = function() {
  console.log('My name is ' + this.name);
};

// 2. 创建实例
const person1 = new Person('Alice');

// 3. 访问实例自身的属性：直接找到
console.log(person1.name); // 输出：Alice（实例自身有 name）

// 4. 访问原型上的方法：实例自身没有，去原型找
person1.sayName(); // 输出：My name is Alice（Person.prototype 上有 sayName）

// 5. 访问 Object.prototype 上的方法：原型链顶端
console.log(person1.toString()); // 输出：[object Object]（Object.prototype 上有 toString）

// 6. 访问不存在的属性：原型链顶端仍没有
console.log(person1.age); // 输出：undefined
```

### 3. 原型链的 “顶端”：Object.prototype

所有 JavaScript 对象（除了 `Object.create(null)`）的原型链最终都会指向 `Object.prototype`，它是原型链的 “根”：

- `Object.prototype` 上有我们常用的方法：`toString()`、`valueOf()`、`hasOwnProperty()` 等；
- `Object.prototype.__proto__` 是 `null`，表示原型链到此结束，没有更上层的原型了。

## 三、原型链的关键特性

### 1. 属性遮蔽（Property Shadowing）

如果实例自身的属性名和原型对象的属性名**相同**，实例自身的属性会 “遮蔽” 原型上的属性，优先使用实例自身的：

```javascript
function Person() {}
Person.prototype.age = 18; // 原型上的 age

const person1 = new Person();
console.log(person1.age); // 输出：18（用原型的）

person1.age = 20; // 给实例自身加 age
console.log(person1.age); // 输出：20（实例自身的遮蔽了原型的）
console.log(person1.__proto__.age); // 输出：18（原型上的 age 还在）
```

### 2. 方法共享：节省内存

原型对象上的方法是**所有实例共享的**，不会每个实例都创建一份，这是原型链最大的优势：

```javascript
function Person(name) {
  this.name = name;
}
Person.prototype.sayName = function() {
  console.log(this.name);
};

const person1 = new Person('Alice');
const person2 = new Person('Bob');

// 两个实例的 sayName 是同一个函数（共享自原型）
console.log(person1.sayName === person2.sayName); // true
```

如果把方法写在构造函数里，每个实例都会创建一份，会浪费内存，所以**共享方法一定要放在原型上**。

### 3. 所有对象都有原型链

不仅是自定义构造函数的实例，JavaScript 中的内置对象（如数组、函数、字符串）也有原型链：

```javascript
// 数组的原型链
const arr = [1, 2, 3];
console.log(arr.__proto__ === Array.prototype); // true
console.log(arr.__proto__.__proto__ === Object.prototype); // true
arr.push(4); // push 方法来自 Array.prototype

// 函数的原型链
function fn() {}
console.log(fn.__proto__ === Function.prototype); // true
console.log(fn.__proto__.__proto__ === Object.prototype); // true
```

## 四、**proto** vs prototype：别搞混

这两个是最容易混淆的概念，核心区别是：

|属性|所属对象|作用|
|---|---|---|
|`prototype`|**构造函数**|指向构造函数的原型对象，用于存放实例共享的属性 / 方法|
|`__proto__`|**所有对象（包括实例、原型对象）**|指向该对象的原型，用于形成原型链|

⚠️ 注意：

- `__proto__` 是历史遗留的非标准属性（虽然现代浏览器都支持），更推荐用 `Object.getPrototypeOf(实例)` 来获取实例的原型：
    
```javascript
    console.log(Object.getPrototypeOf(person1) === Person.prototype); // true
```

## 五、原型链的核心作用：实现继承

原型链是 JavaScript 实现继承的基础，比如之前讲过的「原型链继承」，就是让子类的原型指向父类的实例，从而让子类实例能通过原型链访问父类的属性和方法：

```javascript
// 父类
function Parent() {
  this.hobbies = ['reading', 'coding'];
}
Parent.prototype.sayHi = function() {
  console.log('Hi');
};

// 子类：原型链继承
function Child() {}
Child.prototype = new Parent(); // 子类原型指向父类实例
Child.prototype.constructor = Child;

const child = new Child();
child.sayHi(); // 输出：Hi（通过原型链找到 Parent.prototype 的 sayHi）
```

## 总结

- **三角关系**：构造函数的 `prototype` 指向原型对象，实例的 `__proto__` 指向原型对象，原型对象的 `constructor` 指回构造函数；
- **原型链查找规则**：实例自身 → 实例原型 → 原型的原型 → … → Object.prototype → null；
- **核心优势**：实现属性 / 方法共享，节省内存，是继承的基础；
- **关键特性**：属性遮蔽、所有对象都有原型链。
