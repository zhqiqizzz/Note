异步处理是 JavaScript 中处理**耗时操作**的核心机制，它能让代码在等待异步操作（如网络请求、文件读写、定时器）完成时，**不阻塞后续代码执行**，从而提升程序的性能和用户体验。

以下是 JavaScript 异步处理的发展历程和核心方案详解，从最基础的回调函数到最现代的 `async/await`。

## 一、前置概念：同步 vs 异步

### 1. 同步（Synchronous）

代码**按顺序执行**，后面的代码必须等待前面的代码执行完才能继续。

```javascript
console.log('1');
console.log('2'); // 必须等上面的执行完
console.log('3');
// 输出：1 → 2 → 3
```

### 2. 异步（Asynchronous）

代码**不等待**异步操作完成，直接执行后续代码；等异步操作完成后，再通过**回调函数**、`Promise` 等方式处理结果。

```javascript
console.log('1');
// 定时器是异步操作，不阻塞后续代码
setTimeout(() => {
  console.log('2'); // 1秒后执行
}, 1000);
console.log('3');
// 输出：1 → 3 → 2
```

### 3. 常见异步场景

- 定时器：`setTimeout`、`setInterval`
- 网络请求：`fetch`、`axios`、`XMLHttpRequest`
- 文件读写：Node.js 的 `fs` 模块
- 事件监听：`addEventListener`
- 数据库操作

## 二、异步处理方案 1：回调函数（Callback）

回调函数是最基础的异步处理方式，将一个函数作为参数传给异步操作，等异步操作完成后调用该函数。

### 示例：定时器回调

```javascript
function fetchData(callback) {
  setTimeout(() => {
    const data = { name: '张三' };
    callback(data); // 异步完成后调用回调函数
  }, 1000);
}

// 使用：传入回调函数处理结果
fetchData((res) => {
  console.log(res); // { name: '张三' }
});
```

### 致命问题：回调地狱（Callback Hell）

当需要**依次执行多个异步操作**时，会出现多层嵌套，代码可读性和维护性极差：

```javascript
// 回调地狱：依次请求用户信息、订单信息、商品信息
fetchUser((user) => {
  fetchOrder(user.id, (order) => {
    fetchProduct(order.productId, (product) => {
      console.log(product);
      // 嵌套越来越深，难以维护
    });
  });
});
```

## 三、异步处理方案 2：Promise（ES6）

`Promise` 是 ES6 引入的异步解决方案，用**链式调用**替代回调嵌套，彻底解决了 “回调地狱” 问题。

> 补充： [Promise](Promise.md)
### 1. Promise 的核心概念

`Promise` 是一个对象，代表一个异步操作的最终状态，有三种状态：

- **`pending`（进行中）**：初始状态，异步操作还未完成。
- **`fulfilled`（已成功）**：异步操作成功完成。
- **`rejected`（已失败）**：异步操作失败。

状态一旦从 `pending` 变为 `fulfilled` 或 `rejected`，就**不可逆**。

### 2. 基本用法

#### （1）创建 Promise

用 `new Promise()` 创建，接收一个函数，函数有两个参数：

- `resolve`：将 Promise 状态改为 `fulfilled`，传递成功结果。
- `reject`：将 Promise 状态改为 `rejected`，传递失败原因。

```javascript
function fetchData() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      const success = true;
      if (success) {
        resolve({ name: '张三' }); // 成功，调用 resolve
      } else {
        reject(new Error('请求失败')); // 失败，调用 reject
      }
    }, 1000);
  });
}
```

#### （2）使用 Promise：`then`/`catch`/`finally`

- `then`：处理成功状态（`fulfilled`）。
- `catch`：处理失败状态（`rejected`）。
- `finally`：无论成功失败都会执行（可选）。

```javascript
fetchData()
  .then((res) => {
    console.log(res); // { name: '张三' }
  })
  .catch((err) => {
    console.log(err.message); // 请求失败
  })
  .finally(() => {
    console.log('请求完成'); // 无论成功失败都执行
  });
```

### 3. 链式调用：解决回调地狱

`then` 会返回一个新的 Promise，因此可以**链式调用**，依次执行多个异步操作：

```javascript
// 依次请求用户、订单、商品，无嵌套
fetchUser()
  .then((user) => fetchOrder(user.id))
  .then((order) => fetchProduct(order.productId))
  .then((product) => console.log(product))
  .catch((err) => console.log(err)); // 统一捕获错误
```

### 4. Promise 静态方法

#### （1）`Promise.all()`：并行执行多个 Promise，全部成功才成功

等待所有 Promise 完成，返回一个结果数组；**只要有一个失败，就立即失败**。

```javascript
Promise.all([fetchUser(), fetchOrder(), fetchProduct()])
  .then(([user, order, product]) => {
    console.log(user, order, product); // 全部成功，按顺序返回结果
  })
  .catch((err) => console.log(err)); // 任意一个失败，捕获错误
```

#### （2）`Promise.race()`：竞速，最快完成的 Promise 决定结果

等待**第一个**完成的 Promise（无论成功失败），直接采用它的结果。

```javascript
Promise.race([fetchUser(), fetchOrder()])
  .then((res) => console.log(res)) // 谁先完成用谁的结果
  .catch((err) => console.log(err));
```

#### （3）`Promise.allSettled()`：等待所有 Promise 完成，无论成功失败

等待所有 Promise 完成，返回一个数组，包含每个 Promise 的状态和结果（成功或失败）。

```javascript
Promise.allSettled([fetchUser(), fetchOrder()])
  .then((results) => {
    console.log(results);
    // 输出：
    // [
    //   { status: 'fulfilled', value: { name: '张三' } },
    //   { status: 'rejected', reason: Error: '请求失败' }
    // ]
  });
```

## 四、异步处理方案 3：`async/await`（ES2017）

`async/await` 是 `Promise` 的**语法糖**，让异步代码看起来像**同步代码**，是目前最推荐的异步处理方式。

### 1. 基本用法

- **`async`**：修饰函数，让函数返回一个 Promise（即使函数内部返回的是普通值，也会被包装成 Promise）。
- **`await`**：只能在 `async` 函数内部使用，等待 Promise 完成，直接返回成功结果；如果 Promise 失败，会抛出错误，需用 `try/catch` 捕获。

```javascript
// async 函数
async function getData() {
  try {
    // await 等待 Promise 完成，直接返回结果
    const user = await fetchUser();
    const order = await fetchOrder(user.id);
    const product = await fetchProduct(order.productId);
    console.log(product);
  } catch (err) {
    // 捕获错误
    console.log(err.message);
  } finally {
    console.log('请求完成');
  }
}

// 调用 async 函数
getData();
```

### 2. 并行执行：配合 `Promise.all()`

如果多个异步操作**不需要依次执行**，可以用 `Promise.all()` 并行执行，提升效率：

```javascript
async function getDataParallel() {
  try {
    // 并行请求，同时发起，等待全部完成
    const [user, order, product] = await Promise.all([
      fetchUser(),
      fetchOrder(),
      fetchProduct()
    ]);
    console.log(user, order, product);
  } catch (err) {
    console.log(err);
  }
}
```

## 五、三种方案对比总结

| 方案            | 优点           | 缺点                   | 推荐场景           |
| ------------- | ------------ | -------------------- | -------------- |
| 回调函数          | 简单，无需额外语法    | 回调地狱，难以维护            | 简单的单个异步操作（已淘汰） |
| Promise       | 链式调用，解决回调地狱  | 仍有 `.then` 链，不够直观    | 需要灵活控制异步流程     |
| `async/await` | 代码像同步，最易读易维护 | 需配合 `try/catch` 处理错误 | **绝大多数场景推荐**   |

## 六、实战示例：用 `async/await` 实现数据请求

```javascript
// 模拟 API 请求
function fetchUser() {
  return new Promise((resolve) => {
    setTimeout(() => resolve({ id: 1, name: '张三' }), 1000);
  });
}

function fetchOrder(userId) {
  return new Promise((resolve) => {
    setTimeout(() => resolve({ userId, productId: 100 }), 1000);
  });
}

function fetchProduct(productId) {
  return new Promise((resolve) => {
    setTimeout(() => resolve({ id: productId, name: 'Vue 3 入门' }), 1000);
  });
}

// 使用 async/await 依次请求
async function getFullData() {
  console.log('开始请求...');
  try {
    const user = await fetchUser();
    console.log('用户信息：', user);
    
    const order = await fetchOrder(user.id);
    console.log('订单信息：', order);
    
    const product = await fetchProduct(order.productId);
    console.log('商品信息：', product);
    
    console.log('全部请求完成！');
  } catch (err) {
    console.error('请求出错：', err);
  }
}

// 执行
getFullData();
```

## 总结

异步处理的核心是**不阻塞代码执行**，通过回调、Promise、async/await 三种方案逐步演进，目前 **`async/await`** 是最主流、最易维护的方式，配合 `Promise.all()` 等静态方法可以灵活处理各种异步场景。
