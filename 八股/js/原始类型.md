在 JavaScript 中，**`String`/`Number`/`Boolean`** 等是**内置的 “包装对象（Wrapper Objects）”**，而 **`string`/`number`/`boolean`** 是对应的**原始数据类型（Primitive Types）**。两者密切相关但本质不同，以下是详细对比和扩展说明。

## 一、核心概念：原始类型 vs 包装对象

JavaScript 有 **7 种原始类型**：`string`、`number`、`boolean`、`null`、`undefined`、`symbol`、`bigint`。

为了让原始类型能像对象一样调用方法（如 `'hello'.toUpperCase()`），JS 提供了对应的**包装对象**：`String`、`Number`、`Boolean`、`Symbol`、`BigInt`。

## 二、`String`（包装对象）vs `string`（原始类型）

### 1. 定义方式

#### 原始类型 `string`（最常用）

直接用引号（单 / 双 / 反引号）创建，是最基础的字符串形式：

```javascript
const str1 = 'hello';       // 单引号
const str2 = "world";       // 双引号
const str3 = `hello ${str2}`; // 模板字符串
```

#### 包装对象 `String`（极少用）

用 `new String()` 创建，返回的是一个**对象**，而非原始值：

```javascript
const strObj = new String('hello');
```

### 2. 核心区别

| 特性              | 原始类型 `string` | 包装对象 `String`   |
| --------------- | ------------- | --------------- |
| **`typeof` 结果** | `"string"`    | `"object"`      |
| **本质**          | 基本值，无属性 / 方法  | 真正的对象，有属性 / 方法  |
| **比较（`===`）**   | 按值比较          | 按引用比较（两个对象永不相等） |
| **内存占用**        | 更小            | 更大              |
#### 示例对比：

```javascript
// 1. typeof 不同
const str = 'hello';
const strObj = new String('hello');
console.log(typeof str);    // "string"
console.log(typeof strObj); // "object"

// 2. 比较不同
console.log(str === 'hello');           // true（按值比较）
console.log(strObj === new String('hello')); // false（按引用比较，两个不同对象）

// 3. 但原始类型能调用 String 的方法！
// （JS 会临时“包装”成 String 对象，调用完立即销毁）
console.log(str.toUpperCase()); // "HELLO"
console.log(str.length);        // 5
```

## 三、其他类型的大小写对比

### 1. `Number`（包装对象）vs `number`（原始类型）

#### 原始类型 `number`（常用）

直接用数字创建：

```javascript
const num1 = 123;
const num2 = 3.14;
const num3 = NaN; // 特殊的 number
```

#### 包装对象 `Number`（极少用）

用 `new Number()` 创建：

```javascript
const numObj = new Number(123);
```

#### 区别示例：

```javascript
const num = 123;
const numObj = new Number(123);

console.log(typeof num);    // "number"
console.log(typeof numObj); // "object"
console.log(num === 123);   // true
console.log(numObj === 123); // false（对象 vs 原始值）

// 原始类型也能调用 Number 方法
console.log(num.toFixed(2)); // "123.00"
```

### 2. `Boolean`（包装对象）vs `boolean`（原始类型）

#### 原始类型 `boolean`（常用）

直接用 `true`/`false` 创建：

```javascript
const isTrue = true;
const isFalse = false;
```

#### 包装对象 `Boolean`（极少用）

用 `new Boolean()` 创建：

```javascript
const boolObj = new Boolean(false);
```

#### ⚠️ 关键坑点：包装对象的 “真值”

所有对象（包括 `new Boolean(false)`）在条件判断中都是 **`truthy`（真）**，这会导致逻辑错误！

```javascript
const boolObj = new Boolean(false);

// 错误示例：boolObj 是对象，会被判断为 true！
if (boolObj) {
  console.log('这行会执行！'); // ❌ 即使包装的是 false
}

// 正确：用原始类型
const isFalse = false;
if (isFalse) {
  console.log('这行不会执行'); // ✅
}
```

### 3. `Symbol`/`BigInt`（特殊：不能用 `new`）

`Symbol` 和 `BigInt` 是 ES6+ 新增的原始类型，**不能用 `new` 创建**，只能直接调用函数：

```javascript
// Symbol：创建唯一值
const sym = Symbol('desc'); // 正确
// const symObj = new Symbol(); // ❌ 报错：Symbol is not a constructor

// BigInt：大整数
const big = BigInt(123); // 正确
// const bigObj = new BigInt(123); // ❌ 报错：BigInt is not a constructor
```

## 四、最佳实践

### ✅ 推荐：始终用原始类型

- 直接用 `'string'`、`123`、`true` 等原始值，不要用 `new String()`/`new Number()`/`new Boolean()`。
- 原始类型更轻量、更安全，且能自动调用包装对象的方法（JS 会临时包装）。

### ❌ 避免：用 `new` 创建包装对象

- 仅在极少数需要 “对象形式的原始值” 时使用（几乎不会遇到）。

## 总结

|原始类型（小写）|包装对象（大写）|能否用 `new` 创建|推荐使用|
|---|---|---|---|
|`string`|`String`|✅ 但不推荐|原始类型|
|`number`|`Number`|✅ 但不推荐|原始类型|
|`boolean`|`Boolean`|✅ 但不推荐|原始类型|
|`symbol`|`Symbol`|❌ 不能|直接调用函数|
|`bigint`|`BigInt`|❌ 不能|直接调用函数|

一句话：**日常开发只用原始类型（小写），忘记包装对象（大写）的存在即可**。
