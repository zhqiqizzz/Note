作用域和闭包是 JavaScript 中最核心、最基础的概念之一：作用域规定了变量的**可访问范围**，闭包则利用作用域实现了**变量的持久化访问**。理解这两个概念，是掌握 JavaScript 变量查找、函数设计、代码复用的关键。

## 一、作用域（Scope）

### 1. 什么是作用域？

作用域是代码中**变量、函数的可访问范围**，它规定了：

- 变量在哪里可以被访问，在哪里不能被访问；
- 变量的生命周期（什么时候创建、什么时候销毁）；
- 如何避免命名冲突（不同作用域可以有同名变量）。

简单理解：作用域就像 “变量的活动范围”，每个变量都有自己的 “地盘”，超出地盘就无法访问。

### 2. 作用域的三种类型（JavaScript）

JavaScript 中有三种主要的作用域，按出现顺序依次是：**全局作用域**、**函数作用域**、**块级作用域（ES6 新增）**。

#### （1）全局作用域（Global Scope）

- **定义**：在代码最外层（不在任何函数、块内）声明的变量 / 函数，属于全局作用域；
- **可访问范围**：在代码的**任何地方**都能访问；
- **生命周期**：页面关闭 / 程序结束时才销毁；
- **注意**：浏览器环境中，全局作用域的变量会自动挂载到 `window` 对象上（Node.js 中是 `global`）。

**示例**：

```javascript
// 全局变量：在最外层声明
const globalName = 'Alice';

function sayHi() {
  console.log(globalName); // 函数内部可以访问全局变量
}
sayHi(); // 输出：Alice
console.log(window.globalName); // 输出：Alice（浏览器环境）
```

#### （2）函数作用域（Function Scope）

- **定义**：在**函数内部**声明的变量 / 函数，属于函数作用域；
- **可访问范围**：**仅在函数内部**可以访问，外部无法访问；
- **生命周期**：函数执行时创建，函数执行完后销毁；
- **注意**：`var` 声明的变量是函数作用域的（即使写在块内，也会提升到函数顶部）。

**示例**：

```javascript
function createUser() {
  // 函数作用域变量：仅在函数内部可访问
  const userName = 'Bob';
  var userAge = 18;
  console.log(userName); // 输出：Bob
}

createUser();
console.log(userName); // 报错：userName is not defined（外部无法访问）
```

#### （3）块级作用域（Block Scope，ES6 新增）

- **定义**：在 **块（`{}` 包裹的代码，如 `if`/`for`/`while` 块）**  内部用 `let`/`const` 声明的变量，属于块级作用域；
- **可访问范围**：**仅在块内部**可以访问；
- **生命周期**：块执行时创建，块执行完后销毁；
- **优势**：解决了 `var` 变量提升导致的逻辑混乱，更符合直觉。

**示例**：

```javascript
if (true) {
  // 块级作用域变量：仅在 if 块内可访问
  let blockName = 'Charlie';
  const blockAge = 20;
  console.log(blockName); // 输出：Charlie
}

console.log(blockName); // 报错：blockName is not defined（块外部无法访问）

// 对比 var：没有块级作用域
if (true) {
  var noBlockVar = 'Test';
}
console.log(noBlockVar); // 输出：Test（var 会提升到外部作用域）
```

### 3. 作用域链（Scope Chain）

当访问一个变量时，JavaScript 引擎会按照 **“从内到外”** 的顺序查找变量，这条查找路径就是「作用域链」。

#### 查找规则

1. 先在**当前作用域**找，找到就直接使用；
2. 当前作用域没找到，就去**父级作用域**找；
3. 父级也没找到，继续往上找，直到**全局作用域**；
4. 全局作用域也没找到，报错 `ReferenceError: xxx is not defined`。

**示例**：

```javascript
// 全局作用域
const globalVar = 'Global';

function outer() {
  // outer 函数作用域
  const outerVar = 'Outer';

  function inner() {
    // inner 函数作用域
    const innerVar = 'Inner';

    console.log(innerVar); // 1. 当前作用域找到：输出 Inner
    console.log(outerVar); // 2. 当前没找到，去 outer 找：输出 Outer
    console.log(globalVar); // 3. outer 没找到，去全局找：输出 Global
    console.log(notExist); // 4. 全局也没找到：报错
  }

  inner();
}

outer();
```

## 二、闭包（Closure）

闭包是 JavaScript 中最强大、最容易被误解的特性之一，它的核心是**利用作用域链，让函数在外部执行时，仍能访问其定义时的作用域变量**。

### 1. 什么是闭包？

当一个**内部函数**能够**记住并访问它定义时的词法作用域**（即外部函数的作用域），即使这个内部函数在其词法作用域**之外**执行，就产生了闭包。

简单理解：**“函数内部的函数，即使拿到外部调用，也能‘记住’外部函数的变量”**。

### 2. 闭包的形成条件

闭包的形成需要同时满足三个条件：

1. **函数嵌套函数**：内部函数定义在外部函数内部；
2. **内部函数引用外部函数的变量**：内部函数使用了外部函数的变量 / 参数；
3. **内部函数被返回或传递到外部**：外部函数将内部函数作为返回值，或传递给其他函数。

**经典示例：计数器**

```javascript
function createCounter() {
  // 外部函数的变量：仅在 createCounter 作用域内
  let count = 0;

  // 内部函数：引用了外部的 count
  function increment() {
    count++;
    console.log(count);
  }

  // 关键：将内部函数返回，传递到外部
  return increment;
}

// 调用 createCounter，得到内部函数 increment
const counter = createCounter();

// 在外部调用 increment（超出了 createCounter 的作用域）
counter(); // 输出：1（仍能访问 count）
counter(); // 输出：2（count 被持久化，没有被销毁）
counter(); // 输出：3
```

**为什么 count 没有被销毁？**

正常情况下，`createCounter` 执行完后，`count` 应该被销毁，但因为内部函数 `increment` 被返回并在外部调用，`increment` 仍在引用 `count`，所以 `count` 会被**保留在内存中**，不会被垃圾回收 —— 这就是闭包的核心作用：**让变量持久化**。

### 3. 闭包的核心应用场景

闭包的价值在于**实现私有变量、函数工厂、模块化、数据持久化**，以下是最常见的应用：

#### （1）模拟私有变量（JavaScript 没有原生私有属性）

JavaScript 中没有 `private` 关键字，但可以用闭包模拟 “只能通过特定方法访问的私有变量”：

```javascript
function createUser(initialName) {
  // 私有变量：外部无法直接访问
  let name = initialName;

  return {
    // 暴露的方法：可以访问/修改私有变量
    getName() {
      return name;
    },
    setName(newName) {
      if (newName.length > 0) {
        name = newName;
      }
    }
  };
}

const user = createUser('Alice');
console.log(user.getName()); // 输出：Alice（通过方法访问）
user.setName('Bob');
console.log(user.getName()); // 输出：Bob
console.log(user.name); // 输出：undefined（外部无法直接访问私有变量）
```

#### （2）函数工厂（批量创建类似功能的函数）

利用闭包可以 “记住外部参数” 的特性，批量创建功能相似但参数不同的函数：

```javascript
// 函数工厂：创建“乘以某个数”的函数
function createMultiplier(multiplier) {
  // 内部函数记住了外部的 multiplier
  return function(num) {
    return num * multiplier;
  };
}

// 批量创建函数
const double = createMultiplier(2); // 乘以2
const triple = createMultiplier(3); // 乘以3

console.log(double(5)); // 输出：10
console.log(triple(5)); // 输出：15
```

#### （3）模块化开发（早期 IIFE，立即执行函数表达式）

在 ES6 模块出现之前，闭包是实现模块化的核心方式，通过立即执行函数创建私有作用域，避免全局变量污染：

```javascript
// 模块化：立即执行函数表达式（IIFE）
const Calculator = (function() {
  // 私有变量：外部无法访问
  let result = 0;

  // 暴露的公共方法
  return {
    add(num) {
      result += num;
      return this;
    },
    subtract(num) {
      result -= num;
      return this;
    },
    getResult() {
      return result;
    }
  };
})();

// 使用模块
Calculator.add(10).subtract(3);
console.log(Calculator.getResult()); // 输出：7
console.log(Calculator.result); // 输出：undefined（私有变量无法访问）
```

#### （4）事件监听器中的数据保存

在事件监听器中，闭包可以保存循环中的变量值，避免因变量提升导致的逻辑错误：

```javascript
// 错误示例：var 没有块级作用域，所有点击都输出 3
for (var i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i); // 输出：3, 3, 3
  }, 1000);
}

// 正确示例1：用闭包保存每次循环的 i
for (var i = 0; i < 3; i++) {
  (function(j) {
    setTimeout(() => {
      console.log(j); // 输出：0, 1, 2
    }, 1000);
  })(i);
}

// 正确示例2：用 let 块级作用域（更简单）
for (let i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i); // 输出：0, 1, 2
  }, 1000);
}
```

### 4. 闭包的注意事项：内存泄漏

闭包的优势是 “变量持久化”，但这也是它的缺点：如果闭包使用不当，会导致**内存泄漏**（变量长期占用内存，无法被垃圾回收）。

#### 避坑指南

1. **及时释放不需要的闭包引用**：
    
    当不再需要闭包时，将引用它的变量设为 `null`，让垃圾回收器可以回收：
    
```javascript
    const counter = createCounter();
    counter();
    counter = null; // 释放引用，count 可以被回收
```
    
2. **避免在循环中创建大量闭包**：
    
    循环中创建闭包会占用大量内存，优先用 `let` 块级作用域替代。
    
3. **不要过度使用闭包**：
    
    闭包会增加代码复杂度，能用简单方式实现的就不用闭包。

## 三、总结：作用域与闭包的关系

- **作用域是基础**：规定了变量的可访问范围和生命周期，形成了作用域链；
- **闭包是作用域的延伸应用**：利用作用域链，让函数在外部执行时仍能访问定义时的变量，实现变量持久化、私有变量、模块化等功能；
- **核心记忆点**：
    - 作用域：全局 → 函数 → 块级，从内到外查找变量；
    - 闭包：函数嵌套 + 引用外部变量 + 内部函数外传，记住外部变量。
