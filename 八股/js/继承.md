继承是 JavaScript 面向对象编程的核心特性之一，核心作用是让一个对象（子类 / 派生类）**复用、扩展另一个对象（父类 / 基类）的属性和方法**，实现代码的高效复用与逻辑分层，避免重复开发。

JavaScript 的继承基于**原型链**实现，从 ES5 到 ES6 发展出了多种继承方案，下面从基础原理到最佳实践逐一介绍。

## 一、ES5 继承方案（基于原型链）

ES5 没有专门的继承语法，完全通过操作原型链实现，核心是让子类的原型与父类的原型 / 实例建立关联。

### 1. 原型链继承

#### 核心原理

让**子类的原型对象（`Sub.prototype`）直接指向父类的实例**，这样子类实例就能通过原型链访问父类实例的属性和方法。

#### 代码示例

```javascript
// 父类
function Parent(name) {
  this.name = name;
  this.hobbies = ['reading', 'coding']; // 引用类型属性
}
Parent.prototype.sayName = function() {
  console.log('My name is ' + this.name);
};

// 子类
function Child() {}
// 核心：子类原型指向父类实例
Child.prototype = new Parent('ParentName');
// 修正 constructor 指向（否则会指向 Parent）
Child.prototype.constructor = Child;

// 测试
const child1 = new Child();
const child2 = new Child();

child1.sayName(); // 输出：My name is ParentName
child1.hobbies.push('swimming');
console.log(child2.hobbies); // 输出：['reading', 'coding', 'swimming']（引用属性被共享）
```

#### 优缺点

- **优点**：实现简单，能复用父类原型上的方法；
- **缺点**：
    
    1. 父类的**引用类型属性会被所有子类实例共享**（如示例中的 `hobbies`，一个实例修改会影响其他实例）；
    2. 创建子类实例时，**无法向父类构造函数传参**（父类属性只能在原型上固定）。
    

### 2. 构造函数继承（经典继承）

#### 核心原理

在**子类构造函数内部调用父类构造函数**，通过 `call()`/`apply()` 改变父类构造函数的 `this` 指向，让父类的属性绑定到子类实例上。

#### 代码示例

```javascript
// 父类
function Parent(name) {
  this.name = name;
  this.hobbies = ['reading', 'coding'];
}
Parent.prototype.sayName = function() {
  console.log('My name is ' + this.name);
};

// 子类
function Child(name, age) {
  // 核心：调用父类构造函数，绑定 this 到子类实例
  Parent.call(this, name);
  this.age = age; // 子类自己的属性
}

// 测试
const child1 = new Child('Alice', 10);
const child2 = new Child('Bob', 12);

child1.hobbies.push('swimming');
console.log(child1.hobbies); // 输出：['reading', 'coding', 'swimming']
console.log(child2.hobbies); // 输出：['reading', 'coding']（引用属性不共享）
console.log(child1.name); // 输出：Alice（可以传参）
child1.sayName(); // 报错：sayName is not a function（无法继承父类原型方法）
```
#### 优缺点

- **优点**：
    1. 可以向父类构造函数传参；
    2. 父类的引用类型属性不会被共享；
- **缺点**：
    
    1. **无法继承父类原型上的方法**（只能继承父类构造函数内的属性）；
    2. 方法无法复用，每个子类实例都有父类构造函数的副本，内存浪费。
    

### 3. 组合继承（原型链 + 构造函数，ES5 常用）

#### 核心原理

结合前两种方案的优点：

- 用**构造函数继承**父类的属性（解决传参和引用共享问题）；
- 用**原型链继承**父类原型上的方法（解决方法复用问题）。

#### 代码示例

```javascript
// 父类
function Parent(name) {
  this.name = name;
  this.hobbies = ['reading', 'coding'];
}
Parent.prototype.sayName = function() {
  console.log('My name is ' + this.name);
};

// 子类
function Child(name, age) {
  // 1. 构造函数继承：继承父类属性（第一次调用 Parent 构造函数）
  Parent.call(this, name);
  this.age = age;
}
// 2. 原型链继承：继承父类原型方法（第二次调用 Parent 构造函数）
Child.prototype = new Parent();
Child.prototype.constructor = Child;

// 子类自己的方法
Child.prototype.sayAge = function() {
  console.log('My age is ' + this.age);
};

// 测试
const child1 = new Child('Alice', 10);
const child2 = new Child('Bob', 12);

child1.hobbies.push('swimming');
console.log(child1.hobbies); // 输出：['reading', 'coding', 'swimming']
console.log(child2.hobbies); // 输出：['reading', 'coding']（引用不共享）
child1.sayName(); // 输出：My name is Alice（继承父类方法）
child1.sayAge(); // 输出：My age is 10（子类自己的方法）
```

#### 优缺点

- **优点**：解决了前两种方案的问题，是 ES5 最常用的继承方式；
- **缺点**：**父类构造函数会被调用两次**（一次在 `call()`，一次在 `new Parent()`），导致子类原型上存在多余的父类属性（虽然不影响使用，但不够优雅）。

### 4. 寄生组合式继承（ES5 最佳实践）

#### 核心原理

优化组合继承的 “两次调用父类构造函数” 问题：

- 用**构造函数继承**父类属性；
- 用**空函数作为中介**，让子类原型直接指向父类原型（避免创建父类实例）。

#### 代码示例

```javascript
// 父类
function Parent(name) {
  this.name = name;
  this.hobbies = ['reading', 'coding'];
}
Parent.prototype.sayName = function() {
  console.log('My name is ' + this.name);
};

// 子类
function Child(name, age) {
  // 构造函数继承：继承父类属性
  Parent.call(this, name);
  this.age = age;
}

// 核心：创建空函数 F，让 F.prototype 指向 Parent.prototype
function inheritPrototype(Sub, Super) {
  const F = function() {};
  F.prototype = Super.prototype;
  Sub.prototype = new F(); // 子类原型指向 F 的实例（间接指向父类原型）
  Sub.prototype.constructor = Sub; // 修正 constructor
}

// 调用继承函数
inheritPrototype(Child, Parent);

// 子类自己的方法
Child.prototype.sayAge = function() {
  console.log('My age is ' + this.age);
};
```

#### 优缺点

- **优点**：只调用一次父类构造函数，避免了子类原型上的多余属性，是 ES5 最完美的继承方式；
- **缺点**：实现稍复杂，需要额外的中介函数。

## 二、ES6 Class 继承（语法糖，现代主流）

ES6 引入了 `class` 关键字和 `extends` 语法，提供了更清晰、更符合面向对象习惯的继承写法，**底层仍然基于原型链实现**（是寄生组合式继承的语法糖）。

### 核心语法

- `extends`：声明子类继承自父类；
- `super()`：在子类构造函数中调用父类构造函数（必须在使用 `this` 之前调用）；
- `super.方法名()`：在子类中调用父类的方法。

### 代码示例

```javascript
// 父类（用 class 声明）
class Parent {
  constructor(name) {
    this.name = name;
    this.hobbies = ['reading', 'coding'];
  }
  // 父类方法（直接写在 class 内，自动挂载到原型上）
  sayName() {
    console.log('My name is ' + this.name);
  }
}

// 子类：用 extends 继承父类
class Child extends Parent {
  constructor(name, age) {
    super(name); // 必须先调用 super()，才能使用 this
    this.age = age;
  }
  // 子类自己的方法
  sayAge() {
    console.log('My age is ' + this.age);
  }
  // 重写父类方法
  sayName() {
    super.sayName(); // 调用父类的 sayName
    console.log('(Child version)');
  }
}

// 测试
const child = new Child('Alice', 10);
child.sayName(); 
// 输出：
// My name is Alice
// (Child version)
child.sayAge(); // 输出：My age is 10
```

### 核心优势

1. **语法清晰直观**：`class`、`extends`、`super` 符合传统面向对象的写法，易读易维护；
2. **自动处理原型链**：无需手动操作 `prototype` 和 `constructor`，底层自动完成；
3. **支持静态方法继承**：父类的静态方法（用 `static` 声明）也能被子类继承；
4. **兼容性好**：现代浏览器和 Node.js 原生支持，通过 Babel 可转译为 ES5 兼容旧环境。

## 三、继承方式对比总结

表格

|继承方式|核心原理|优点|缺点|适用场景|
|---|---|---|---|---|
|原型链继承|子类原型指向父类实例|实现简单，能复用父类原型方法|引用属性共享，无法传参|简单的方法复用，无引用属性|
|构造函数继承|子类构造函数内调用父类构造函数|可传参，引用属性不共享|无法继承父类原型方法，方法无法复用|仅需继承父类属性|
|组合继承|原型链继承方法 + 构造函数继承属性|解决前两种的问题，功能完整|父类构造函数调用两次|ES5 常用场景|
|寄生组合式继承|空函数中介，子类原型直接指向父类原型|完美解决组合继承的问题，无冗余|实现稍复杂|ES5 最佳实践|
|ES6 Class 继承|`extends`+`super`，语法糖|语法清晰，易维护，支持静态方法|需转译兼容旧环境|现代前端开发主流|

## 四、实际开发建议

- **现代项目优先用 ES6 Class**：语法清晰，开发效率高，配合 Babel 可兼容所有环境；
- **理解原型链底层**：虽然用 `class`，但要知道底层是原型链，遇到问题能快速定位；
- **避免过度继承**：优先用 “组合优于继承” 的设计思想，减少继承带来的耦合
