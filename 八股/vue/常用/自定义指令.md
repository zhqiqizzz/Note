Vue 自定义指令是对内置指令（`v-model`/`v-if`/`v-show`等）的扩展，核心用于**封装可复用的底层 DOM 操作逻辑**，是 Vue 数据驱动渲染模式的补充 —— 当你需要直接操作 DOM、且逻辑需要跨组件复用时，自定义指令是最优解。

以下内容基于 **Vue 3**（主流推荐），同时标注与 Vue 2 的核心差异。

## 一、核心基础

### 1. 什么是自定义指令

- 本质：以 `v-` 为前缀的自定义属性，通过钩子函数控制绑定 DOM 元素的行为。
- 核心定位：**只处理 DOM 相关的通用逻辑**，不负责业务数据处理、接口请求等（优先用组件 / 组合式函数）。
- 典型场景：输入框自动聚焦、按钮防抖、权限元素显隐、图片懒加载、拖拽交互等。

### 2. 两种注册方式

#### （1）局部自定义指令（组件内使用）

在 `<script setup>` 中，**以 `v` 开头的驼峰式变量会被自动识别为自定义指令**，无需额外注册，模板中用短横线格式使用。

```vue
<template>
  <!-- 模板中使用 v-focus -->
  <input v-focus placeholder="自动聚焦的输入框" />
</template>

<script setup>
// 定义局部指令：v开头的驼峰命名
const vFocus = {
  // 元素挂载到DOM后触发，执行聚焦
  mounted: (el) => el.focus()
}
</script>
```

非 `<script setup>` 场景，需通过 `directives` 选项注册：

```vue
<script>
export default {
  directives: {
    // 注册指令
    focus: {
      mounted: (el) => el.focus()
    }
  }
}
</script>
```

#### （2）全局自定义指令（全应用可用）

在 `main.js` 中通过 `app.directive` 注册，所有组件都可直接使用。

```javascript
// main.js
import { createApp } from 'vue'
import App from './App.vue'

const app = createApp(App)

// 注册全局 v-focus 指令
app.directive('focus', {
  mounted: (el) => el.focus()
})

app.mount('#app')
```

### 3. 函数简写（最常用）

如果你的指令只需要在 `mounted` 和 `updated` 时执行相同逻辑，可直接简写为一个函数，无需写完整的钩子对象：

```vue
<script setup>
// 简写：函数会在 mounted 和 updated 时自动执行
const vColor = (el, binding) => {
  el.style.color = binding.value // 设置字体颜色
}
</script>

<template>
  <p v-color="'red'">红色文字</p>
  <p v-color="'blue'">蓝色文字</p>
</template>
```

## 二、指令的生命周期钩子

Vue 3 指令的钩子与组件生命周期完全对齐，更易理解和记忆，每个钩子都有明确的触发时机：

|钩子名|触发时机|常用场景|
|---|---|---|
|`created`|元素的属性 / 事件监听器被应用之前，元素还未挂载到 DOM|极少使用|
|`beforeMount`|指令第一次绑定到元素，元素挂载到 DOM**之前**触发|极少使用|
|`mounted`|指令绑定的元素**挂载到 DOM 之后**触发（⭐️最常用）|自动聚焦、初始化 DOM 属性、绑定事件|
|`beforeUpdate`|所在组件更新前，元素更新**之前**触发|保存更新前的 DOM 状态|
|`updated`|所在组件更新后，元素更新**之后**触发|响应数据变化，更新 DOM 属性|
|`beforeUnmount`|指令绑定的元素卸载**之前**触发|极少使用|
|`unmounted`|指令绑定的元素**卸载之后**触发（⭐️常用）|清理事件监听、定时器，避免内存泄漏|
### Vue 2 → Vue 3 钩子对应关系

|Vue 2 钩子|Vue 3 对应钩子|
|---|---|
|`bind`|`beforeMount`|
|`inserted`|`mounted`|
|`update`|`beforeUpdate`|
|`componentUpdated`|`updated`|
|`unbind`|`unmounted`|

## 三、钩子函数的核心参数

每个钩子都会按顺序接收 4 个参数：`(el, binding, vnode, prevVnode)`，核心常用的是前两个。

### 1. `el`（核心）

指令绑定的**真实 DOM 元素**，可以直接操作：

```javascript
mounted: (el) => {
  el.focus() // 聚焦输入框
  el.style.color = 'red' // 修改样式
  el.addEventListener('click', () => {}) // 绑定事件
}
```

### 2. `binding`（核心）

一个对象，包含指令的所有配置信息，最常用的属性如下：

| 属性名         | 说明                                                                        |
| ----------- | ------------------------------------------------------------------------- |
| `value`     | 指令绑定的值，如 `v-color="'red'"` 中，value 为 `'red'`                              |
| `oldValue`  | 指令上一次的值，仅在 `beforeUpdate`/`updated` 中可用                                   |
| `arg`       | 指令的参数，如 `v-fixed:top="20"` 中，arg 为 `'top'`                                |
| `modifiers` | 指令的修饰符对象，如 `v-click.once.stop` 中，modifiers 为 `{ once: true, stop: true }` |
| `instance`  | 使用该指令的组件实例                                                                |
### 3. `vnode` / `prevVnode`

Vue 内部的虚拟 DOM 节点，仅用于高级场景，日常开发几乎不用。

## 四、实战常用示例

### 1. 自动聚焦指令 `v-focus`

最基础的入门示例，页面加载后自动聚焦输入框：

```vue
<template>
  <input v-focus placeholder="页面加载自动聚焦" />
</template>

<script setup>
const vFocus = {
  mounted: (el) => el.focus()
}
</script>
```

### 2. 颜色 + 修饰符指令 `v-color`

支持动态颜色、加粗修饰符，演示 `value` 和 `modifiers` 的用法：

```vue
<template>
  <p v-color.bold="textColor">红色加粗文字</p>
  <button @click="textColor = 'blue'">切换为蓝色</button>
</template>

<script setup>
import { ref } from 'vue'
const textColor = ref('red')

// 函数简写，mounted和updated都执行
const vColor = (el, binding) => {
  el.style.color = binding.value
  // 有bold修饰符则加粗
  if (binding.modifiers.bold) {
    el.style.fontWeight = 'bold'
  }
}
</script>
```

### 3. 防抖点击指令 `v-debounce`

防止按钮重复点击，支持自定义防抖时间，演示 `arg` 和事件绑定：

```vue
<template>
  <button v-debounce:1000="handleSubmit">提交（1秒防抖）</button>
</template>

<script setup>
const handleSubmit = () => {
  console.log('提交成功，防抖生效')
}

const vDebounce = {
  mounted: (el, binding) => {
    let timer = null
    const handler = binding.value // 绑定的回调函数
    const delay = binding.arg || 500 // 防抖时间，默认500ms

    el.addEventListener('click', (e) => {
      clearTimeout(timer)
      timer = setTimeout(() => handler(e), delay)
    })
  },
  // 元素卸载时清理定时器，避免内存泄漏
  unmounted: (el) => {
    el.removeEventListener('click')
  }
}
</script>
```

### 4. 权限控制指令 `v-permission`

根据用户权限自动显隐元素，后台管理系统高频使用：

```vue
<template>
  <button v-permission="'user:add'">新增用户</button>
  <button v-permission="'user:delete'">删除用户</button>
</template>

<script setup>
// 模拟当前用户的权限列表
const userPermissions = ['user:add', 'user:edit']

const vPermission = {
  mounted: (el, binding) => {
    const requiredPermission = binding.value
    // 无权限则直接移除DOM元素
    if (!userPermissions.includes(requiredPermission)) {
      el.remove()
    }
  }
}
</script>
```

### 5. 动态参数指令 `v-fixed`

支持动态切换固定定位方向，演示动态指令参数：

```vue
<template>
  <div v-fixed:[direction]="20" style="width: 100px; height: 100px; background: red;">
    固定定位元素
  </div>
  <button @click="direction = 'left'">切换为左固定</button>
</template>

<script setup>
import { ref } from 'vue'
const direction = ref('top') // 响应式参数

const vFixed = (el, binding) => {
  el.style.position = 'fixed'
  el.style[binding.arg] = binding.value + 'px'
}
</script>
```

## 五、最佳实践与注意事项

### 1. 明确使用边界，不要滥用

- ✅ 正确场景：**和 DOM 强相关的通用逻辑**（自动聚焦、防抖、懒加载、拖拽、权限显隐等）。
- ❌ 错误场景：用指令处理表单提交、接口请求、业务数据逻辑（优先用组件 / 组合式函数）。

### 2. 避免内存泄漏

在 `unmounted` 钩子中，必须清理指令绑定的**事件监听、定时器、第三方实例**，否则会造成内存泄漏：

```javascript
const vScroll = {
  mounted: (el) => {
    el.addEventListener('scroll', handleScroll)
  },
  unmounted: (el) => {
    // 卸载时移除事件监听
    el.removeEventListener('scroll', handleScroll)
  }
}
```

### 3. 优先使用组合式函数

如果逻辑不强制依赖 DOM 操作，优先用**组合式函数（Composables）**，比指令更灵活、更符合 Vue 3 的 Composition API 设计理念。

### 4. 单一职责原则

一个指令只做一件事，不要把多个逻辑揉在同一个指令里（比如不要把防抖和权限控制写在一起），降低维护成本。

### 5. 兼容 Vue 内置特性

自定义指令不要破坏 Vue 的响应式特性，避免直接修改组件数据，所有数据变化都应该通过 Vue 的响应式系统处理。

## 六、Vue 3 与 Vue 2 的核心差异

1. **钩子函数重命名**：对齐组件生命周期，更易理解，废弃了 Vue 2 的`bind`/`inserted`等钩子。
2. **注册方式简化**：`<script setup>` 中`v`开头的驼峰变量自动识别为指令，无需额外注册。
3. **移除指令`this`**：Vue 3 指令钩子中没有`this`，如需访问组件实例，通过`binding.instance`获取。
4. **`v-model`兼容**：Vue 3 中组件的`v-model`机制变化，指令绑定的元素不会再受组件`v-model`的影响。
