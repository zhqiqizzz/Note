在编程中，尤其是处理对象和数组等复杂数据结构时，“拷贝”是一个非常基础且重要的概念。深拷贝和浅拷贝的核心区别在于**是否递归地复制了对象内部引用的所有数据**。

简单来说：

- **浅拷贝 (Shallow Copy)**：只复制了对象的“表面”，内部的“零件”还是共用的。
- **深拷贝 (Deep Copy)**：复制了对象及其内部所有的“零件”，创建了一个完全独立的副本。

为了理解它们，我们首先要明确一个核心概念：**引用类型**。在 JavaScript、Python 等语言中，对象、数组等是引用类型。变量存储的不是数据本身，而是指向内存中数据存储位置的“地址”（或称指针）。
### 什么是浅拷贝？

浅拷贝会创建一个新的对象（或数组），然后将原对象**第一层**的属性值复制过去。

- 如果属性值是**基本类型**（如数字、字符串），则直接复制这个值。
- 如果属性值是**引用类型**（如另一个对象、数组），则复制的是这个引用类型的**内存地址**。

**结果就是**：新对象和原对象在顶层是独立的，但它们内部嵌套的对象或数组，仍然指向同一块内存。修改其中一个对象的嵌套属性，会影响到另一个。

#### 代码示例 (JavaScript)

```javascript
const original = {
  name: 'Alice',
  info: {
    age: 25,
    city: 'Beijing'
  }
};

// 使用展开运算符 (...) 进行浅拷贝
const shallowCopy = { ...original };

// 1. 修改顶层的基本类型属性
shallowCopy.name = 'Bob';
console.log(original.name); // 输出: 'Alice' (不受影响)
console.log(shallowCopy.name); // 输出: 'Bob'

// 2. 修改嵌套的引用类型属性
shallowCopy.info.age = 26;
console.log(original.info.age); // 输出: 26 (被影响了！)
console.log(shallowCopy.info.age); // 输出: 26
```

#### 常见的浅拷贝方法

|语言|方法|
|---|---|
|**JavaScript**|展开运算符 (`{...obj}`, `[...arr]`)、`Object.assign()`、`Array.prototype.slice()`、`Array.prototype.concat()`|
|**Python**|`copy.copy()`、切片操作 (`[:]`)、`dict.copy()`|
|**Java**|实现 `Cloneable` 接口并重写 `clone()` 方法（默认是浅拷贝）|
### 什么是深拷贝？

深拷贝会递归地遍历整个对象树，为原对象及其内部引用的**所有层级的子对象**都创建一份全新的、独立的副本。

**结果就是**：新对象和原对象在内存中是完全独立的两个实体，互不干扰。修改任何一个对象，都不会影响到另一个。

#### 代码示例 (JavaScript)

```javascript
const original = {
  name: 'Alice',
  info: {
    age: 25,
    city: 'Beijing'
  }
};

// 使用 structuredClone() 进行深拷贝 (现代浏览器推荐)
const deepCopy = structuredClone(original);

// 1. 修改顶层属性
deepCopy.name = 'Bob';
console.log(original.name); // 输出: 'Alice' (不受影响)

// 2. 修改嵌套属性
deepCopy.info.age = 26;
console.log(original.info.age); // 输出: 25 (完全不受影响！)
console.log(deepCopy.info.age); // 输出: 26
```

#### 常见的深拷贝方法

|语言|方法|特点与限制|
|---|---|---|
|**JavaScript**|`structuredClone()`|**现代标准方案**。支持循环引用、Date、Map等，但无法拷贝函数、`undefined`。|
|**JavaScript**|`JSON.parse(JSON.stringify(obj))`|**传统“土办法”**。简单但有严重缺陷：会丢失函数、`undefined`、`Symbol`，无法处理循环引用，`Date`会变字符串。|
|**Python**|`copy.deepcopy()`|**标准方案**。功能强大，但性能开销较大，且无法拷贝文件句柄、线程锁等特殊对象。|
|**Java**|序列化/反序列化|通过将对象转为字节流再还原的方式实现深拷贝，要求对象实现 `Serializable` 接口。|
### 如何选择？性能与场景

|特性|浅拷贝 (Shallow Copy)|深拷贝 (Deep Copy)|
|---|---|---|
|**内存占用**|低，共享嵌套对象|高，完全复制所有对象|
|**执行速度**|**快**|**慢** (尤其是深层嵌套时)|
|**独立性**|部分独立，嵌套属性会互相影响|完全独立，互不干扰|
|**适用场景**|对象结构简单，或**明确希望共享**内部数据|需要完全隔离数据，避免副作用，如复杂的状态管理|
