## 一、标准首选：`addEventListener` 方法

这是现代浏览器通用的标准方案，功能最完善、灵活性最高，也是企业级开发的唯一推荐用法。

### 1. 基础语法

```javascript
element.addEventListener(eventType, handler, options/useCapture)
```

#### 参数详解

| 参数          | 说明                                                                                          | 核心注意点                           |
| :---------- | :------------------------------------------------------------------------------------------ | :------------------------------ |
| `eventType` | 事件类型字符串，比如 `click`、`input`、`scroll`、`DOMContentLoaded`                                      | **不能加`on`前缀**，比如不能写`onclick`    |
| `handler`   | 事件处理函数，事件触发时执行，支持具名函数、匿名函数、箭头函数                                                             | 如需移除监听，**必须用具名函数**              |
| 第三个参数（可选）   | 两种格式：<br><br>1. 布尔值`useCapture`：是否在捕获阶段触发，默认`false`（冒泡阶段）<br><br>2. 配置对象`options`：精细化控制事件行为 | 常用配置：`once`、`passive`、`capture` |
### 2. 基础使用示例

```html
<button id="btn">点击按钮</button>
<script>
  // 1. 获取DOM元素
  const btn = document.getElementById('btn');

  // 2. 绑定点击事件（具名函数，推荐）
  function handleBtnClick(e) {
    console.log('按钮被点击了', e);
  }
  btn.addEventListener('click', handleBtnClick);

  // 3. 也可以用匿名函数（不推荐，无法移除）
  btn.addEventListener('click', () => {
    console.log('匿名函数触发');
  });
</script>
```

### 3. 进阶配置项（高频实用）

第三个参数的配置对象，能解决很多开发中的高频需求：

```javascript
// 1. once: 事件只触发一次，触发后自动移除监听
btn.addEventListener('click', () => {
  console.log('只会触发一次');
}, { once: true });

// 2. passive: 提升滚动性能，禁止在函数中调用preventDefault()
// 高频用于scroll、touchmove等事件，解决移动端滚动卡顿
window.addEventListener('scroll', () => {
  // 页面滚动逻辑
}, { passive: true });

// 3. capture: 控制在捕获阶段触发事件，默认false（冒泡阶段）
parent.addEventListener('click', () => {
  console.log('父元素捕获阶段触发');
}, { capture: true });
```

### 4. 核心优势

1. 同一个元素的同一个事件，可绑定**多个不同的处理函数**，不会互相覆盖；
2. 可精确控制事件触发阶段（捕获 / 冒泡）；
3. 支持`once`、`passive`等精细化配置，适配复杂场景；
4. 可通过`removeEventListener`精准移除监听，避免内存泄漏。

## 二、传统兼容方案：DOM0 级 `onxxx` 属性绑定

这是 JS 早期的事件绑定方式，写法极简，兼容性拉满（兼容所有老旧浏览器），但有明显局限性。

### 1. 用法示例

```html
<button id="btn">点击按钮</button>
<script>
  const btn = document.getElementById('btn');
  
  // 绑定点击事件
  btn.onclick = function(e) {
    console.log('按钮点击触发', e);
  };

  // 移除事件：直接赋值null
  btn.onclick = null;
</script>
```

### 2. 优缺点

- ✅ 优点：写法极简、代码量少，兼容 IE6 等极端老旧浏览器；
- ❌ 缺点：**同一个元素的同一个事件，只能绑定一个处理函数**，后面的赋值会直接覆盖前面的；无法控制事件触发阶段，只能在冒泡阶段执行。

## 三、不推荐：HTML 内联事件绑定

直接在 HTML 标签中通过`onxxx`属性绑定事件，是最早期的写法，**现代开发完全不推荐**。
### 1. 用法示例

```html
<!-- 直接在标签内写执行逻辑 -->
<button onclick="console.log('点击触发')">按钮1</button>

<!-- 绑定全局函数 -->
<button onclick="handleClick()">按钮2</button>
<script>
  // 函数必须挂载到全局window上，否则会报错
  window.handleClick = function() {
    console.log('全局函数触发');
  };
</script>
```

### 2. 不推荐的核心原因

1. **结构与行为强耦合**：HTML 负责结构，JS 负责行为，混写不利于代码维护和迭代；
2. 全局作用域污染：处理函数必须挂载到`window`上，容易出现命名冲突；
3. 无安全性保障：存在 XSS 脚本注入风险；
4. 灵活性极差：无法控制事件阶段，无法批量绑定，不支持复杂逻辑。
## 四、进阶实战与最佳实践

### 1. 事件委托（事件代理）：动态元素必用方案

#### 核心原理

利用**事件冒泡**机制：子元素的事件会向上冒泡到父元素，给父元素绑定一次监听，就能处理所有子元素的事件，尤其适合**动态生成的 DOM 元素**、列表循环元素。

#### 实战示例

```html
<ul id="list">
  <li>列表项1</li>
  <li>列表项2</li>
  <li>列表项3</li>
  <!-- 后续动态新增的li，无需重新绑定事件 -->
</ul>
<script>
  const list = document.getElementById('list');

  // 给父元素ul绑定监听，代理所有子元素li的点击事件
  list.addEventListener('click', (e) => {
    // e.target: 实际触发事件的元素（点击的li）
    // e.currentTarget: 绑定事件的元素（ul）
    const target = e.target;
    // 判断触发元素是否是目标li
    if (target.matches('li')) {
      console.log('点击了列表项:', target.textContent);
    }
  });

  // 动态新增li，无需重新绑定事件，自动生效
  const newLi = document.createElement('li');
  newLi.textContent = '动态新增的列表项';
  list.appendChild(newLi);
</script>
```

#### 核心优势

- 减少事件绑定数量，节省内存开销；
- 动态新增 / 删除的子元素，无需重复绑定 / 解绑事件；
- 代码量大幅减少，维护成本更低。

### 2. 正确移除事件监听（避坑重点）

很多新手会踩「事件无法移除」的坑，核心原因是**处理函数的引用不一致**。

#### 错误示例（无法移除）

```javascript
const btn = document.getElementById('btn');

// 匿名函数：每次创建都是一个全新的函数引用
btn.addEventListener('click', () => {
  console.log('点击');
});

// 移除无效：两个匿名函数引用地址不同，不是同一个函数
btn.removeEventListener('click', () => {
  console.log('点击');
});
```

#### 正确示例（可正常移除）

```javascript
const btn = document.getElementById('btn');

// 具名函数：引用地址固定
function handleClick() {
  console.log('点击');
}

// 绑定
btn.addEventListener('click', handleClick);
// 移除：必须和绑定的是同一个函数引用
btn.removeEventListener('click', handleClick);
```

#### 补充

如果事件只需要触发一次，直接用`{ once: true }`配置，触发后会自动移除，无需手动调用`removeEventListener`。

### 3. 处理函数的`this`指向问题

```javascript
const btn = document.getElementById('btn');

// 1. 普通函数：this 指向触发事件的DOM元素（btn）
btn.addEventListener('click', function() {
  console.log(this); // 输出<button>元素
});

// 2. 箭头函数：this 指向外层作用域的this（不会绑定到元素）
btn.addEventListener('click', () => {
  console.log(this); // 输出window，或Vue/React组件实例
});
```

> 提示：Vue/React 组件中，如需在事件函数中访问组件实例，优先使用箭头函数，或通过`bind`绑定 this。

### 4. 事件对象`event`核心用法

事件处理函数的第一个参数，就是事件对象`event`（常简写为`e`），包含事件的所有信息，高频用法如下：

```javascript
btn.addEventListener('click', (e) => {
  // 1. 阻止默认行为：比如a标签跳转、表单提交、右键菜单
  e.preventDefault();

  // 2. 阻止事件冒泡/捕获：不让事件向上传递给父元素
  e.stopPropagation();

  // 3. 核心属性
  console.log(e.target); // 实际触发事件的元素
  console.log(e.currentTarget); // 绑定事件的元素
  console.log(e.clientX, e.clientY); // 鼠标点击的坐标
});
```

### 5. 高频事件优化

针对`scroll`、`resize`、`mousemove`、`input`等高频触发的事件，需要做防抖 / 节流处理，避免频繁执行逻辑导致页面卡顿：

```javascript
// 节流函数：固定时间内只执行一次
function throttle(fn, delay = 200) {
  let timer = null;
  return function(...args) {
    if (!timer) {
      timer = setTimeout(() => {
        fn.apply(this, args);
        timer = null;
      }, delay);
    }
  };
}

// 滚动事件添加节流，200ms内只执行一次
window.addEventListener('scroll', throttle(() => {
  console.log('滚动逻辑执行');
}, 200), { passive: true });
```

## 五、主流框架中的事件绑定

Vue、React 等框架对原生事件监听做了封装，简化了写法，本质还是基于原生事件体系。

### 1. Vue 3 事件绑定

```vue
<template>
  <!-- 基础点击事件 -->
  <button @click="handleClick">按钮</button>
  <!-- 只触发一次 -->
  <button @click.once="handleOnce">只触发一次</button>
  <!-- 阻止默认行为 -->
  <a href="/" @click.prevent="handleLink">链接</a>
  <!-- 事件委托 -->
  <ul @click="handleListClick">
    <li v-for="item in list" :key="item.id">{{ item.name }}</li>
  </ul>
</template>

<script setup>
const handleClick = () => {
  console.log('点击触发');
};
const handleOnce = () => {
  console.log('只触发一次');
};
const handleLink = () => {
  console.log('链接点击，不会跳转');
};
const handleListClick = (e) => {
  if (e.target.matches('li')) {
    console.log('点击了列表项');
  }
};
</script>
```
