## 一、先搞懂 Vue 3 双向绑定的 “骨架”

Vue 3 的双向数据绑定，本质是 **「响应式系统」+「视图交互」** 的结合：

1. **数据 → 视图**：数据变了，视图自动更新（这是核心，靠 “响应式系统” 实现）；
2. **视图 → 数据**：用户在表单里输入内容，数据自动变（这是辅助，靠 “事件监听” 实现）。

我们先花 80% 的精力，搞懂最核心的 **「响应式系统」**—— 这是 Vue 3 最厉害的地方，也是和 Vue 2 最大的区别。

---

## 二、响应式系统的第一步：用 Proxy 给对象 “装监控”

Vue 3 响应式系统的基础是 **ES6 的 `Proxy`（代理）**。我们先彻底搞懂 Proxy，因为它是整个响应式的 “监控器”。

> 补充： [Proxy](Proxy.md)
### 1. Proxy 是什么？

你可以把 `Proxy` 理解成**给目标对象套了一层 “监控壳”**—— 外界对这个对象的所有操作（读属性、写属性、删属性），都必须先经过这层壳，我们可以在壳里拦截这些操作，做额外的事情。

### 2. 最简单的 Proxy 例子

先看代码，直观感受一下：

```javascript
// 1. 目标对象：我们要监控的普通对象
const person = {
  name: "张三",
  age: 20
};

// 2. 用 Proxy 给 person 装个“监控壳”
const personProxy = new Proxy(person, {
  // 拦截“读属性”操作：比如 personProxy.name
  get(target, prop) {
    console.log(`【监控】有人在读取属性：${prop}`);
    return target[prop]; // 把原本的属性值返回
  },

  // 拦截“写属性”操作：比如 personProxy.age = 25
  set(target, prop, value) {
    console.log(`【监控】有人在修改属性：${prop}，新值是：${value}`);
    target[prop] = value; // 把新值存到目标对象里
    return true; // 告诉 Proxy 修改成功了
  }
});

// 3. 操作“监控壳”（注意：必须操作 personProxy，操作 person 没用）
console.log(personProxy.name); 
// 输出：【监控】有人在读取属性：name → 张三

personProxy.age = 25; 
// 输出：【监控】有人在修改属性：age，新值是：25
```

### 3. 核心概念拆解

- **target（目标对象）**：被监控的那个普通对象（比如上面的 `person`）。
- **handler（处理器对象）**：用来定义 “监控规则” 的对象（比如上面的 `{ get, set }`）。
- **proxy（代理对象）**：`new Proxy()` 返回的新对象（比如上面的 `personProxy`）——**只有操作 proxy，才会触发监控**。

## 三、响应式系统的第二步：副作用函数（effect）

现在我们有了 Proxy 监控对象，但监控到数据变化后，怎么自动更新视图呢？这就需要 **「副作用函数」**。

### 1. 先搞懂什么是 “副作用”

“副作用” 就是：**你做了一件事，除了这件事本身，还对外部产生了其他影响**。

在前端开发中，**最典型的副作用就是 “更新页面视图”**—— 比如你修改了数据，页面上的文字跟着变了，这就是副作用。

### 2. 副作用函数的简单例子

我们先写一个简单的副作用函数：

```javascript
// 目标对象
const state = { count: 0 };

// 副作用函数：更新页面视图
function updateView() {
  document.getElementById("count").textContent = state.count;
  console.log("【副作用】页面已更新！");
}

// 初始化时执行一次，渲染页面
updateView();
```

现在问题来了：我们修改 `state.count` 时，怎么让 `updateView()` 自动执行呢？

## 四、响应式系统的第三步：依赖收集（track）+ 触发更新（trigger）

现在我们把 Proxy 和副作用函数结合起来：

- **读取属性时（get）**：做「依赖收集」—— 记录 “这个副作用函数依赖了这个属性”；
- **修改属性时（set）**：做「触发更新」—— 找到 “所有依赖这个属性的副作用函数，依次执行”。

### 1. 先准备一个 “依赖关系表”

我们需要一个数据结构，来存储 “哪个属性被哪些副作用函数依赖了”，就像一个 “通讯录”：

```javascript
// 依赖关系表：WeakMap(目标对象 -> Map(属性名 -> Set(副作用函数集合)))
const targetMap = new WeakMap();

// 全局变量：临时保存当前正在执行的副作用函数
let activeEffect = null;
```

这个结构看起来复杂，其实很简单：

- 第一层 `WeakMap`：存 “目标对象” 和它对应的 “属性 - 副作用映射”；
- 第二层 `Map`：存 “属性名” 和它对应的 “副作用函数集合”；
- 第三层 `Set`：存依赖这个属性的所有副作用函数（用 Set 是为了避免重复）。

### 2. 依赖收集（track）：记录 “谁依赖了谁”

当读取属性时，我们把当前正在执行的副作用函数，存到依赖关系表里：

```javascript
function track(target, prop) {
  // 如果没有正在执行的副作用函数，就不用收集
  if (!activeEffect) return;

  // 1. 从 targetMap 中取出当前目标对象对应的 Map
  let depsMap = targetMap.get(target);
  if (!depsMap) {
    depsMap = new Map();
    targetMap.set(target, depsMap);
  }

  // 2. 从 depsMap 中取出当前属性对应的 Set（副作用函数集合）
  let deps = depsMap.get(prop);
  if (!deps) {
    deps = new Set();
    depsMap.set(prop, deps);
  }

  // 3. 把当前副作用函数加入 Set（完成依赖收集）
  deps.add(activeEffect);
  console.log(`【依赖收集】属性 ${prop} 被依赖了`);
}
```

### 3. 触发更新（trigger）：数据变了，执行副作用

当修改属性时，我们从依赖关系表里，找到所有依赖这个属性的副作用函数，依次执行：

```javascript
function trigger(target, prop) {
  // 1. 从 targetMap 中取出当前目标对象对应的 Map
  const depsMap = targetMap.get(target);
  if (!depsMap) return;

  // 2. 从 depsMap 中取出当前属性对应的副作用函数集合
  const effects = depsMap.get(prop);
  if (!effects) return;

  // 3. 依次执行所有副作用函数
  console.log(`【触发更新】属性 ${prop} 变了，执行副作用`);
  effects.forEach(effect => effect());
}
```

### 4. 副作用函数注册器（effect）：把副作用函数包装一下

我们需要一个函数，来注册副作用函数，并且在注册时立即执行一次，触发依赖收集：

```javascript
function effect(fn) {
  // 包装一下副作用函数
  const effectFn = () => {
    activeEffect = effectFn; // 把当前副作用函数赋值给全局变量
    fn(); // 执行副作用函数，触发 get，完成依赖收集
    activeEffect = null; // 执行完清空
  };
  effectFn(); // 初始化时执行一次
  return effectFn;
}
```

## 五、把它们拼起来：一个极简版的 Vue 3 响应式系统

现在我们把上面的代码全部拼起来，看看完整的效果：

```html
<!-- 页面上的元素 -->
<div>
  点击次数：<span id="count">0</span>
  <button id="btn">点击+1</button>
</div>

<script>
  // 1. 依赖关系表
  const targetMap = new WeakMap();
  let activeEffect = null;

  // 2. 依赖收集
  function track(target, prop) {
    if (!activeEffect) return;
    let depsMap = targetMap.get(target);
    if (!depsMap) {
      depsMap = new Map();
      targetMap.set(target, depsMap);
    }
    let deps = depsMap.get(prop);
    if (!deps) {
      deps = new Set();
      depsMap.set(prop, deps);
    }
    deps.add(activeEffect);
    console.log(`【依赖收集】属性 ${prop} 被依赖了`);
  }

  // 3. 触发更新
  function trigger(target, prop) {
    const depsMap = targetMap.get(target);
    if (!depsMap) return;
    const effects = depsMap.get(prop);
    if (!effects) return;
    console.log(`【触发更新】属性 ${prop} 变了，执行副作用`);
    effects.forEach(effect => effect());
  }

  // 4. 用 Proxy 创建响应式对象
  function reactive(target) {
    return new Proxy(target, {
      get(target, prop, receiver) {
        const res = Reflect.get(target, prop, receiver);
        track(target, prop); // 读取属性时，收集依赖
        // 如果是对象，递归代理（懒执行：只有访问时才代理）
        return typeof res === 'object' && res !== null ? reactive(res) : res;
      },
      set(target, prop, value, receiver) {
        const oldValue = Reflect.get(target, prop, receiver);
        const res = Reflect.set(target, prop, value, receiver);
        // 只有值真的变了，才触发更新
        if (oldValue !== value) {
          trigger(target, prop); // 修改属性时，触发更新
        }
        return res;
      }
    });
  }

  // 5. 副作用函数注册器
  function effect(fn) {
    const effectFn = () => {
      activeEffect = effectFn;
      fn();
      activeEffect = null;
    };
    effectFn();
    return effectFn;
  }

  // ---------------- 测试代码 ----------------
  // 6. 创建响应式对象
  const state = reactive({ count: 0 });

  // 7. 注册副作用函数：更新页面
  effect(() => {
    document.getElementById("count").textContent = state.count;
    console.log("【副作用】页面已更新！");
  });

  // 8. 点击按钮，修改数据
  document.getElementById("btn").addEventListener("click", () => {
    state.count++;
  });
</script>
```

### 运行效果

1. 页面加载时，会输出：
    
```plaintext
    【依赖收集】属性 count 被依赖了
    【副作用】页面已更新！
```
    
2. 点击按钮，`state.count` 加 1，会输出：
    
```plaintext
    【触发更新】属性 count 变了，执行副作用
    【依赖收集】属性 count 被依赖了
    【副作用】页面已更新！
```

3. 页面上的 “点击次数” 会自动更新！

## 六、Vue 3 中实际的响应式 API

上面我们写的是极简版原理，Vue 3 实际提供了更完善的响应式 API，我们来看看最常用的几个：

### 1. `reactive`：创建响应式对象

用于创建一个响应式的**对象 / 数组 / Map/Set**，就是我们上面原理中 `reactive` 函数的完善版：

```javascript
import { reactive } from 'vue';

const state = reactive({
  count: 0,
  user: { name: "张三" }
});

// 修改数据，自动触发更新
state.count++;
state.user.name = "李四";
```

### 2. `ref`：创建响应式的 “原始值”

`reactive` 只能用于对象 / 数组，对于原始值（数字、字符串、布尔值），Vue 3 提供了 `ref`：

```javascript
import { ref } from 'vue';

const count = ref(0);
const name = ref("张三");

// 注意：在 JS 中访问/修改 ref 的值，需要加 .value
console.log(count.value); // 0
count.value++;

// 在模板中使用，不需要加 .value，Vue 会自动解包
// <div>{{ count }}</div>
```

### 3. `computed`：计算属性

用于创建一个**依赖其他响应式数据的计算值**，只有依赖的数据变化时，才会重新计算：

```javascript
import { ref, computed } from 'vue';

const firstName = ref("张");
const lastName = ref("三");

// 计算属性：依赖 firstName 和 lastName
const fullName = computed(() => {
  return firstName.value + lastName.value;
});

console.log(fullName.value); // 张三
firstName.value = "李";
console.log(fullName.value); // 李四
```

### 4. `watch`：监听数据变化

用于监听某个响应式数据的变化，执行自定义逻辑：

```javascript
import { ref, watch } from 'vue';

const count = ref(0);

// 监听 count 的变化
watch(count, (newValue, oldValue) => {
  console.log(`count 从 ${oldValue} 变成了 ${newValue}`);
});

count.value++; // 输出：count 从 0 变成了 1
```

## 七、双向绑定的直观体现：`v-model` 语法糖

现在我们搞懂了响应式系统（数据 → 视图），再看**视图 → 数据**，最典型的就是 `v-model`。

### 1. `v-model` 本质是语法糖

在 Vue 3 中，`v-model` 其实是 **「数据绑定」+「事件监听」** 的简写：

```html
<!-- 写法 1：用 v-model -->
<input v-model="message">

<!-- 写法 2：等价的完整写法（语法糖展开） -->
<input 
  :modelValue="message" 
  @update:modelValue="message = $event"
>
```

### 2. 完整的 `v-model` 示例

我们用 Vue 3 的 Composition API，写一个完整的双向绑定示例：

```html
<template>
  <div>
    <input v-model="message" placeholder="输入点什么">
    <p>你输入的是：{{ message }}</p>
  </div>
</template>

<script setup>
import { ref } from 'vue';

// 创建响应式数据
const message = ref("");
</script>
```

### 3. 原理拆解

这个示例的完整流程是：

1. **初始化**：`message` 是 `ref` 创建的响应式数据，模板渲染时，读取 `message`，触发依赖收集；
2. **视图 → 数据**：用户在输入框输入内容，触发 `@update:modelValue` 事件，把输入的值赋给 `message.value`；
3. **数据 → 视图**：`message.value` 变化，触发响应式系统的 `trigger`，执行副作用函数，重新渲染模板，页面上的 `<p>` 标签内容更新。

## 八、总结：Vue 3 双向绑定的完整流程

我们把所有内容串起来，总结一下完整的流程：

### 1. 初始化阶段

1. 用 `reactive`/`ref` 创建响应式数据（内部用 `Proxy` 代理对象）；
2. 执行组件的渲染函数（副作用函数），读取响应式数据；
3. 读取数据时，触发 `Proxy` 的 `get` 拦截，调用 `track` 完成依赖收集（记录 “渲染函数依赖了这个属性”）。

### 2. 数据变化阶段

1. 用户修改响应式数据（比如 `state.count++` 或 `v-model` 输入）；
2. 修改数据时，触发 `Proxy` 的 `set` 拦截；
3. 调用 `trigger`，从依赖关系表中找到所有依赖该属性的副作用函数（比如渲染函数）；
4. 依次执行副作用函数，重新渲染视图。
