根据搜索结果，Vue Router 主要提供 **3 种核心传参方式**，以下是详细介绍：

## 一、Query 传参（查询参数）

参数显示在 URL 问号后面，类似 `?id=1&name=张三`

### 路由配置

```javascript
// router/index.js
{
  path: '/search',
  name: 'Search',
  component: () => import('@/views/Search.vue')
}
```

### 传递参数

```javascript
// Vue 3 (Composition API)
import { useRouter } from 'vue-router'
const router = useRouter()

router.push({
  path: '/search',
  query: {
    id: 1,
    name: '张三',
    keyword: 'vue'
  }
})

// 或使用 router-link
<router-link :to="{ path: '/search', query: { id: 1, name: '张三' } }">跳转</router-link>
```

### 接收参数

```javascript
import { useRoute } from 'vue-router'
const route = useRoute()

// 获取单个参数
console.log(route.query.id)
console.log(route.query.name)

// 获取全部参数
console.log(route.query) // { id: '1', name: '张三', keyword: 'vue' }
```

### 特点

|优点|缺点|
|---|---|
|✅ 刷新页面参数不丢失|❌ 参数暴露在URL中|
|✅ 适合搜索、筛选场景|❌ 不适合传递敏感数据|
|✅ 可分享带参数的链接|❌ URL可能过长|

## 二、Params 传参（动态路由参数）

参数嵌入在路径中，类似 `/user/123`
### 路由配置

```javascript
{
  path: '/user/:id',           // 单参数
  name: 'User',
  component: () => import('@/views/User.vue')
}

// 或多参数
{
  path: '/user/:id/:age',
  name: 'UserDetail',
  component: () => import('@/views/UserDetail.vue')
}
```

### 传递参数

```javascript
// 方式1：使用 name + params（推荐）
router.push({
  name: 'User',
  params: {
    id: 123
  }
})

// 方式2：直接拼接路径
router.push(`/user/${123}`)

// 方式3：使用 router-link
<router-link :to="{ name: 'User', params: { id: 123 } }">用户详情</router-link>
```

### 接收参数

```javascript
import { useRoute } from 'vue-router'
const route = useRoute()

console.log(route.params.id)   // 123
console.log(route.params.age)  // 如果有多个参数
```

### 特点

|优点|缺点|
|---|---|
|✅ URL更简洁美观|❌ 刷新后若未配置路由可能丢失|
|✅ SEO友好|❌ 必须提前定义路由参数|
|✅ 适合详情页、资源ID场景|❌ 参数仍暴露在URL中|

## 三、Props 传参（推荐方式）

将路由参数直接映射为组件 props，**解耦组件与路由**
### 路由配置

```javascript
{
  path: '/user/:id',
  name: 'User',
  component: () => import('@/views/User.vue'),
  props: true  // 将 params 映射为 props
}

// 或使用函数模式（更灵活）
{
  path: '/user/:id',
  component: User,
  props: (route) => ({ 
    id: route.params.id,
    from: route.query.from 
  })
}

// 或使用对象模式（固定值）
{
  path: '/user',
  component: User,
  props: { id: 123, name: '默认用户' }
}
```

### 接收参数

```vue
<!-- Vue 3 Composition API -->
<script setup>
defineProps({
  id: {
    type: [String, Number],
    required: true
  },
  name: {
    type: String,
    default: ''
  }
})
</script>

<!-- Vue 3 Options API -->
<script>
export default {
  props: ['id', 'name'],
  mounted() {
    console.log(this.id)
  }
}
</script>
```
### 特点

|优点|缺点|
|---|---|
|✅ 组件与路由解耦，复用性强|❌ 需要额外配置|
|✅ 类型安全，便于测试|❌ 仅适用于已定义的路由参数|
|✅ 代码更清晰规范||

---

## 四、三种方式对比总结

|传参方式|URL显示|刷新保留|适用场景|推荐度|
|---|---|---|---|---|
|**query**|`?id=1&name=张三`|✅ 是|搜索、筛选、分页|⭐⭐⭐⭐|
|**params**|`/user/123`|✅ 是|详情页、资源ID|⭐⭐⭐⭐|
|**props**|取决于配置|✅ 是|组件复用、大型项目|⭐⭐⭐⭐⭐|

## 五、Vue 3 完整示例

```vue
<!-- 列表页：传递参数 -->
<script setup>
import { useRouter } from 'vue-router'
const router = useRouter()

const goToDetail = (id) => {
  // 方式1：params
  router.push({ name: 'Detail', params: { id } })
  
  // 方式2：query
  router.push({ path: '/detail', query: { id, type: 'article' } })
}
</script>

<!-- 详情页：接收参数 -->
<script setup>
import { useRoute } from 'vue-router'
const route = useRoute()

// 接收 params
const id = route.params.id

// 接收 query
const type = route.query.type
</script>
```

## 💡 最佳实践建议

1. **敏感数据**：不要通过路由传参，使用 Pinia/Vuex 或 sessionStorage
2. **复杂对象**：使用 query 时注意URL长度限制，复杂数据建议存状态管理
3. **SEO需求**：优先使用 params，URL更清晰
4. **组件复用**：使用 props 模式，降低耦合度
5. **Vue 3项目**：推荐使用 `useRouter()` 和 `useRoute()` 组合式API
