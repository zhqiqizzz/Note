路由守卫是 Vue Router 提供的**路由跳转拦截机制**，它允许我们在路由跳转的不同阶段（跳转前、跳转中、跳转后）插入自定义逻辑，实现权限控制、页面标题设置、数据预加载、离开确认等核心功能，是单页应用（SPA）中路由管理的 “安全闸” 和 “预处理中心”。

## 一、路由守卫的核心分类

Vue Router 4 的路由守卫分为三大类，按作用范围从大到小依次是：

1. **全局守卫**：作用于所有路由跳转；
2. **路由独享守卫**：仅作用于特定路由；
3. **组件内守卫**：仅作用于特定组件。

所有守卫的核心参数都是 `to`（即将进入的目标路由）、`from`（当前离开的路由），部分守卫支持 `next()`（控制跳转流程的回调）。

| 钩子                         | 触发时机                                      | 核心拦截能力                                                                            |
| -------------------------- | ----------------------------------------- | --------------------------------------------------------------------------------- |
| `beforeEach`（全局前置守卫）       | **路由跳转开始前**，任何路由跳转都会触发                    | 最强拦截能力：可以取消跳转（`return false`）、重定向到其他页面（`return '/login'`）、放行（`return true`/ 无返回值） |
| `beforeEnter`（路由独享守卫）      | 进入**特定路由**前触发                             | 只针对单个路由做拦截，逻辑同 `beforeEach`                                                       |
| `afterEach`（全局后置守卫）        | 在**路由跳转完成后**触发（此时组件已经挂载）                  | 页面埋点、滚动到顶部等                                                                       |
| `beforeRouteEnter`（组件内守卫）  | 在**进入该组件对应的路由前**触发，此时**组件实例还未创建**         | 组件数据预加载等                                                                          |
| `beforeRouteUpdate`（组件内守卫） | 当前路由参数变化（如 `/user/1` → `/user/2`），组件复用时触发 | 拦截参数变化，重新加载数据                                                                     |
| `beforeRouteLeave`（组件内守卫）  | 离开当前路由前触发                                 | 拦截离开，比如提示用户「表单未保存，确定要离开吗？」                                                        |

## 二、全局守卫

全局守卫在 `src/router/index.js` 中配置，作用于**所有路由跳转**，是最常用的守卫类型。

### 1. 全局前置守卫：`router.beforeEach`

#### 触发时机

在**任何路由跳转开始前**触发，是路由跳转的第一道 “关卡”。
#### 核心作用

- 权限控制（如未登录用户禁止访问需要登录的页面）；
- 页面标题设置；
- 路由跳转拦截（如禁止访问某些页面）。
#### 参数说明

- `to`：即将进入的目标路由对象（包含 `path`、`params`、`query`、`meta` 等）；
- `from`：当前正在离开的路由对象；
- `next()`（可选，Vue Router 4 推荐用 `return` 替代）：控制跳转流程的回调。

#### 跳转控制方式

Vue Router 4 推荐用 `return`

|操作|`next()` 写法|`return` 写法（推荐）|说明|
|---|---|---|---|
|继续跳转|`next()`|`return true` 或不返回|正常进入目标路由|
|取消跳转|`next(false)`|`return false`|中断跳转，停留在当前页面|
|跳转到其他路由|`next('/login')` 或 `next({ name: 'Login' })`|`return '/login'` 或 `return { name: 'Login' }`|重定向到指定路由|
#### 示例

权限控制 + 页面标题

```javascript
// src/router/index.js
router.beforeEach((to, from) => {
  // 1. 设置页面标题
  document.title = to.meta.title || '默认标题';

  // 2. 权限控制：检查是否需要登录
  const isLoggedIn = localStorage.getItem('token'); // 模拟登录状态
  if (to.meta.requiresAuth && !isLoggedIn) {
    // 未登录，跳转到登录页，并携带当前路径（登录后可跳转回来）
    return { name: 'Login', query: { redirect: to.fullPath } };
  }

  // 3. 已登录用户禁止访问登录页
  if (to.name === 'Login' && isLoggedIn) {
    return { name: 'Home' };
  }

  // 4. 其他情况，继续跳转
  return true;
});
```

### 2. 全局解析守卫：`router.beforeResolve`

#### 触发时机

在**所有路由守卫和异步路由组件解析完成后**、**路由跳转确认前**触发。

#### 核心作用

与 `beforeEach` 类似，但它是在所有守卫（包括组件内守卫）执行完后才触发，适合处理**需要确保所有异步组件加载完成**的逻辑（如数据预加载的兜底）。
#### 示例

```javascript
router.beforeResolve((to, from) => {
  console.log('所有路由守卫解析完成，即将跳转');
  // 可以在这里做最后的数据检查
});
```

### 3. 全局后置钩子：`router.afterEach`

#### 触发时机

在**路由跳转完成后**触发（此时组件已经挂载）。
#### 核心特点

- **没有 `next()` 参数**，无法控制跳转流程；
- 是路由跳转的 “收尾” 阶段。
#### 核心作用

- 页面埋点（统计页面访问量）；
- 关闭加载状态（如跳转前显示的 loading 动画）；
- 滚动到页面顶部（也可在路由配置的 `scrollBehavior` 中实现）。
#### 示例

```javascript
router.afterEach((to, from) => {
  // 1. 关闭加载状态
  console.log('路由跳转完成，关闭 loading');
  // 2. 页面埋点
  console.log('访问页面：', to.path);
  // 3. 滚动到顶部
  window.scrollTo(0, 0);
});
```

## 三、路由独享守卫：`beforeEnter`

#### 触发时机

仅在**进入特定路由前**触发，作用范围仅为该路由。
#### 配置方式

在路由规则的配置对象中添加 `beforeEnter` 属性。
#### 核心作用

- 特定路由的权限控制（如仅管理员可访问的页面）；
- 特定路由的数据预加载。
#### 示例

```javascript
// src/router/index.js
const routes = [
  {
    path: '/admin',
    name: 'Admin',
    component: Admin,
    meta: { title: '管理后台', requiresAdmin: true },
    // 路由独享守卫
    beforeEnter: (to, from) => {
      const isAdmin = localStorage.getItem('role') === 'admin';
      if (!isAdmin) {
        // 非管理员，跳转到首页
        return { name: 'Home' };
      }
      return true;
    }
  }
];
```

## 四、组件内守卫

组件内守卫在**组件内部**配置，仅作用于该组件对应的路由，分为三个守卫：

### 1. `beforeRouteEnter`

#### 触发时机

在**进入该组件对应的路由前**触发，此时**组件实例还未创建**，因此**无法访问 `this`**。

#### 核心作用

- 组件数据预加载（在进入页面前就获取数据，提升体验）；
- 进入页面的权限验证。
#### 特殊用法

**通过 `next(vm => {})` 访问组件实例**

因为此时组件还未创建，无法直接用 `this`，但可以通过 `next()` 的回调函数获取组件实例（Vue Router 4 中也可以用 `return` + 回调）。
#### 示例：数据预加载

```vue
<script setup>
import { ref, onMounted } from 'vue';
const userData = ref(null);

// 组件内守卫：beforeRouteEnter
// 注意：setup 语法糖中，组件内守卫需要从 vue-router 引入
import { onBeforeRouteEnter } from 'vue-router';
onBeforeRouteEnter(async (to, from, next) => {
  // 预加载数据：在进入页面前就获取
  const res = await fetchUser(to.params.id);
  // 通过 next 的回调访问组件实例
  next(vm => {
    vm.userData = res.data;
  });
});
</script>
```

### 2. `beforeRouteUpdate`

#### 触发时机

在**当前路由的参数变化，但组件被复用时**触发（如从 `/user/123` 跳转到 `/user/456`，UserDetail 组件会被复用，不会重新创建）。

#### 核心作用

- 路由参数变化时，重新获取数据（如用户 ID 变化时，重新获取用户详情）。
#### 示例

```vue
<script setup>
import { ref } from 'vue';
import { onBeforeRouteUpdate } from 'vue-router';
const userData = ref(null);

// 路由参数变化时触发
onBeforeRouteUpdate(async (to, from) => {
  if (to.params.id !== from.params.id) {
    // ID 变化，重新获取数据
    userData.value = await fetchUser(to.params.id);
  }
});

// 也可以用watch监听id
</script>
```

### 3. `beforeRouteLeave`

#### 触发时机

在**离开该组件对应的路由前**触发。

#### 核心作用

- 离开页面确认（如表单未保存时，提示用户是否确认离开）；
- 清理组件数据（如清除定时器、取消未完成的请求）。

#### 示例：表单未保存确认

```vue
<script setup>
import { ref } from 'vue';
import { onBeforeRouteLeave } from 'vue-router';
const formData = ref({ name: '' });
const isSaved = ref(false);

// 离开页面前触发
onBeforeRouteLeave((to, from) => {
  if (!isSaved.value && formData.value.name) {
    // 表单未保存，提示用户
    const confirmLeave = window.confirm('表单未保存，确认离开吗？');
    if (!confirmLeave) {
      // 用户取消，停留在当前页面
      return false;
    }
  }
  // 确认离开，继续跳转
  return true;
});
</script>
```

## 五、路由守卫的完整执行顺序

当一个路由跳转发生时，所有守卫的执行顺序如下（从先到后）：

1. 触发 `beforeRouteLeave`（离开的组件内守卫）；
2. 触发全局前置守卫 `beforeEach`；
3. 触发 `beforeRouteUpdate`（复用的组件内守卫）；
4. 触发路由独享守卫 `beforeEnter`；
5. 解析异步路由组件；
6. 触发 `beforeRouteEnter`（进入的组件内守卫）；
7. 触发全局解析守卫 `beforeResolve`；
8. 路由跳转确认；
9. 触发全局后置钩子 `afterEach`；
10. 组件挂载。

## 六、路由守卫注意事项

1. **`next()` 的使用（Vue Router 4 推荐用 `return`）**：
    
    - Vue Router 4 中，`next()` 仍可使用，但推荐用 `return` 替代，代码更简洁；
    - 如果使用 `next()`，必须确保**只调用一次**，否则会导致逻辑错误。
    
2. **`beforeRouteEnter` 中无法访问 `this`**：
    
    - 因为此时组件实例还未创建，需要通过 `next(vm => {})` 的回调访问实例。
    
3. **异步守卫的处理**：
    
    - 如果守卫中有异步操作（如请求接口），可以用 `async/await`，确保异步操作完成后再继续跳转。
    
4. **避免死循环**：
    
    - 重定向时要注意条件判断，避免 “跳转到 A → 守卫拦截跳转到 B → 守卫又拦截跳转到 A” 的死循环。
## 总结

- **核心作用**：在路由跳转的不同阶段插入自定义逻辑，控制跳转流程；
- **三大分类**：全局守卫（所有路由）、路由独享守卫（特定路由）、组件内守卫（特定组件）；
- **常见场景**：权限控制、页面标题设置、数据预加载、离开确认、埋点。
