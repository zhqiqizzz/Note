路由跳转拦截是路由守卫的核心应用场景，指在路由跳转过程中，通过守卫**中断当前跳转**或**修改跳转方向**，实现权限控制、离开确认、重复跳转防护等功能，是单页应用（SPA）路由安全管理的关键。

## 一、核心拦截控制方式（Vue Router 4）

Vue Router 4 推荐用 `return` 语句控制跳转流程，替代旧版的 `next()`，代码更简洁直观：

|控制操作|`return` 写法|说明|
|---|---|---|
|**继续跳转**|`return true` 或不返回|正常进入目标路由|
|**中断跳转**|`return false`|取消跳转，停留在当前页面|
|**重定向跳转**|`return '/login'` 或 `return { name: 'Login' }`|中断当前跳转，改为跳转到指定路由|

## 二、常见拦截场景及实现

> 补充： 为 [路由守卫](路由守卫.md) 相关知识

### 1. 权限拦截：未登录用户禁止访问私密页面

这是最常用的拦截场景，用**全局前置守卫 `beforeEach`** 实现，作用于所有路由。
#### 核心逻辑

- 检查目标路由是否需要登录（通过 `meta.requiresAuth` 标记）；
- 未登录用户跳转到登录页，并携带当前路径（登录后可跳转回来）；
- 已登录用户禁止重复访问登录页。
#### 代码示例

```javascript
// src/router/index.js
router.beforeEach((to, from) => {
  // 模拟登录状态：从 localStorage 读取 token
  const isLoggedIn = localStorage.getItem('token');

  // 场景1：未登录用户访问需要登录的页面 → 跳登录页
  if (to.meta.requiresAuth && !isLoggedIn) {
    // 携带当前路径，登录后可直接跳转回来
    return { name: 'Login', query: { redirect: to.fullPath } };
  }

  // 场景2：已登录用户访问登录页 → 跳首页
  if (to.name === 'Login' && isLoggedIn) {
    return { name: 'Home' };
  }

  // 其他情况：继续跳转
  return true;
});
```

#### 路由配置标记需要登录的页面

```javascript
const routes = [
  {
    path: '/user',
    name: 'UserCenter',
    component: UserCenter,
    meta: { requiresAuth: true } // 标记需要登录
  },
  {
    path: '/login',
    name: 'Login',
    component: Login,
    meta: { requiresAuth: false }
  }
];
```

### 2. 离开页面拦截：表单未保存提示用户

用**组件内守卫 `beforeRouteLeave`** 实现，仅作用于当前组件，防止用户误操作丢失未保存的表单数据。
#### 核心逻辑

- 检查表单是否已保存；
- 未保存时弹出确认框，用户取消则停留在当前页面，确认则继续跳转。
#### 代码示例

```vue
<script setup>
import { ref } from 'vue';
import { onBeforeRouteLeave } from 'vue-router';

// 表单数据
const formData = ref({
  name: '',
  content: ''
});
// 是否已保存
const isSaved = ref(false);

// 保存表单
const saveForm = () => {
  // 模拟保存接口
  console.log('保存表单:', formData.value);
  isSaved.value = true;
};

// 离开页面前拦截
onBeforeRouteLeave((to, from) => {
  // 表单未保存且有内容 → 提示用户
  if (!isSaved.value && (formData.value.name || formData.value.content)) {
    const confirmLeave = window.confirm('表单未保存，确认离开吗？');
    if (!confirmLeave) {
      // 用户取消 → 中断跳转，停留在当前页面
      return false;
    }
  }
  // 确认离开 → 继续跳转
  return true;
});
</script>
```

### 3. 特定路由拦截：仅管理员可访问管理后台

用**路由独享守卫 `beforeEnter`** 实现，仅作用于特定路由（如管理后台），避免在全局守卫中写过多条件判断。
#### 核心逻辑

- 检查用户角色是否为管理员；
- 非管理员用户跳转到首页。
#### 代码示例

```javascript
// src/router/index.js
const routes = [
  {
    path: '/admin',
    name: 'Admin',
    component: Admin,
    meta: { title: '管理后台' },
    // 路由独享守卫：仅作用于当前路由
    beforeEnter: (to, from) => {
      // 模拟用户角色
      const userRole = localStorage.getItem('role');
      if (userRole !== 'admin') {
        // 非管理员 → 跳首页
        return { name: 'Home' };
      }
      // 管理员 → 继续跳转
      return true;
    }
  }
];
```

### 4. 重复跳转拦截：防止用户快速重复点击

用**全局前置守卫**实现，避免因用户快速点击导致的多次跳转或接口重复请求。
#### 核心逻辑

- 记录上一次跳转的时间；
- 两次跳转间隔过短（如 300ms）则中断当前跳转。
#### 代码示例

```javascript
// src/router/index.js
let lastJumpTime = 0; // 记录上一次跳转时间

router.beforeEach((to, from) => {
  const now = Date.now();
  const jumpInterval = now - lastJumpTime;

  // 两次跳转间隔小于 300ms → 中断跳转
  if (jumpInterval < 300) {
    return false;
  }

  // 更新上一次跳转时间
  lastJumpTime = now;
  return true;
});
```

## 三、进阶：异步拦截（请求接口判断权限）

如果权限判断需要请求后端接口（如验证 token 是否有效），可以用 `async/await` 实现异步拦截。

#### 代码示例

```javascript
// src/router/index.js
router.beforeEach(async (to, from) => {
  // 仅对需要登录的页面做异步验证
  if (to.meta.requiresAuth) {
    try {
      // 异步请求后端验证 token
      const res = await checkTokenValid();
      if (!res.data.valid) {
        // token 无效 → 清除本地状态，跳登录页
        localStorage.removeItem('token');
        return { name: 'Login' };
      }
    } catch (error) {
      // 请求失败 → 跳登录页
      return { name: 'Login' };
    }
  }
  return true;
});
```

## 四、注意事项

### 1. 避免死循环

重定向时要注意条件判断，避免 “跳转到 A → 拦截跳转到 B → 又拦截跳转到 A” 的死循环。

```javascript
// 错误示例：死循环
router.beforeEach((to, from) => {
  const isLoggedIn = localStorage.getItem('token');
  if (!isLoggedIn) {
    return { name: 'Login' }; // 跳登录页，但登录页也会触发守卫，再次判断未登录，继续跳登录页
  }
  return true;
});

// 正确示例：排除登录页
router.beforeEach((to, from) => {
  const isLoggedIn = localStorage.getItem('token');
  // 关键：如果目标是登录页，直接放行
  if (to.name === 'Login') {
    return true;
  }
  if (!isLoggedIn) {
    return { name: 'Login' };
  }
  return true;
});
```

### 2. 合理使用 `return`

Vue Router 4 中推荐用 `return` 替代 `next()`，如果使用 `next()`，必须确保**只调用一次**，否则会导致逻辑错误。

### 3. 拦截提示友好化

拦截时尽量给用户明确的提示（如 “未登录，请先登录”、“表单未保存”），避免用户困惑。

## 总结

- **核心原理**：通过路由守卫在跳转过程中中断或修改跳转方向；
- **控制方式**：`return true` 继续、`return false` 中断、`return 路径` 重定向；
- **常见场景**：权限拦截、离开确认、特定路由拦截、重复跳转防护；
- **注意事项**：避免死循环、合理使用 `return`、提示友好化。
