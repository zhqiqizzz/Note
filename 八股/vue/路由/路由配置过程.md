Vue Router 4 是 Vue 3 的官方路由库，通过监听 URL 变化，在不刷新页面的情况下切换组件，实现单页应用（SPA）的 “页面跳转” 效果。下面详细介绍从安装到使用的完整流程。

## 一、前置基础：路由模式选择

在配置前，先明确两种核心路由模式：

|模式|URL 格式|原理|服务器要求|适用场景|
|---|---|---|---|---|
|**History 模式**|`http://example.com/user/123`|利用 HTML5 `history.pushState`/`replaceState` API|需要服务器配置（所有路由指向 `index.html`，避免刷新 404）|生产环境主流，URL 美观，利于 SEO|
|**Hash 模式**|`http://example.com/#/user/123`|利用 URL 的 `hash`（`#` 后部分）变化，不向服务器发请求|无需服务器配置|开发环境、简单项目，兼容性好|

## 二、完整配置步骤

### 1. 安装依赖

```bash
# npm
npm install vue-router@4
# yarn
yarn add vue-router@4
# pnpm
pnpm add vue-router@4
```

### 2. 创建路由配置文件

在项目中新建 `src/router/index.js`（或 `.ts`），用于定义路由规则和创建路由实例。
#### 完整配置示例

```javascript
// src/router/index.js
import { createRouter, createWebHistory, createWebHashHistory } from 'vue-router';
// 引入页面组件（建议用路由懒加载，优化首屏性能）
const Home = () => import('../views/Home.vue');
const UserList = () => import('../views/UserList.vue');
const UserDetail = () => import('../views/UserDetail.vue');
const UserCenter = () => import('../views/UserCenter.vue');
const UserProfile = () => import('../views/UserProfile.vue');
const UserOrders = () => import('../views/UserOrders.vue');
const Login = () => import('../views/Login.vue');
const NotFound = () => import('../views/NotFound.vue');

// 1. 定义路由规则：建立 URL 路径 → 组件的映射
const routes = [
  // 首页
  {
    path: '/',
    name: 'Home',
    component: Home,
    meta: { title: '首页', requiresAuth: false } // 自定义元信息
  },
  // 用户列表页
  {
    path: '/users',
    name: 'UserList',
    component: UserList,
    meta: { title: '用户列表', requiresAuth: false }
  },
  // 用户详情页：动态路由参数 :id
  {
    path: '/users/:id',
    name: 'UserDetail',
    component: UserDetail,
    meta: { title: '用户详情', requiresAuth: false }
  },
  // 用户中心：嵌套路由示例
  {
    path: '/user',
    name: 'UserCenter',
    component: UserCenter,
    meta: { title: '用户中心', requiresAuth: true }, // 需要登录
    // 嵌套子路由
    children: [
      {
        path: 'profile', // 子路由 path 不需要 /，完整路径 /user/profile
        name: 'UserProfile',
        component: UserProfile,
        meta: { title: '个人信息' }
      },
      {
        path: 'orders',
        name: 'UserOrders',
        component: UserOrders,
        meta: { title: '订单管理' }
      }
    ]
  },
  // 登录页
  {
    path: '/login',
    name: 'Login',
    component: Login,
    meta: { title: '登录' }
  },
  // 重定向：访问 /home 自动跳转到 /
  {
    path: '/home',
    redirect: '/'
  },
  // 404 页面：匹配所有未定义的路由（必须放在最后）
  {
    path: '/:pathMatch(.*)*',
    name: 'NotFound',
    component: NotFound,
    meta: { title: '页面不存在' }
  }
];

// 2. 创建路由实例
const router = createRouter({
  // 路由模式：History 模式（生产环境推荐）
  history: createWebHistory(import.meta.env.BASE_URL),
  // Hash 模式（开发环境可选）：
  // history: createWebHashHistory(),
  routes, // 路由规则
  // 可选：自定义滚动行为（跳转路由时滚动到顶部）
  scrollBehavior(to, from, savedPosition) {
    if (savedPosition) {
      return savedPosition;
    } else {
      return { top: 0 };
    }
  }
});

// 3. 全局前置守卫：权限控制、页面标题设置
router.beforeEach((to, from, next) => {
  // 设置页面标题
  document.title = to.meta.title || '默认标题';

  // 权限控制：检查是否需要登录
  const isLoggedIn = localStorage.getItem('token'); // 模拟登录状态
  if (to.meta.requiresAuth && !isLoggedIn) {
    // 未登录，跳转到登录页，并携带当前路径（登录后可跳转回来）
    next({ name: 'Login', query: { redirect: to.fullPath } });
  } else {
    next(); // 必须调用 next() 才能继续跳转
  }
});

// 4. 全局后置钩子：可选，用于埋点、加载状态关闭等
router.afterEach((to, from) => {
  console.log('路由跳转完成：', from.path, '→', to.path);
});

// 5. 导出路由实例
export default router;
```

### 3. 挂载路由到 Vue 应用

在 `src/main.js` 中引入路由实例，并通过 `app.use(router)` 挂载。

```javascript
// src/main.js
import { createApp } from 'vue';
import App from './App.vue';
import router from './router'; // 引入路由实例

const app = createApp(App);

app.use(router); // 挂载路由

app.mount('#app');
```

### 4. 配置路由出口和导航

在 `src/App.vue` 中配置：

- `<router-view />`：路由出口，匹配到的组件会渲染在这里；
- `<router-link />`：路由导航组件，替代 `<a>` 标签，实现不刷新跳转。

```vue
<!-- src/App.vue -->
<template>
  <div id="app">
    <!-- 导航栏 -->
    <nav>
      <!-- router-link：to 属性可以是字符串或对象 -->
      <router-link to="/">首页</router-link> |
      <router-link :to="{ name: 'UserList' }">用户列表</router-link> |
      <router-link :to="{ name: 'UserCenter' }">用户中心</router-link>
    </nav>
    <!-- 路由出口：组件渲染位置 -->
    <router-view />
  </div>
</template>

<style>
/* 可选：高亮当前路由 */
.router-link-active {
  color: #42b983;
  font-weight: bold;
}
</style>
```

### 5. 嵌套路由的子出口配置

如果有嵌套路由（如用户中心），需要在父组件中配置子路由出口 `<router-view />`。

```vue
<!-- src/views/UserCenter.vue -->
<template>
  <div>
    <h2>用户中心</h2>
    <!-- 子导航 -->
    <nav>
      <router-link :to="{ name: 'UserProfile' }">个人信息</router-link> |
      <router-link :to="{ name: 'UserOrders' }">订单管理</router-link>
    </nav>
    <!-- 子路由出口：子组件渲染位置 -->
    <router-view />
  </div>
</template>
```

## 三、路由跳转与接收参数

### 1. 编程式跳转（在 JS 中跳转）

```vue
<script setup>
import { useRouter } from 'vue-router';
const router = useRouter();

// 方式1：通过 path 跳转（简单场景）
const goHome = () => {
  router.push('/');
};

// 方式2：通过 name + params（推荐，动态路由）
const goToDetail = (userId) => {
  router.push({ name: 'UserDetail', params: { id: userId } });
};

// 方式3：通过 name + query（查询参数）
const goToUserList = (page) => {
  router.push({ name: 'UserList', query: { page: page, keyword: 'test' } });
};

// 方式4：通过 name + state（路由状态，不显示在 URL）
const goToDetailWithState = (user) => {
  router.push({ 
    name: 'UserDetail', 
    params: { id: user.id },
    state: { name: user.name, hobbies: user.hobbies }
  });
};

// 方式5：替换当前路由（不保留历史记录）
const replaceToHome = () => {
  router.replace('/');
};

// 方式6：前进/后退
const goBack = () => {
  router.back(); // 后退一页
  // router.go(-1); // 等价于 back()
  // router.go(1); // 前进一页
};
</script>
```

### 2. 接收参数


```vue
<script setup>
import { useRoute } from 'vue-router';
const route = useRoute();

// 1. 接收动态路由参数（注意是字符串，需手动转类型）
const userId = Number(route.params.id);

// 2. 接收查询参数
const page = Number(route.query.page);
const keyword = route.query.keyword;

// 3. 接收路由状态
const state = route.state || {};
const userName = state.name;

// 4. 接收路由元信息
const title = route.meta.title;
const requiresAuth = route.meta.requiresAuth;
</script>
```

## 四、进阶优化：路由懒加载

在路由配置中，使用 `() => import('./Component.vue')` 替代直接引入组件，可以实现**路由懒加载**：

- 只有访问该路由时，才会加载对应的组件代码；
- 大幅减少首屏加载的 JS 体积，提升首屏性能。

```javascript
// 懒加载写法（推荐）
const Home = () => import('../views/Home.vue');

// 对比：直接引入写法（不推荐，所有组件代码打包在一个文件中）
// import Home from '../views/Home.vue';
```

## 五、配置核心流程总结

Vue Router 4 的完整配置流程可归纳为：

1. **安装依赖**：引入 `vue-router@4`；
2. **定义规则**：在 `src/router/index.js` 中建立 “URL → 组件” 映射；
3. **创建实例**：选择路由模式，创建路由实例，配置守卫；
4. **挂载应用**：在 `main.js` 中通过 `app.use(router)` 挂载；
5. **配置出口**：在 `App.vue` 中用 `<router-view />` 指定渲染位置；
6. **配置导航**：用 `<router-link />` 实现声明式跳转；
7. **编程跳转**：用 `useRouter()` 实现 JS 中跳转，用 `useRoute()` 接收参数。
