热更新（Hot Module Replacement，简称 **HMR**）是一种**在应用运行时局部更新代码、无需刷新整个页面**的技术，核心价值是**保留应用状态 + 提升开发效率**。

### 一、核心概念

- **作用**：修改代码后，仅替换发生变化的 “模块”（如组件、工具函数、样式文件），而不重新加载整个页面。
- **关键优势**：
    
    - 保留应用状态（如表单输入、滚动位置、组件内部状态）；
    - 毫秒级反馈，修改后立即看到效果，大幅提升开发体验。

### 二、前端主流热更新方案

目前前端开发中，最常用的是 **Webpack HMR** 和 **Vite 热更新**，两者原理不同，但目标一致。

#### 1. Webpack HMR（经典方案）

Webpack 是最早普及 HMR 的构建工具之一，通过 `webpack-dev-server` 实现。

**核心原理**：

1. **建立连接**：开发服务器与浏览器建立 WebSocket 长连接；
2. **监听变化**：文件修改后，Webpack 重新编译变化的模块，生成 “更新补丁”；
3. **推送更新**：通过 WebSocket 向浏览器发送更新信号（包含变化模块的 hash）；
4. **局部替换**：浏览器中的 `HMR Runtime` 接收信号，请求并替换旧模块，触发 `accept` 回调更新应用。

**示例（Vue + Webpack）**：

```javascript
// webpack.config.js 中启用 HMR
module.exports = {
  devServer: {
    hot: true, // 开启热更新
  },
};

// 组件中手动接受更新（Vue 等框架已内置，无需手写）
if (module.hot) {
  module.hot.accept('./Component.vue', () => {
    // 替换组件后的逻辑
  });
}
```

#### 2. Vite 热更新（现代方案）

Vite 基于 **ES Modules (ESM)** 实现，无需打包，热更新速度比 Webpack 更快（尤其大型项目）。

**核心原理**：

1. **原生 ESM 加载**：开发服务器直接向浏览器提供 ES 模块，无需打包成 bundle；
2. **依赖预构建**：首次启动时预构建第三方依赖（如 Vue、Lodash），缓存到本地；
3. **精准更新**：文件修改时，Vite 仅重新编译变化的模块，通过 WebSocket 通知浏览器；
4. **直接替换**：浏览器直接请求新的 ES 模块，替换旧模块，无需处理 bundle 级别的更新。

**优势**：

- 速度与项目规模无关（只更新变化的模块）；
- 框架级优化（如 Vue SFC 热更新、React Fast Refresh）。

### 三、框架层面的热更新优化

现代前端框架（Vue、React）已深度集成 HMR，实现**组件级热更新**，甚至保留组件内部状态：

#### 1. Vue SFC 热更新

- 修改 `<template>` 或 `<script>`：局部更新组件，保留组件状态（如 `data`、`props`）；
- 修改 `<style>`：通过 `vite-plugin-vue` 或 `vue-loader` 直接替换样式，无需重新渲染组件。

#### 2. React Fast Refresh

- React 官方提供的热更新方案，替代旧的 React Hot Loader；
- 支持**函数组件和 Hooks 状态保留**，修改组件后仅更新逻辑，不丢失 `useState`、`useRef` 的值。

### 四、热更新 vs Live Reload

| 特性   | 热更新 (HMR)    | Live Reload (自动刷新) |
| ---- | ------------ | ------------------ |
| 更新范围 | 局部替换变化的模块    | 刷新整个页面             |
| 状态保留 | ✅ 保留（如表单、滚动） | ❌ 丢失               |
| 反馈速度 | 毫秒级          | 秒级（需重新加载页面）        |
| 适用场景 | 日常开发         | 简单项目或 HMR 不可用时     |

### 五、其他场景的热更新

除了前端开发，热更新也广泛应用于其他领域：

- **移动端**：如 React Native 的 CodePush、Flutter 的热重载（Hot Reload），可在不发版的情况下更新 App；
- **后端**：如 Node.js 的 `nodemon`（虽非严格 HMR，但可自动重启服务）、Java 的 JRebel。

### 总结

热更新是现代前端开发的**标配能力**，核心价值是**提升开发体验**：

- 小型项目：Webpack HMR 或 Vite 均可；
- 中大型项目：优先 Vite（热更新速度更快）；
- 框架开发：直接使用 Vue/React 内置的 HMR 方案，无需手动配置。
