在 Vue.js 中，**父子组件通信**是最基础、最常用的组件交互方式，核心遵循 “**单向数据流**” 原则：父组件通过 `props` 向子组件传数据，子组件通过 `emit` 向父组件发事件。以下是详细介绍：

### 一、父组件 → 子组件：Props（属性传递）

父组件通过 **自定义属性** 向子组件传递数据，子组件通过 `defineProps` 声明并接收。
#### 1. 父组件传值

```vue
<!-- 父组件：Parent.vue -->
<template>
  <div>
    <h2>父组件</h2>
    <!-- 通过 v-bind 传递数据（静态值直接写，动态值用 : 绑定） -->
    <ChildComponent 
      title="欢迎使用 Vue" 
      :count="parentCount" 
      :user-info="{ name: '张三', age: 25 }"
    />
  </div>
</template>

<script setup>
import { ref } from 'vue'
import ChildComponent from './ChildComponent.vue'

const parentCount = ref(10)
</script>
```

#### 2. 子组件接收（Props 声明）

```vue
<!-- 子组件：ChildComponent.vue -->
<template>
  <div class="child">
    <h3>子组件</h3>
    <p>标题：{{ title }}</p>
    <p>计数：{{ count }}</p>
    <p>用户：{{ userInfo?.name }}</p>
  </div>
</template>

<script setup>
// 1. 声明 props（数组形式，简单场景用）
// defineProps(['title', 'count', 'userInfo'])

// 2. 声明 props（对象形式，推荐：可指定类型、默认值、必填校验）
const props = defineProps({
  title: String,
  count: {
    type: Number,
    default: 0,
    required: true
  },
  userInfo: {
    type: Object,
    default: () => ({}) // 对象/数组的默认值需用函数返回
  }
})

// 可通过 props.xxx 访问 props 值
console.log(props.title)
</script>
```

⚠️ **重要**：Props 是**只读**的，子组件**不能直接修改** props 值（否则会报错），需通过 `emit` 通知父组件修改。

### 二、子组件 → 父组件：Emit（事件触发）

子组件通过 `defineEmits` 声明事件，用 `$emit` 触发事件并传递数据，父组件通过 `v-on`（或 `@`）监听事件。
#### 1. 子组件触发事件

```vue
<!-- 子组件：ChildComponent.vue -->
<template>
  <div class="child">
    <h3>子组件</h3>
    <button @click="handleIncrement">点击 +1</button>
    <button @click="handleSendMsg">给父组件发消息</button>
  </div>
</template>

<script setup>
// 声明要触发的事件
const emit = defineEmits(['increment', 'send-message'])

const handleIncrement = () => {
  emit('increment') // 触发事件，不传参
}

const handleSendMsg = () => {
  emit('send-message', '你好，父组件！') // 触发事件，传递数据
}
</script>
```

#### 2. 父组件监听事件

```vue
<!-- 父组件：Parent.vue -->
<template>
  <div>
    <h2>父组件</h2>
    <p>计数：{{ count }}</p>
    <p>收到的消息：{{ message }}</p>

    <!-- 监听子组件事件 -->
    <ChildComponent 
      @increment="count++" 
      @send-message="handleReceiveMsg"
    />
  </div>
</template>

<script setup>
import { ref } from 'vue'
import ChildComponent from './ChildComponent.vue'

const count = ref(0)
const message = ref('')

// 接收子组件传递的数据
const handleReceiveMsg = (msg) => {
  message.value = msg
}
</script>
```

### 三、进阶：父子组件双向绑定（v-model）

Vue 3 中，`v-model` 是 `props` + `emit` 的**语法糖**，可简化 “父传子、子改父” 的双向绑定场景。

#### 1. 父组件使用 `v-model`

```vue
<!-- 父组件 -->
<template>
  <div>
    <h2>父组件</h2>
    <p>输入内容：{{ inputValue }}</p>
    <!-- v-model 等价于 :modelValue="inputValue" + @update:modelValue="inputValue = $event" -->
    <ChildComponent v-model="inputValue" />
  </div>
</template>

<script setup>
import { ref } from 'vue'
import ChildComponent from './ChildComponent.vue'
const inputValue = ref('')
</script>
```

#### 2. 子组件实现双向绑定

```vue
<!-- 子组件 -->
<template>
  <div class="child">
    <h3>子组件</h3>
    <input 
      :value="modelValue" 
      @input="handleInput"
      placeholder="请输入内容"
    />
  </div>
</template>

<script setup>
// 1. 接收 props（默认名是 modelValue）
const props = defineProps(['modelValue'])
// 2. 声明事件（默认名是 update:modelValue）
const emit = defineEmits(['update:modelValue'])

const handleInput = (e) => {
  emit('update:modelValue', e.target.value) // 触发更新
}
</script>
```

#### 自定义 `v-model` 名称

若需绑定多个值，可给 `v-model` 指定名称：

```vue
<!-- 父组件 -->
<ChildComponent v-model:title="pageTitle" v-model:content="pageContent" />

<!-- 子组件 -->
<script setup>
const props = defineProps(['title', 'content'])
const emit = defineEmits(['update:title', 'update:content'])
</script>
```

### 四、补充：父组件访问子组件实例（$refs）

父组件可通过 `ref` 绑定子组件，直接访问子组件的 `data`、`methods`（**仅用于特殊场景，不推荐滥用**）。

```vue
<!-- 父组件 -->
<template>
  <div>
    <ChildComponent ref="childRef" />
    <button @click="callChildMethod">调用子组件方法</button>
  </div>
</template>

<script setup>
import { ref, onMounted } from 'vue'
import ChildComponent from './ChildComponent.vue'

const childRef = ref(null) // 定义 ref，与模板中的 ref="childRef" 对应

onMounted(() => {
  console.log(childRef.value.someData) // 访问子组件 data
})

const callChildMethod = () => {
  childRef.value.someMethod() // 调用子组件方法
}
</script>
```

### 五、通信方式总结

|场景|推荐方式|说明|
|---|---|---|
|父组件 → 子组件传数据|Props|单向数据流，子组件只读。|
|子组件 → 父组件发通知|Emit|子组件触发事件，父组件监听处理。|
|父子组件双向绑定|v-model|Props + Emit 的语法糖，简化代码。|
|父组件直接操作子组件|$refs|仅用于特殊场景（如调用子组件方法、获取子组件 DOM），不推荐滥用。|
