在 Vue 组件树中，**跨层通信**指的是**非直接父子关系**（如祖孙、兄弟、无嵌套组件）之间的数据传递或事件触发。以下是 Vue 3 中最常用的跨层通信方案，按场景分类介绍：

### 一、Provide / Inject（依赖注入）

- **功能**：祖先组件 “提供”（provide）数据，后代组件 “注入”（inject）使用，**跳过中间层级**。
- **适用场景**：祖先向深层后代传递静态 / 响应式数据（如主题、全局配置）。
- **示例**：
```javascript
    // 祖先组件（如 App.vue）
    <script setup>
    import { provide, ref } from 'vue'
    const theme = ref('dark') // 响应式数据
    provide('app-theme', theme) // 提供数据（key, value）
    </script>
    
    // 深层后代组件
    <script setup>
    import { inject } from 'vue'
    const theme = inject('app-theme') // 注入数据
    </script>
```

- **优点**：无需层层传递 props，代码简洁。
- **注意**：若需修改数据，建议在 provide 组件中定义方法并一起 provide，避免直接在后代修改。

### 二、Pinia（全局状态管理）

- **功能**：通过**集中式 Store** 管理全局状态，所有组件可平等读写。
- **适用场景**：复杂应用（多组件共享状态、异步数据管理、跨模块通信）。
- **核心优势**：
    
    - 单例 Store：确保状态全局唯一。
    - 响应式自动更新：State 变化时所有使用组件自动刷新。
    - 去掉 Mutations：直接在 Actions 中修改 State，代码更简洁。
### 三、事件总线（Mitt）

- **功能**：通过第三方库 `mitt` 实现**任意组件间的事件触发与监听**。
- **适用场景**：兄弟组件、无嵌套组件间的一次性事件通信（如弹窗开关、消息通知）。
- **用法**：
1. 安装：`npm install mitt`
2. 创建事件总线：
```js
// src/utils/eventBus.js
import mitt from 'mitt'
export default mitt()
```

3. 组件中使用：
```javascript
// 组件 A（发送事件）
import eventBus from '@/utils/eventBus'
eventBus.emit('user-updated', { name: '张三' })

// 组件 B（监听事件）
import eventBus from '@/utils/eventBus'
eventBus.on('user-updated', (data) => {
  console.log(data.name) // 张三
})
```

- **注意**：组件销毁时需 `off` 取消监听，避免内存泄漏。

### 四、Props / Emit + $attrs（透传）

- **功能**：通过 props 层层向下传递，或用 `$attrs` 透传未声明的属性 / 事件。
- **适用场景**：仅跨 1-2 层的简单数据传递（不建议多层，会导致 “props drilling”）。
- **示例**（$attrs 透传）：
```vue
    <!-- 父组件 -->
    <ChildComponent msg="hello" @click="handleClick" />
    
    <!-- 子组件（中间层） -->
    <template>
      <!-- 透传所有未声明的 attrs 给孙组件 -->
      <GrandChildComponent v-bind="$attrs" />
    </template>
```
### 五、方案选择建议

| 场景           | 推荐方案                  |
| ------------ | --------------------- |
| 祖先→深层后代传数据   | Provide / Inject      |
| 多组件共享全局状态    | Pinia                 |
| 兄弟 / 任意组件传事件 | Mitt（事件总线）            |
| 简单跨 1-2 层传数据 | Props / Emit + $attrs |
