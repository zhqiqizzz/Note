在 Vue.js 中，**组件缓存**是通过内置组件 `<keep-alive>` 实现的，它能将不活动的组件实例**保留在内存中**（而非销毁重建），从而**保留组件状态、避免重复渲染、提升性能**。

### 一、核心作用

- **保留状态**：比如表单输入内容、列表滚动位置、标签页选中状态等，切换后再回来不会丢失。
- **性能优化**：避免组件重复执行 `created`、`mounted` 等生命周期钩子，减少不必要的 DOM 操作和数据请求。

### 二、基本用法

`<keep-alive>` 是一个**抽象组件**（不会渲染成真实 DOM 元素），只需用它包裹需要缓存的动态组件或路由视图即可。
#### 1. 缓存动态组件

```vue
<template>
  <div>
    <button @click="currentTab = 'TabA'">Tab A</button>
    <button @click="currentTab = 'TabB'">Tab B</button>

    <!-- 用 keep-alive 包裹动态组件 -->
    <keep-alive>
      <component :is="currentTab"></component>
    </keep-alive>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import TabA from './TabA.vue'
import TabB from './TabB.vue'

const currentTab = ref('TabA')
</script>
```

#### 2. 缓存路由组件（Vue Router）

```javascript
import { createRouter, createWebHistory } from 'vue-router'
import ListPage from '@/views/ListPage.vue'
import DetailPage from '@/views/DetailPage.vue'

const routes = [
  { path: '/', redirect: '/list' },
  { path: '/list', component: ListPage, meta: { keepAlive: true } }, // 标记需要缓存
  { path: '/detail/:id', component: DetailPage }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router
```

```vue
<!-- App.vue -->
<template>
  <div>
    <router-link to="/home">首页</router-link>
    <router-link to="/list">列表页</router-link>

    <!-- 缓存路由视图 -->
    <keep-alive>
      <router-view></router-view>
    </keep-alive>
  </div>
  <div> 
    <!-- 用 keep-alive 包裹，只缓存 meta.keepAlive 为 true 的组件 --> 
    <router-view v-slot="{ Component }"> 
	  <keep-alive> 
		  <component :is="Component" v-if="$route.meta.keepAlive" /> 
	  </keep-alive> 
      <component :is="Component" v-if="!$route.meta.keepAlive" /> 
    </router-view> 
   </div>
</template>
```

### 三、关键属性

通过 `include`、`exclude`、`max` 三个属性精细控制缓存逻辑：

| 属性        | 作用                                     |
| --------- | -------------------------------------- |
| `include` | 只有**组件名匹配**的组件会被缓存（字符串、正则、数组）。         |
| `exclude` | **组件名匹配**的组件不会被缓存（优先级高于 `include`）。    |
| `max`     | 最多缓存的组件实例数（超过时用 **LRU（最近最少使用）** 策略销毁）。 |
#### 示例：

```vue
<keep-alive include="TabA,TabB" :max="5">
  <component :is="currentTab"></component>
</keep-alive>
```

⚠️ 注意：被缓存的组件**必须设置 `name` 选项**，`include`/`exclude` 才能生效：

```vue
<!-- TabA.vue -->
<script setup>
// Vue 3 <script setup> 中需显式定义 name
import { defineOptions } from 'vue'
defineOptions({ name: 'TabA' })
</script>
```

### 四、专属生命周期钩子

被 `<keep-alive>` 缓存的组件，切换时**不会重新执行 `created`/`mounted`**，而是触发以下两个钩子：

| 钩子            | 触发时机           | 典型使用场景         |
| ------------- | -------------- | -------------- |
| `activated`   | 组件被**激活**（显示）时 | 重新请求数据、恢复播放状态等 |
| `deactivated` | 组件被**停用**（隐藏）时 | 暂停定时器、保存草稿等    |
#### 示例：

```vue
<template>
  <div class="list-page">
    <h2>文章列表</h2>
    <ul>
      <li v-for="item in list" :key="item.id" @click="goDetail(item.id)">
        {{ item.title }}
      </li>
    </ul>
  </div>
</template>

<script setup>
import { ref, onActivated, onDeactivated } from 'vue'
import { useRouter } from 'vue-router'

const router = useRouter()
const list = ref([])
let scrollPosition = 0 // 用于保存滚动位置

// 模拟生成列表数据
for (let i = 1; i <= 50; i++) {
  list.value.push({ id: i, title: `这是第 ${i} 篇文章` })
}

// 进入详情页
const goDetail = (id) => {
  router.push(`/detail/${id}`)
}

// --- 关键：缓存相关生命周期 ---
// 1. 离开页面（被停用）时：保存滚动位置
onDeactivated(() => {
  scrollPosition = window.scrollY || document.documentElement.scrollTop
  console.log('离开列表页，保存滚动位置：', scrollPosition)
})

// 2. 回到页面（被激活）时：恢复滚动位置
onActivated(() => {
  window.scrollTo(0, scrollPosition)
  console.log('回到列表页，恢复滚动位置：', scrollPosition)
})
</script>

<style scoped>
li {
  padding: 15px;
  border-bottom: 1px solid #eee;
  cursor: pointer;
}
</style>
```

> 补充： 若组件没被缓存，可以用 路由跳转的钩子函数 记录滚轮位置

### 五、实现原理（核心逻辑）

Vue 内部通过以下步骤实现缓存：
#### 1. 首次渲染组件 A：创建 VNode，并存入缓存

1. 执行 `render` 函数，生成组件 A 的**虚拟 DOM（VNode）**；
2. `keep-alive` 会拦截这个 VNode，做两件事：
    
    - 把组件 A 的 VNode 存入自己的 **缓存对象（`cache`）** 里（比如用组件的 name 或 key 作为缓存的 key）；
    - 给 VNode 打个标记：“这个组件是被缓存的”；
    
3. Vue 根据 VNode 创建组件 A 的实例和真实 DOM，挂载到页面上。

#### 2. 切换到组件 B：缓存组件 A 的 VNode，创建 / 复用组件 B

1. 执行 `render` 函数，生成组件 B 的 VNode；
2. `keep-alive` 先处理旧的组件 A：
    
    - **不销毁组件 A 的实例和真实 DOM**（这是关键！）；
    - 只是把组件 A 的真实 DOM 从页面上 “移走”（隐藏起来）；
    - 触发组件 A 的 `deactivated` 钩子（失活）；
    
3. 然后处理新的组件 B：
    
    - 检查缓存里有没有组件 B 的 VNode；
    - 如果没有（第一次用），就重复步骤 1：创建 VNode、存入缓存、创建实例和真实 DOM；
    - 如果有（之前用过），就直接从缓存里拿出组件 B 的 VNode；
    
4. 把组件 B 的真实 DOM 挂载到页面上，触发 `activated` 钩子（激活）。

#### 3. 再切回组件 A：直接复用缓存的 VNode

1. 执行 `render` 函数，生成组件 A 的 VNode；
2. `keep-alive` 检查缓存：“哦，组件 A 的 VNode 我已经存过了”；
3. 直接从缓存里拿出组件 A 的 VNode；
4. 处理旧的组件 B：移走真实 DOM，触发 `deactivated`；
5. 把组件 A 之前隐藏的真实 DOM 重新 “放回到” 页面上；
6. 触发组件 A 的 `activated` 钩子；
7. **关键**：组件 A 的状态（比如输入框里的文字、计数器的数值）完全保留，不用重新初始化！

#### 原理：
1. **缓存 VNode**：将组件的虚拟 DOM（VNode）存储在内存中（`cache` 对象）。
2. **匹配规则**：根据 `include`/`exclude` 判断是否缓存当前组件。
3. **LRU 策略**：当缓存数量超过 `max` 时，销毁**最久未被访问**的组件实例。
4. **复用实例**：再次渲染时，若命中缓存则直接复用之前的组件实例，而非重新创建。
### 六、使用场景

- **表单填写**：用户填表单到一半切换页面，回来保留已填内容。
- **列表滚动**：列表页滚动到某位置，进入详情页再回来，保持滚动位置。
- **标签页切换**：频繁切换的标签页（如仪表盘），避免重复请求数据。
### 七、注意事项

- `<keep-alive>` 自身不会渲染成 DOM 元素，也不会出现在组件树中。
- 缓存的是**组件实例**，因此 `data` 会保留，但路由参数变化时需在 `activated` 中手动更新数据。
- 若需强制刷新缓存组件，可通过 `key` 绑定动态值（如路由参数），或在 `activated` 中重置数据。
