Vue 组件优化主要从**渲染性能**、**打包体积**、**代码组织**三个维度入手，结合 Vue 3 的特性（如 Composition API、虚拟 DOM 优化），可以显著提升应用的流畅度和加载速度。以下是核心优化策略：
## 一、渲染性能优化：减少不必要的更新

### 1. `v-memo`：缓存组件 / 元素的渲染结果

`v-memo` 是 Vue 3.2+ 新增的指令，用于**缓存组件或元素的渲染结果**，只有当依赖项变化时才重新渲染，适合**列表渲染**或**复杂组件**的性能优化。

#### 示例：列表渲染优化

```vue
<template>
  <div v-for="item in list" :key="item.id" v-memo="[item.isActive]">
    <!-- 只有 item.isActive 变化时，该列表项才重新渲染 -->
    <p :class="{ active: item.isActive }">{{ item.name }}</p>
  </div>
</template>
```
#### 说明：

- `v-memo="[dep1, dep2]"`：只有依赖项 `dep1`/`dep2` 变化时，才重新渲染该元素 / 组件。
- 若 `v-memo="[]"`（空数组），则只渲染一次（等价于 `v-once`）。

### 2. 避免不必要的响应式：`shallowRef`/`shallowReactive`/`markRaw`

Vue 的响应式系统默认会递归监听对象的所有属性，对于**大对象**（如第三方库实例、不需要深层响应式的数据），可以用以下 API 减少性能开销：

| API               | 作用                              | 适用场景                                            |
| ----------------- | ------------------------------- | ----------------------------------------------- |
| `shallowRef`      | 只监听 `.value` 的变化，不递归监听对象内部属性。   | 大对象、只需要顶层响应式的数据（如分页信息 `{ page: 1, size: 10 }`）。 |
| `shallowReactive` | 只监听对象的顶层属性，不递归监听深层属性。           | 同上。                                             |
| `markRaw`         | 标记一个对象为 “非响应式”，Vue 不会对其进行响应式处理。 | 第三方库实例（如 ECharts、Mapbox）、静态配置对象。                |
#### 示例：

```javascript
import { shallowRef, markRaw } from 'vue'
import ECharts from 'echarts'

// 1. shallowRef：只监听 .value 的变化，不递归监听 chart 内部
const chart = shallowRef(null)

// 2. markRaw：标记 ECharts 实例为非响应式，避免 Vue 递归监听
chart.value = markRaw(ECharts.init(document.getElementById('chart')))
```

### 3. 列表渲染优化：正确使用 `key` + 虚拟滚动

#### （1）正确使用 `key`

- **不要用 `index` 作为 `key`**：当列表插入 / 删除元素时，`index` 变化会导致 Vue 错误复用 DOM，引发性能问题或状态错乱。
- **用唯一 ID 作为 `key`**：确保 Vue 能高效复用和更新 DOM 节点。
    
```vue
    <div v-for="item in list" :key="item.id">{{ item.name }}</div>
```

#### （2）虚拟滚动（长列表优化）

当列表项超过 **1000+** 时，即使 Vue 虚拟 DOM 优化得再好，渲染大量 DOM 仍会卡顿。此时需用**虚拟滚动**：只渲染**可视区域**的列表项，大幅减少 DOM 数量。

推荐库：`vue-virtual-scroller`、`vue-virtual-scroll-list`。
#### 示例（vue-virtual-scroller）：

```vue
<template>
  <RecycleScroller
    class="scroller"
    :items="list"
    :item-size="50"
    key-field="id"
    v-slot="{ item }"
  >
    <div class="item">{{ item.name }}</div>
  </RecycleScroller>
</template>

<script setup>
import { RecycleScroller } from 'vue-virtual-scroller'
import 'vue-virtual-scroller/dist/vue-virtual-scroller.css'
</script>
```

### 4. `v-once`：渲染静态内容

对于**只需要渲染一次、后续不会更新**的静态内容，用 `v-once` 标记，Vue 会缓存其渲染结果，后续更新时跳过。

```vue
<template>
  <div v-once>
    <!-- 以下内容只渲染一次，后续数据变化也不会更新 -->
    <h1>静态标题</h1>
    <p>静态描述文本</p>
  </div>
</template>
```

## 二、打包体积优化：按需加载

### 1. 异步组件：`defineAsyncComponent`

将非首屏必需的组件（如弹窗、抽屉、二级页面）定义为**异步组件**，Vue 会在需要时才加载该组件的代码（分割成独立的 chunk），减少首屏打包体积。
#### 示例：

```vue
<template>
  <div>
    <button @click="showModal = true">打开弹窗</button>
    <!-- 只有 showModal 为 true 时，才加载 Modal 组件的代码 -->
    <Modal v-if="showModal" @close="showModal = false" />
  </div>
</template>

<script setup>
import { ref, defineAsyncComponent } from 'vue'

// 定义异步组件：动态导入 Modal.vue
const Modal = defineAsyncComponent(() => import('./Modal.vue'))

const showModal = ref(false)
</script>
```

### 2. 路由懒加载

与异步组件类似，路由组件也可以通过 `import()` 动态导入，实现**按路由分割代码**，首屏只加载当前路由的代码。
#### 示例（Vue Router 4）：

```javascript
// router/index.js
import { createRouter, createWebHistory } from 'vue-router'

const routes = [
  {
    path: '/',
    component: () => import('@/views/Home.vue') // 路由懒加载
  },
  {
    path: '/about',
    component: () => import('@/views/About.vue') // 路由懒加载
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})
```

### 3. Tree Shaking：移除未使用的代码

确保项目使用 **ES 模块**（`import`/`export`），打包工具（Webpack/Vite）会自动通过 Tree Shaking 移除未使用的代码。

- 避免使用 `require()`（CommonJS 模块无法 Tree Shaking）。
- 对于第三方库，优先选择支持 ES 模块的版本（如 `lodash-es` 替代 `lodash`）。

## 三、代码组织优化：提升可维护性与复用性

### 1. Composition API + Composables：逻辑复用

将可复用的逻辑提取到**组合式函数（Composables）** 中，替代 Vue 2 的 `mixins`（避免命名冲突、来源不明），提升代码的可维护性和复用性。
#### 示例：提取 “鼠标位置” 逻辑

```javascript
// composables/useMouse.js
import { ref, onMounted, onUnmounted } from 'vue'

export function useMouse() {
  const x = ref(0)
  const y = ref(0)

  const update = (e) => {
    x.value = e.clientX
    y.value = e.clientY
  }

  onMounted(() => window.addEventListener('mousemove', update))
  onUnmounted(() => window.removeEventListener('mousemove', update))

  return { x, y }
}
```
#### 在组件中使用：

```vue
<script setup>
import { useMouse } from '@/composables/useMouse'
const { x, y } = useMouse() // 直接复用鼠标位置逻辑
</script>
```

### 2. 避免复杂模板表达式：用 `computed`

模板中应只包含**简单的插值**，复杂逻辑应放到 `computed` 中（`computed` 有缓存，只有依赖变化时才重新计算）。
#### 反例：

```vue
<template>
  <p>{{ user.name.toUpperCase() + ' - ' + user.age + '岁' }}</p>
</template>
```
#### 正例：

```vue
<template>
  <p>{{ userInfo }}</p>
</template>

<script setup>
import { computed } from 'vue'
const userInfo = computed(() => `${user.name.toUpperCase()} - ${user.age}岁`)
</script>
```

### 3. 合理使用 `defineOptions`

在 `<script setup>` 中，通过 `defineOptions` 显式定义组件的 `name`、`inheritAttrs` 等选项，方便调试和 `<keep-alive>` 使用。

```vue
<script setup>
import { defineOptions } from 'vue'

// 定义组件 name（用于 keep-alive 的 include/exclude）
defineOptions({ name: 'UserCard' })
</script>
```

## 四、其他优化建议

1. **避免过深的组件嵌套**：过深的组件树会增加 Vue 的递归遍历开销，尽量保持组件层级扁平。
2. **合理使用 `Teleport`**：将弹窗、提示框等组件挂载到 `body` 下，避免样式污染和布局问题（如 `z-index` 层级）。
3. **使用生产环境构建**：部署时确保使用 Vue 的生产环境版本（Vite 自动处理，Webpack 需设置 `NODE_ENV=production`），会移除开发警告和调试工具，体积更小、性能更好。
