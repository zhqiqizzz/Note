Vue 组件的**生命周期**是指组件从**创建→挂载→更新→销毁**的整个过程中，Vue 自动触发的一系列**钩子函数**，让我们可以在特定阶段执行代码（如初始化数据、操作 DOM、清理定时器等）。

以下是 **Vue 3 Composition API** 的生命周期详解（主流推荐写法），并附与 Options API 的对比。

## 一、Vue 3 生命周期阶段（4 个核心阶段）

Vue 3 的生命周期钩子以 `onXxx` 形式命名，需从 `vue` 中引入，在 `setup()` 中调用。

| 阶段       | 钩子                | 触发时机                                              | 典型使用场景                                 |
| -------- | ----------------- | ------------------------------------------------- | -------------------------------------- |
| **创建阶段** | `setup()`         | 组件实例创建前执行（替代 Vue 2 的 `beforeCreate` 和 `created`）。 | 初始化响应式数据、定义函数、引入组合式函数（Composables）。    |
| **挂载阶段** | `onBeforeMount`   | 组件挂载到 DOM **之前**触发（此时 DOM 还未渲染）。                  | 极少使用（此时无法操作 DOM）。                      |
|          | `onMounted`       | 组件挂载到 DOM **之后**触发（此时 DOM 已渲染完成）。                 | 操作 DOM（如获取元素、初始化第三方库）、发起 API 请求、启动定时器。 |
| **更新阶段** | `onBeforeUpdate`  | 组件数据变化，DOM 更新 **之前**触发。                           | 获取更新前的 DOM 状态（如滚动位置）。                  |
|          | `onUpdated`       | 组件数据变化，DOM 更新 **之后**触发。                           | 操作更新后的 DOM（如重新初始化第三方库）。                |
| **卸载阶段** | `onBeforeUnmount` | 组件卸载 **之前**触发（此时组件仍可用）。                           | 清理定时器、取消事件监听、销毁第三方库实例（避免内存泄漏）。         |
|          | `onUnmounted`     | 组件卸载 **之后**触发（此时组件已从 DOM 移除）。                     | 极少使用。                                  |

## 二、`<keep-alive>` 专属生命周期

当组件被 `<keep-alive>` 缓存时，切换不会重新走 “挂载 / 卸载” 流程，而是触发以下两个钩子：

| 钩子              | 触发时机                | 典型使用场景                |
| --------------- | ------------------- | --------------------- |
| `onActivated`   | 缓存组件被**激活**（显示）时触发。 | 恢复播放状态、重新请求数据、恢复滚动位置。 |
| `onDeactivated` | 缓存组件被**停用**（隐藏）时触发。 | 暂停播放、保存草稿、暂停定时器。      |

## 三、完整示例代码（展示触发顺序）


```vue
<template>
  <div>
    <h3>生命周期示例</h3>
    <p>计数：{{ count }}</p>
    <button @click="count++">+1</button>
  </div>
</template>

<script setup>
import { ref, onBeforeMount, onMounted, onBeforeUpdate, onUpdated, onBeforeUnmount, onUnmounted } from 'vue'

const count = ref(0)

// 1. setup()：创建阶段（直接在这里写初始化逻辑）
console.log('1. setup()：组件实例创建中')

// 2. 挂载前
onBeforeMount(() => {
  console.log('2. onBeforeMount：DOM 挂载前')
})

// 3. 挂载后（最常用）
onMounted(() => {
  console.log('3. onMounted：DOM 挂载完成')
  // 示例：在这里发起 API 请求、操作 DOM
})

// 4. 更新前
onBeforeUpdate(() => {
  console.log('4. onBeforeUpdate：DOM 更新前')
})

// 5. 更新后
onUpdated(() => {
  console.log('5. onUpdated：DOM 更新完成')
})

// 6. 卸载前
onBeforeUnmount(() => {
  console.log('6. onBeforeUnmount：组件卸载前')
  // 示例：在这里清理定时器、取消事件监听
})

// 7. 卸载后
onUnmounted(() => {
  console.log('7. onUnmounted：组件卸载完成')
})
</script>
```

**控制台输出顺序**（首次加载）：

```plaintext
1. setup()：组件实例创建中
2. onBeforeMount：DOM 挂载前
3. onMounted：DOM 挂载完成
```

点击按钮更新数据时：

```plaintext
4. onBeforeUpdate：DOM 更新前
5. onUpdated：DOM 更新完成
```

## 四、Vue 3 vs Vue 2 生命周期对比

Vue 3 仍支持 Vue 2 的 Options API 写法（如 `mounted`、`beforeDestroy`），但推荐用 Composition API 的 `onXxx` 钩子。

| Vue 2 (Options API) | Vue 3 (Composition API) | 说明           |
| ------------------- | ----------------------- | ------------ |
| `beforeCreate`      | `setup()`               | 被 `setup` 替代 |
| `created`           | `setup()`               | 被 `setup` 替代 |
| `beforeMount`       | `onBeforeMount`         | 命名调整         |
| `mounted`           | `onMounted`             | 命名调整         |
| `beforeUpdate`      | `onBeforeUpdate`        | 命名调整         |
| `updated`           | `onUpdated`             | 命名调整         |
| `beforeDestroy`     | `onBeforeUnmount`       | 命名调整（更语义化）   |
| `destroyed`         | `onUnmounted`           | 命名调整（更语义化）   |
| `activated`         | `onActivated`           | 命名调整         |
| `deactivated`       | `onDeactivated`         | 命名调整         |
## 五、常见使用场景总结

|场景|推荐钩子|
|---|---|
|初始化数据、定义函数|`setup()`|
|操作 DOM、发起 API 请求|`onMounted`|
|清理定时器、取消事件监听|`onBeforeUnmount`|
|缓存组件恢复状态|`onActivated`|
|缓存组件暂停状态|`onDeactivated`|
