首屏性能优化的核心目标是：**减少首屏加载的资源体积、减少首屏请求数量、提升资源加载速度**。本方案结合 **Vite `manualChunks` 依赖拆分**、**Gzip/Brotli 压缩**、**路由懒加载**三大核心策略，形成可直接落地的完整优化方案。

## 一、策略一：路由懒加载（首屏按需加载，最直接有效）

路由懒加载是首屏优化的基础，核心逻辑是：**首屏只加载当前路由需要的组件，非首屏路由的组件拆分成独立的 chunk，用户访问对应路由时才加载**，从根源上减少首屏的资源体积。
### 1. Vue 3 + Vue Router 4 写法

```javascript
// router/index.js
import { createRouter, createWebHistory } from 'vue-router';

const routes = [
  {
    path: '/',
    name: 'Home',
    // 首屏路由：直接引入，保证首屏渲染速度
    component: () => import('@/views/Home.vue')
  },
  {
    path: '/about',
    name: 'About',
    // 非首屏路由：动态引入，自动拆分成独立 chunk
    component: () => import(/* webpackChunkName: "about" */ '@/views/About.vue')
  },
  {
    path: '/list',
    name: 'List',
    component: () => import(/* webpackChunkName: "list" */ '@/views/List.vue')
  }
];

const router = createRouter({
  history: createWebHistory(),
  routes
});

export default router;
```

### 2. 关键说明

- 首屏路由可以直接引入，也可以动态引入，Vite 会自动优化；
- `/* webpackChunkName: "xxx" */` 是魔法注释，虽然 Vite 主要用 Rollup，但这个注释依然有效，可以给拆分后的 chunk 命名，方便调试；
- 路由懒加载后，非首屏路由的组件、样式、依赖都会被拆分成独立的 `.js`/`.css` 文件，首屏不会加载。

## 二、策略二：Vite `manualChunks` 依赖精细拆分（利用浏览器缓存）

Vite 默认的拆包策略是：

- 把所有第三方依赖（`node_modules`）打包进一个 `vendor.js`；
- 把业务代码打包进一个 `index.js`。

这种策略的问题是：**只要更新了任意一个第三方依赖，整个 `vendor.js` 都会失效，用户需要重新下载所有依赖**。

通过 `manualChunks` 可以手动控制依赖的拆分规则，把不同类型的依赖拆分成独立的 chunk，最大化利用浏览器的长期缓存。

### 1. 核心拆分原则

|拆分维度|说明|示例|
|:--|:--|:--|
|**Vue 生态核心库**|最稳定、更新频率最低的依赖，单独拆分，长期缓存|`vue`、`vue-router`、`pinia`|
|**UI 组件库**|体积大、更新频率中等，单独拆分|`element-plus`、`ant-design-vue`、`vant`|
|**工具库**|按功能分组拆分，比如图表库、HTTP 库|`echarts`、`axios`、`lodash-es`|
|**业务公共模块**|多个路由复用的业务组件、工具函数，单独拆分|`@/components/common`、`@/utils`|
### 2. Vite 配置代码

```javascript
// vite.config.js
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';
import { resolve } from 'path';

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src')
    }
  },
  build: {
    rollupOptions: {
      output: {
        // 核心：manualChunks 手动拆包
        manualChunks: {
          // 1. Vue 生态核心库（最稳定，长期缓存）
          'vue-vendor': ['vue', 'vue-router', 'pinia'],
          // 2. UI 组件库（体积大，单独拆分）
          'ui-vendor': ['element-plus', '@element-plus/icons-vue'],
          // 3. 图表库（按需拆分）
          'chart-vendor': ['echarts'],
          // 4. 工具库（按需拆分）
          'utils-vendor': ['axios', 'lodash-es', 'dayjs']
        },
        // 对拆分后的 chunk 进行命名规范
        chunkFileNames: 'assets/js/[name]-[hash].js',
        entryFileNames: 'assets/js/[name]-[hash].js',
        assetFileNames: 'assets/[ext]/[name]-[hash].[ext]'
      }
    }
  }
});
```

### 3. 进阶：按模块路径自动拆分

如果依赖较多，不想手动一个个列，可以用函数形式的 `manualChunks`，按路径自动分组：

```javascript
manualChunks(id) {
  // 拆分 node_modules 里的依赖
  if (id.includes('node_modules')) {
    if (id.includes('vue') || id.includes('pinia') || id.includes('vue-router')) {
      return 'vue-vendor';
    }
    if (id.includes('element-plus') || id.includes('ant-design')) {
      return 'ui-vendor';
    }
    if (id.includes('echarts') || id.includes('three')) {
      return 'chart-vendor';
    }
    // 其他第三方依赖统一打包
    return 'vendor';
  }
  // 拆分业务公共组件
  if (id.includes('src/components/common')) {
    return 'common-components';
  }
}
```

## 三、策略三：Gzip/Brotli 压缩（减少资源体积 60%+）

文本类资源（JS、CSS、HTML）经过 Gzip 压缩后，体积可减少 60% 以上；Brotli 压缩比 Gzip 更优，体积可再减少 20% 左右。Vite 本身不直接处理压缩，需要通过插件实现。

### 1. 安装依赖

```bash
npm install vite-plugin-compression -D
```

### 2. Vite 配置代码

```javascript
// vite.config.js
import viteCompression from 'vite-plugin-compression';

export default defineConfig({
  plugins: [
    vue(),
    // Gzip 压缩
    viteCompression({
      algorithm: 'gzip', // 压缩算法
      ext: '.gz', // 压缩文件后缀
      threshold: 10240, // 只压缩大于 10KB 的文件
      deleteOriginFile: false, // 不删除源文件（兼容不支持 Gzip 的服务器）
      compressionOptions: { level: 9 } // 压缩级别，9 最高
    }),
    // Brotli 压缩（可选，比 Gzip 更优）
    viteCompression({
      algorithm: 'brotliCompress',
      ext: '.br',
      threshold: 10240,
      deleteOriginFile: false,
      compressionOptions: { level: 11 }
    })
  ]
});
```

### 3. 服务器配置（关键）

压缩后的文件需要服务器配合才能生效，以 Nginx 为例：

```nginx
server {
  # 开启 Gzip
  gzip on;
  gzip_vary on;
  gzip_min_length 10240;
  gzip_types text/plain text/css text/xml text/javascript application/javascript application/json application/xml+rss;
  
  # 开启 Brotli（需要 ngx_brotli 模块）
  brotli on;
  brotli_comp_level 6;
  brotli_types text/plain text/css text/xml text/javascript application/javascript application/json application/xml+rss;
  
  # 优先使用 Brotli，其次 Gzip
  location ~* \.(js|css)$ {
    try_files $uri $uri.br $uri.gz =404;
  }
}
```

## 四、配套优化策略

### 1. 预加载首屏关键资源

用 `<link rel="preload">` 预加载首屏核心的 JS、CSS、字体文件，让浏览器优先加载这些资源：

```html
<!-- index.html -->
<head>
  <!-- 预加载首屏核心 JS -->
  <link rel="preload" href="/assets/js/index-abc123.js" as="script">
  <!-- 预加载首屏核心 CSS -->
  <link rel="preload" href="/assets/css/index-def456.css" as="style">
  <!-- 预加载首屏字体 -->
  <link rel="preload" href="/assets/font/main.woff2" as="font" type="font/woff2" crossorigin>
</head>
```

### 2. 图片优化

- 首屏图片用 WebP/AVIF 格式，体积减少 30%-50%；
- 非首屏图片用懒加载（`loading="lazy"`）；
- 小图标用 SVG 或雪碧图，减少请求数量；
- 图片用 CDN 加速，开启图片自适应裁剪。

### 3. CDN 加速

- 把打包后的静态资源（JS、CSS、图片、字体）上传到 CDN；
- 配置 `publicPath` 为 CDN 域名：
    
```javascript
    // vite.config.js
    export default defineConfig({
      base: 'https://static.xxx.com/' // CDN 域名
    });
```

### 4. 代码精简

- 开启 Tree Shaking（Vite 默认开启），删除未使用的代码；
- UI 组件库按需引入，不要全量引入；
- 用轻量库替代大库，比如用 `dayjs` 替代 `moment.js`，用 `lodash-es` 替代 `lodash`。

## 五、完整 `vite.config.js` 配置示例

```javascript
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';
import { resolve } from 'path';
import viteCompression from 'vite-plugin-compression';

export default defineConfig(({ mode }) => {
  const isProd = mode === 'production';
  return {
    plugins: [
      vue(),
      // 生产环境才开启压缩
      isProd && viteCompression({
        algorithm: 'gzip',
        ext: '.gz',
        threshold: 10240,
        deleteOriginFile: false,
        compressionOptions: { level: 9 }
      }),
      isProd && viteCompression({
        algorithm: 'brotliCompress',
        ext: '.br',
        threshold: 10240,
        deleteOriginFile: false,
        compressionOptions: { level: 11 }
      })
    ],
    resolve: {
      alias: {
        '@': resolve(__dirname, 'src')
      }
    },
    base: isProd ? 'https://static.xxx.com/' : '/', // 生产环境用 CDN
    build: {
      target: 'es2015',
      minify: 'terser',
      terserOptions: {
        compress: {
          drop_console: isProd, // 生产环境删除 console
          drop_debugger: isProd
        }
      },
      rollupOptions: {
        output: {
          manualChunks: {
            'vue-vendor': ['vue', 'vue-router', 'pinia'],
            'ui-vendor': ['element-plus', '@element-plus/icons-vue'],
            'chart-vendor': ['echarts'],
            'utils-vendor': ['axios', 'lodash-es', 'dayjs']
          },
          chunkFileNames: 'assets/js/[name]-[hash].js',
          entryFileNames: 'assets/js/[name]-[hash].js',
          assetFileNames: 'assets/[ext]/[name]-[hash].[ext]'
        }
      }
    }
  };
});
```

## 六、效果验证方法

优化完成后，用以下工具验证效果：

1. **Chrome DevTools Network 面板**：
    
    - 看首屏加载的资源数量是否减少；
    - 看资源体积是否明显变小（Gzip 压缩后）；
    - 看资源的 `from disk cache`/`from memory cache` 比例是否提升（依赖拆分后缓存效果）。
    
2. **Lighthouse**：
    
    - 看「Performance」分数是否提升；
    - 看「First Contentful Paint (FCP)」「Largest Contentful Paint (LCP)」「Time to Interactive (TTI)」等核心指标是否优化。
    
3. **Vite 打包分析**：
    
    安装 `rollup-plugin-visualizer` 插件，生成打包体积分析图，检查是否还有可优化的空间：
    
```bash
    npm install rollup-plugin-visualizer -D
```
    
```javascript
// vite.config.js
import { visualizer } from 'rollup-plugin-visualizer';
plugins: [visualizer({ open: true })]
```

## 总结

这套方案的核心逻辑是：

1. **路由懒加载**：首屏只加载必要的业务代码；
2. **`manualChunks` 拆分**：把依赖按稳定性分组，最大化利用浏览器缓存；
3. **Gzip/Brotli 压缩**：大幅减少资源体积；
4. **配套优化**：从资源加载、图片、CDN 等维度进一步提升。
