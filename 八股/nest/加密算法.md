Argon2 和 bcrypt 是目前**工业界最推荐的两种密码哈希算法**，专门用于安全存储用户密码。它们与 MD5、SHA-256 等普通哈希函数的核心区别是：**故意设计得计算缓慢且资源密集**，能够有效抵抗暴力破解、彩虹表攻击和 GPU/ASIC 加速攻击。

## 一、密码哈希的核心要求

在深入算法之前，先明确安全密码哈希必须满足的 4 个核心特性：

1. **单向性**：只能从密码计算哈希值，无法从哈希值反推原始密码
2. **抗碰撞性**：不同的密码几乎不可能产生相同的哈希值
3. **可配置的计算成本**：可以根据硬件发展调整计算难度，保持长期安全
4. **自动加盐**：每个密码使用唯一的随机盐值，防止彩虹表攻击

普通哈希函数（MD5、SHA-1、SHA-256）只满足前两个条件，计算速度太快，不适合存储密码。

## 二、bcrypt：成熟稳定的行业标准

bcrypt 是 1999 年由 Niels Provos 和 David Mazières 设计的密码哈希算法，基于 Blowfish 对称加密算法改进而来。它是目前使用最广泛、兼容性最好的密码哈希算法。

### 1. 核心设计思想

bcrypt 的核心创新是**自适应哈希函数**：它的计算成本可以随着硬件性能的提升而增加，确保即使在未来更快的计算机上，暴力破解仍然不可行。

### 2. 完整工作流程

bcrypt 的哈希过程分为 3 个主要阶段：

#### 阶段 1：生成随机盐值

- 自动生成一个 **128 位（16 字节）的加密安全随机盐值**
- 盐值是每个密码唯一的，即使两个用户使用相同的密码，最终的哈希值也会不同
- 盐值会直接存储在最终的哈希字符串中，不需要单独存储

#### 阶段 2：密钥扩展（EksBlowfish 算法）

这是 bcrypt 最核心的部分，也是它计算缓慢的原因：

1. 将用户密码和盐值作为输入，生成一个初始的 Blowfish 密钥
2. 对这个密钥进行 **2^cost 次迭代加密**（cost 就是成本因子）
3. 每次迭代都会修改密钥的状态，迭代次数越多，计算越慢
4. 最终生成一个经过扩展的密钥

#### 阶段 3：哈希计算

1. 使用扩展后的密钥加密一个固定的字符串：`"OrpheanBeholderScryDoubt"`
2. 连续加密 64 次，得到最终的 192 位（24 字节）哈希值
3. 将版本号、成本因子、盐值和哈希值拼接成最终的字符串

### 3. bcrypt 哈希字符串格式

bcrypt 的最终输出是一个长度为 60 个字符的字符串，格式如下：

```plaintext
$2a$10$N9qo8uLOickgx2ZMRZo5IeZOg9zIu0pJ7xTiVf5QW4S6UaG9t7u5m
└─┘ └┘ └──────────────────────┘└────────────────────────────┘
  │  │            │                        │
版本号 成本因子      盐值(22字符)            哈希值(31字符)
```

- **版本号**：`$2a$`、`$2b$`、`$2y$` 都是常见的版本，修复了历史上的一些漏洞
- **成本因子**：范围 4-31，默认值 10，计算时间随 2^cost 指数增长
- **盐值**：16 字节原始盐值经过 base64 编码后得到 22 个字符
- **哈希值**：24 字节原始哈希值经过 base64 编码后得到 31 个字符

### 4. 密码验证过程

验证密码时，bcrypt 不需要解密哈希值：

1. 从存储的哈希字符串中提取版本号、成本因子和盐值
2. 使用相同的参数和用户输入的密码，重新计算哈希值
3. 比较新计算的哈希值和存储的哈希值是否一致
4. 如果一致，密码正确；否则，密码错误

### 5. bcrypt 的局限性

- **最大密码长度限制**：只使用密码的前 72 字节，超过部分会被忽略
- **仅针对 CPU 优化**：对 GPU 和 ASIC 加速攻击的抵抗能力不如 Argon2
- **内存占用固定**：无法调整内存使用量，容易被专用硬件并行破解

## 三、Argon2：现代密码哈希的黄金标准

Argon2 是 2015 年 **密码哈希竞赛 (PHC)** 的获胜者，是目前最安全、最先进的密码哈希算法。它专门针对现代硬件设计，能够同时抵抗 CPU、GPU 和 ASIC 攻击。
### 1. Argon2 的三个版本

Argon2 提供了三个不同的版本，适用于不同的安全场景：

|版本|特点|适用场景|
|---|---|---|
|**Argon2d**|优先抵抗 GPU 攻击，使用数据相关的内存访问|加密货币、区块链等不需要抵抗侧信道攻击的场景|
|**Argon2i**|优先抵抗侧信道攻击，使用数据无关的内存访问|密码存储、身份验证等需要抵抗侧信道攻击的场景|
|**Argon2id**|混合版本，结合了 Argon2d 和 Argon2i 的优点|**推荐使用**，适用于绝大多数场景|
### 2. 核心设计思想

Argon2 的核心创新是**内存难函数 (Memory-Hard Function)**：它不仅消耗 CPU 时间，还消耗大量内存，使得攻击者需要同时投入大量的计算资源和内存资源才能进行暴力破解，大大提高了攻击成本。

### 3. 完整工作流程

Argon2 的哈希过程比 bcrypt 更复杂，分为 4 个主要阶段：

#### 阶段 1：参数初始化

输入参数包括：

- 密码（任意长度）
- 盐值（16 字节以上的随机值）
- 时间成本（迭代次数 t）
- 内存成本（使用的内存大小 m，单位 KB）
- 并行度（使用的线程数 p）
- 输出哈希长度（默认 32 字节）

#### 阶段 2：内存填充

1. 根据内存成本参数 m，分配一块大小为 m KB 的连续内存
2. 将内存划分为多个 1KB 的块，形成一个二维数组
3. 使用密码、盐值和其他参数，通过伪随机函数生成初始块
4. 按照特定的顺序，对内存块进行多次迭代填充和更新
5. 每次迭代都会读取和写入多个内存块，使得内存访问模式不可预测

#### 阶段 3：压缩函数

1. 对填充后的内存块进行多次压缩操作
2. 每次压缩操作都会混合多个内存块的内容
3. 压缩过程同时消耗 CPU 时间和内存带宽

#### 阶段 4：输出哈希

1. 从最终的内存状态中提取出一个摘要
2. 对摘要进行最终的哈希计算，得到指定长度的输出哈希值
3. 将所有参数、盐值和哈希值拼接成最终的字符串

### 4. Argon2 哈希字符串格式

Argon2 的最终输出是一个标准的编码字符串，格式如下：

```plaintext
$argon2id$v=19$m=65536,t=3,p=4$c29tZXNhbHQ$RdescudvJCsgt3ub+b+dWRWJTmaaJObG
└─────┘ └──┘ └──────────────┘ └────────┘ └──────────────────────────────┘
   │     │         │              │                      │
 版本   算法版本   参数(m,t,p)     盐值(base64)           哈希值(base64)
```

- **版本**：`argon2d`、`argon2i`、`argon2id`
- **算法版本**：当前最新版本是 19（0x13）
- **参数**：`m`= 内存成本 (KB)，`t`= 时间成本 (迭代次数)，`p`= 并行度 (线程数)
- **盐值**：随机盐值经过 base64 编码
- **哈希值**：最终哈希值经过 base64 编码

### 5. Argon2 的优势

- **同时抵抗 CPU、GPU 和 ASIC 攻击**：内存难函数设计使得专用硬件的优势大大降低
- **高度可配置**：可以独立调整时间、内存和并行度三个参数，适应不同的硬件和安全需求
- **无密码长度限制**：支持任意长度的密码
- **标准化**：已经被纳入多个国际标准（如 RFC 9106）
- **更安全的设计**：经过密码学界的广泛审查和分析

## 四、bcrypt vs Argon2：全面对比

|特性|bcrypt|Argon2|
|---|---|---|
|发布时间|1999 年|2015 年|
|设计目标|抵抗 CPU 攻击|同时抵抗 CPU、GPU 和 ASIC 攻击|
|可配置参数|仅成本因子 (cost)|时间 (t)、内存 (m)、并行度 (p)|
|内存使用|固定约 4KB|可配置 (64KB - 几 GB)|
|密码长度限制|72 字节|无限制|
|抗 GPU 攻击能力|弱|强|
|抗侧信道攻击能力|中等|强 (Argon2i/Argon2id)|
|成熟度|极高，经过 20 多年验证|高，经过密码学界广泛审查|
|兼容性|极好，几乎所有语言和框架都支持|好，大多数现代语言和框架都支持|
|推荐指数|⭐⭐⭐⭐|⭐⭐⭐⭐⭐|
### 选择建议

- **新项目优先选择 Argon2id**：这是目前最安全、最推荐的密码哈希算法
- **兼容旧系统使用 bcrypt**：如果需要兼容旧的系统或框架，可以使用 bcrypt
- **绝对不要使用**：MD5、SHA-1、SHA-256 等普通哈希函数，即使加盐也不安全

## 五、JavaScript 实现代码

### 1. bcrypt 实现（使用 bcryptjs 库）

bcryptjs 是纯 JavaScript 实现的 bcrypt 库，不需要编译，兼容性最好。

```bash
npm install bcryptjs
```

```javascript
const bcrypt = require('bcryptjs');

/**
 * 生成 bcrypt 哈希
 * @param {string} password - 原始密码
 * @param {number} cost - 成本因子，默认10，范围4-31
 * @returns {string} 哈希字符串
 */
async function hashPassword(password, cost = 10) {
  // 自动生成盐值并哈希密码
  const hash = await bcrypt.hash(password, cost);
  return hash;
}

/**
 * 验证 bcrypt 密码
 * @param {string} password - 用户输入的密码
 * @param {string} hash - 存储的哈希字符串
 * @returns {boolean} 密码是否正确
 */
async function verifyPassword(password, hash) {
  const isValid = await bcrypt.compare(password, hash);
  return isValid;
}

// 使用示例
async function main() {
  const password = 'mySecurePassword123';
  
  // 生成哈希
  const hash = await hashPassword(password);
  console.log('哈希值:', hash);
  // 输出示例: $2a$10$N9qo8uLOickgx2ZMRZo5IeZOg9zIu0pJ7xTiVf5QW4S6UaG9t7u5m
  
  // 验证密码
  const isValid = await verifyPassword(password, hash);
  console.log('密码是否正确:', isValid); // true
  
  const isInvalid = await verifyPassword('wrongPassword', hash);
  console.log('错误密码是否通过:', isInvalid); // false
}

main();
```

### 2. Argon2 实现（使用 argon2 库）

argon2 是基于 WebAssembly 实现的 Argon2 库，性能最好，支持所有三个版本。

```bash
npm install argon2
```

```javascript
const argon2 = require('argon2');

/**
 * 生成 Argon2id 哈希（推荐）
 * @param {string} password - 原始密码
 * @returns {string} 哈希字符串
 */
async function hashPassword(password) {
  const hash = await argon2.hash(password, {
    type: argon2.argon2id, // 使用推荐的 Argon2id 版本
    memoryCost: 65536,    // 内存成本：64MB (65536 KB)
    timeCost: 3,          // 时间成本：3次迭代
    parallelism: 4,       // 并行度：4个线程
    hashLength: 32        // 输出哈希长度：32字节
  });
  return hash;
}

/**
 * 验证 Argon2 密码
 * @param {string} password - 用户输入的密码
 * @param {string} hash - 存储的哈希字符串
 * @returns {boolean} 密码是否正确
 */
async function verifyPassword(password, hash) {
  const isValid = await argon2.verify(hash, password);
  return isValid;
}

// 使用示例
async function main() {
  const password = 'mySecurePassword123';
  
  // 生成哈希
  const hash = await hashPassword(password);
  console.log('哈希值:', hash);
  // 输出示例: $argon2id$v=19$m=65536,t=3,p=4$c29tZXNhbHQ$RdescudvJCsgt3ub+b+dWRWJTmaaJObG
  
  // 验证密码
  const isValid = await verifyPassword(password, hash);
  console.log('密码是否正确:', isValid); // true
  
  const isInvalid = await verifyPassword('wrongPassword', hash);
  console.log('错误密码是否通过:', isInvalid); // false
}

main();
```
